# 资源调度模块标签体系

**标签**：#资源调度 #标签系统 #DDD #Incident #状态机 #闭环
**位置**：05_DataModel/

---

## 设计思路

- **视角**：以"资源调度决策"为核心，标注需要调什么资源、调度状态、执行效果
- **核心原则**：5-7个主标签，强关联Incident聚合根，突出可执行性和 traceable
- **关键价值**：快速过滤"已确认需求、可立即执行"的调度方案

---

## Type（内容形态）

| 标签 | 说明 |
|------|------|
| #type/dispatch-plan | 资源调度方案（核心） |
| #type/resource-req | 具体资源需求清单 |
| #type/assignment | 资源分配与任务指派 |
| #type/execution | 执行记录与反馈 |
| #type/feedback | 现场执行反馈 |
| #type/summary | 调度总结报告 |

## State（工作流状态）

| 标签 | 说明 |
|------|------|
| #state/Pending | 待调度（已接收研判结论） |
| #state/Assigned | 已分配资源与任务 |
| #state/Enroute | 资源在途 |
| #state/Arrived | 资源到达现场 |
| #state/Executing | 正在执行 |
| #state/Completed | 调度完成 |
| #state/rollback | 需要回退调整 |
| #state/proposed | 提案阶段 |
| #state/accepted | 已正式采纳 |

| 标签 | 说明 |

| #source/field | 现场实时反馈 |
| #source/command | 指挥中心指令 |
| #source/norm | 预案或标准流程 |

## Topic（领域主题）

| 标签 | 说明 |
|------|------|
| #topic/资源调度 | 核心标签 |
| #topic/资源需求 | 资源需求相关 |
| #topic/任务指派 | 任务分配相关 |
| #topic/执行反馈 | 执行情况反馈 |

---

## Value / Priority

| 标签 | 说明 |
|------|------|
| #value/urgent | 紧急调度（需立即执行） |
| #value/high-priority | 高优先级 |
| #value/critical | 重大/高危场景 |
| #value/routine | 常规调度 |

---

## Transition（状态转换）

| 标签 | 说明 |
|------|------|
| #transition/Confirmed→Pending | 已接收研判结论，进入调度 |
| #transition/Pending→Assigned | 完成资源分配 |
| #transition/Assigned→Enroute | 资源出发 |
| #transition/Enroute→Arrived | 资源到达 |
| #transition/Executing→Completed | 执行完成 |
| #transition/rollback | 需要回退调整 |

---

## 三模块闭环总结

| 模块 | 核心输出状态 | 下一模块输入标签 | 关键Transition |
|------|-------------|-----------------|----------------|
| 接警问询 | #state/Assessed | #source/inquiring | Inquiring→Assessed |
| 警情研判 | #state/Confirmed | #source/assessment | Preliminary→Deep / Deep→Confirmed |
| 资源调度 | #state/Assigned / Completed | - | Confirmed→Pending / Pending→Assigned |

---

## 过滤规则

| 优先级 | 标签组合 | 动作 |
|--------|----------|------|
| 可立即执行 | #state/Assigned + #transition/Pending→Assigned + #value/urgent | 直接执行 |
| 重点关注 | #transition/rollback + #value/critical | 立即调整 |
| 可复盘 | #state/Completed + #type/feedback | 事后复盘 |

## Dataview 查询

```dataview
TABLE current_state, risk_level, resources, eta, file.mtime
FROM "04-调度"
WHERE contains(tags, "#topic/资源调度") AND contains(tags, "#state/Assigned")
SORT eta ASC, risk_level DESC
```
