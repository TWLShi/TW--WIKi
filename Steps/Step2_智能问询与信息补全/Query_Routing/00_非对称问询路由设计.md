## 1. 核心设计理念

**非对称问询路由** —— 用极低成本（1-3秒）过滤器，保护后续高成本资源

| 维度 | 说明 |
|------|------|
| 本质 | 80/20法则工程化落地 |
| 核心 | 高价值不对称博弈 |
| 目标 | 火灾15秒首调达成率从70-80%提升至95%+ |

**设计背景**：
- 火灾警情属于"不可逆高损失"类型（火势指数级蔓延、人命窗口极短），必须在15-20秒内完成首调
- 社会救助类警情（开锁、马蜂窝等）时效要求低，却占用同等结构化问询资源
- 导致系统负荷高、接警员疲劳、高危警情被稀释

**核心原则**：先定大类，再结构化；火灾抢空间，救助抢边界

## 2. 20/80 原则（帕累托法则）

**核心定义**：80% 的结果来自 20% 的原因

### 在消防接处警中的应用

```
┌─────────────────────────────────────────┐
│           全部警情 (100%)                │
├─────────────────────────────────────────┤
│  火灾类 (20%)   │   社会救助类 (80%)     │
│  ─────────────  │   ─────────────       │
│  高损失         │   低损失               │
│  短时效         │   长时效               │
│  生命危险       │   财产/动物            │
│  ═══════════════╪═════════════════════  │
│  消耗 80%       │   消耗 20%             │
│  算力/人力      │   算力/人力            │
└─────────────────────────────────────────┘
```

### 优化思路

用 20% 的成本（1-3秒分流）筛选出 20% 的高价值警情，优先保护：

| 资源 | 分配给 |
|------|--------|
| 80% 算力 | 火灾/高危抢险（20%警情） |
| 20% 算力 | 社会救助（80%警情） |

**本质**：把正确的事以极低成本先做对

## 3. 开场白与首句分流（0-3秒，极速二选一）

### 标准话术（最终推荐版）
> "你好119，是火灾还是其他救助？"

### 逻辑设计要点
- 二选一框架：强制报警人快速定性，降低认知负荷
- "火灾"前置：利用强语感实现不对称分流
- 分流结果：
  - **火灾** → 立即进入**红色路径**（火灾特权路由）
  - **其他救助** → 进入**蓝色路径**（常态救助路由）
  - **非职责/无关** → 直接拦截，引导转警或挂断，不生成建模

---

## 4. 高权重词库（NLP引擎底层支撑）

用于AI后台实时提取、置信度计算与强制路径锁定。

| 类别 | 极高权重（0.95-1.0） | 高权重（0.85-0.95） | 系统动作触发 |
|------|----------------------|----------------------|-------------|
| **火灾扑救** | 火光冲天、熊熊大火、火舌窜出、整栋楼烧着了 | 黑烟滚滚、冒火星子、浓烟封路、烟很大出不去了 | 强制红色路径，点亮火团图标 |
| **抢险救援** | 电梯坠落、坍塌埋压、车祸被困、溺水、跳楼轻生 | 煤气泄漏、有毒气体、化学品翻车、手脚被卡 | 强制高时效路径，点亮被困/危化图标 |
| **社会救助** | 马蜂窝（巨大）、反锁家中（有老人小孩）、进水排水 | 蛇进家、树倒压车、广告牌要掉 | 蓝色路径，低资源占用 |
| **理化/专项物质** | 油库、加气站、氢气瓶、锂电池起火、充电桩爆炸 | 苯、甲醇、盐酸、放射性物质 | 激活对应专项场景卡（化工/新能源） |
| **报警人状态/背景音** | 救命啊！（惨叫）、要炸了！（极速语速） | 爆炸声、玻璃破碎、火啸声、警报器响 | 置信度强制补满，禁止路径降级 |

### 算法逻辑
- 单一高权重词 → C ≥ 0.85
- 组合词 → C ≥ 0.98 → UI强制反馈（按钮闪绿、场景预激活）

## 5. 路由路径总览

| 时序 | 红色路径（火灾确认） | 蓝色路径（其他救助） |
|------|----------------------|----------------------|
| 0-3s | 开场白二选一 → 火灾确认 | 开场白二选一 → 救助确认 |
| 3-10s | **地址霸权**：强制问地址 | **边界核实**：问"具体是什么事？有危险吗？" |
| 10-15s | 一号调派（15s KPI） | 通过职责判定后才问地址（仅2D定位） |
| 15-70s | 结构化画像：人命 → 物理属性 → 理化本质 | 按需调派（放入非紧急任务池） |
| 系统资源 | 100%高算力：3D建模、CAD加载、消火栓水压实时拉取 | 轻量化：仅文字+2D平面定位 |

## 6. 火灾确认机制（准火灾 → 正式火灾）

**双重激活逻辑**（人机并行，防止误判/漏判）：

### 6.1 人工确认
接警员听到高权重词 → 点击"火灾确认"按钮 → 激活正式火灾场景

### 6.2 AI自动确认
- 高权重词置信度 > 阈值 → "火灾按钮"闪绿（准火灾场景）
- 准火灾场景下：
  - 推荐首问与正式相同（地址问询）
  - 接警员若开始问地址，或在准场景录入任何信息 → 自动转为正式火灾场景
  - 若需回滚，必须手动操作（但系统记录审计）

### 6.3 模糊核实话术
> "请核实：您现在是看到火了，还是只看到烟？"

| 回答 | 结果 |
|------|------|
| 看到火 | 确认火灾（C=1.0，强制红色路径） |
| 只看到烟 | 疑似火灾（C=0.7，侦察调派） |
| 都不是 | 路径切换至蓝色抢险/救助 |

### 6.4 模糊警情二次判定补偿
- 并行补盲：CCTV烟火识别、多路报警合并、重点单位强制高危预案
- 拦截规则：听到"炸了""救命"禁止降级；否定火灾必须追加"有人被困/危险品泄漏吗？"

## 7. 火灾确认后结构化问询（红色路径核心）

### 通用首问（地址霸权，最终推荐版）
> "消防救援，请讲明：是居民家还是单位着火？在哪个地址？"

**逻辑优势**：
- 二选一去行政歧义：居民答"家里" → 激活居民住宅卡；单位答"工厂" → 激活化工/工贸卡
- 其他答"路边/商场" → 激活对应外部/商业卡
- 兼顾地址强制获取

### 场景卡切换后专项问询示例

| 场景 | 专项问询 |
|------|----------|
| 高层/居民住宅 | 几层着火？有人被困吗？着火部位是厨房/卧室？ |
| 高速/高架 | 您在哪个方向（南向北）？最近下匝道？路边K标多少？ |
| 大跨度/园区 | 在园区哪个方位？几号门进最快？ |
| 化工/单位 | 着火物质是什么？有无危险品储存？ |

**问询顺序铁律**：地址 → 人命状态（Who） → 物理属性（层数/材质） → 理化本质（RSP/危险品）

## 8. 全流程KPI与审计要点

### 核心KPI

| 指标 | 合格线 | 满分线 |
|------|--------|--------|
| 定性耗时 | ≤15秒 | ≤10秒 |
| 地址抢占率 | 火灾定性后5秒内问地址 | 即时 |
| 路径切换速度 | 识别后快速切换 | 即时 |

### 审计维度与权重

| 维度 | 权重 | 考核要点 | 违规示例 | 预警类型 |
|------|------|----------|----------|----------|
| 路径切换速度 | 30% | 识别后快速切换路由 | 火灾已提却仍问身份 | 黄色 |
| 地址抢占率 | 40% | 火灾定性后5秒内必须问地址 | 火灾确认10秒后仍未问地址 | 红色 |
| 原语提取精度 | 20% | 高权重词出现后是否激活场景卡 | 听到"冒黑烟"却未提级 | 黄色 |
| 话术简洁度 | 10% | 避免重复/无效问询 | 对马蜂窝问"建筑耐火等级" | 黄色 |

### 实时预警

| 预警 | 触发条件 |
|------|----------|
| 🔴 红色 | 火灾定性10秒内未问地址 |
| 🟡 黄色 | 救助场景问火灾专属问题 |
| 🔵 蓝色 | 报警人喊"人命关天"仍死问火灾大类 |

**审计终极目标**：确保资源向最高时效倾斜，保护算力，降低无效疲劳，提升15秒首调率。

## 10. 风险对冲

| 风险 | 场景 | 对冲措施 |
|------|------|----------|
| 误分流 | 报警人极度慌乱，只说"救命！有人掉河里了！" | AI监听"困/烧/烫/电/爆"等关键词，强制锁火灾路径 |
| 中间地带 | 化学品泄漏、高压电击 | 增加"第三路径"或追问"是人命危险还是财产/动物？" |
| 报警人心理摩擦 | 少数人急到直接喊具体情况绕过选择 | 话术微调，语气更柔 |
