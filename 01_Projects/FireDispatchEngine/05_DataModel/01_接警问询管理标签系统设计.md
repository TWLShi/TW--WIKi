# 接警问询管理标签系统设计

**标签**：#DDD #标签系统 #接警问询 #Frontmatter #统一语言
**位置**：05_DataModel/

---

## 设计原则

- **领域驱动**：紧扣统一语言（GL-001~GL-017）和核心聚合根
- **分层分类**：便于检索、过滤、统计
- **可扩展**：支持后续新增标签
- **前缀规范**：`inq:` 表示问询模块专属标签

## Frontmatter 模板

```yaml
---
title: 【接警问询】xxx功能/场景/记录
document_id: DES-INQ-xxx
version: v1.0
created_date: 2026-04-04
last_updated: 2026-04-04
author: [你的名字]
module: 接警问询管理

# 核心标签（必填，至少3个）
tags:
  - inq:问询集
  - inq:正式立案
  - inq:状态-问询中
  - inq:围栏判定
  - inq:灾害类型-火灾
  - inq:智能体推送

# 业务状态
inquiry_status: 立案状态已判定
filing_status: FORMAL_FILING  # PRE_FILING / FORMAL_FILING / NOT_FILING / INQUIRY_FAILURE
call_characterization: FIRE    # FIRE / RESCUE / SOCIAL_RESCUE / HARASSMENT

# 关键要素
key_elements:
  - 警情地址
  - 报警人位置
  - 支队辖区
  - 消防站辖区

# 异常相关
anomaly_type: 通话中断          # 无声 / 通话中断 / 网络中断 / 无
anomaly_recovered: true        # 是否已恢复

# 队列与决策
queue: 正式立案队列
merge_candidate: true          # 是否为归并候选

# 评审维度
review_dimension:
  - 事件分配队列准
  - 处理事件快
  - Q/A记录没有错

# 关联信息
related_gl: ["GL-010", "GL-011", "GL-013"]
related_aggregate: 接警问询聚合根
related_methods:
  - recordPushedElements
  - determineFilingStatus
  - handleCallAnomalyRecovery

priority: 高
review_status: 已评审
---
```
