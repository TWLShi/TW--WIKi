# 警情研判模块标签体系

**标签**：#警情研判 #标签系统 #DDD #Incident #状态机
**位置**：05_DataModel/

---

## 设计思路

- **视角**：以"研判需要什么决策输入"为导向
- **核心原则**：克制（5-7个标签）、前缀统一、状态机延续、强关联 Incident 聚合根
- **关键价值**：研判输出成为资源调度/出警/处置的高质量输入

---

## Type（内容形态）

| 标签 | 说明 |
|------|------|
| #type/assessment | 警情等级评估与风险研判 |
| #type/elements | 结构化警情要素（从接警继承） |
| #type/recommend | 处置/资源调度推荐 |
| #type/summary | 研判摘要（供指挥快速阅读） |
| #type/report | 完整研判报告（含依据、串并案分析） |
| #type/decision | 最终决策记录（等级/资源/优先级） |

---

## State（工作流状态）

| 标签 | 说明 |
|------|------|
| #state/Assessing | 研判进行中 |
| #state/Assessed | 研判完成，可进入资源调度 |
| #state/rollback | 需要回退补充 |
| #state/proposed | 初步提案阶段 |
| #state/accepted | 已通过指挥评审 |
| #state/closed | 警情处置完成（归档） |

---

## Source（来源）

| 标签 | 说明 |
|------|------|
| #source/inquiry | 来自接警问询 |
| #source/history | 历史警情/串并案 |
| #source/norm | 消防/公安规范、预案 |
| #source/ai | AI 辅助研判结果 |
| #source/field | 出警反馈或现场补充 |

---

## Topic（领域主题）

| 标签 | 说明 |
|------|------|
| #topic/警情研判 | 研判模块核心 |
| #topic/接警问询 | 与接警强关联（继承） |
| #topic/资源调度 | 输出给调度模块 |
| #topic/状态机 | 状态迁移记录 |
| #topic/risk | 风险相关 |

---

## 进阶标签

| 标签 | 说明 |
|------|------|
| #transition/Inquiring→Assessed | 接警转换完成 |
| #transition/Assessing→Assessed | 研判自身转换完成 |
| #transition/Assessed→Dispatch | 可进入资源调度 |
| #value/high-signal | 高信号警情（优先研判） |
| #value/critical | 高危（伤亡/燃气/爆炸等） |
| #risk/level-high/medium/low | 风险等级 |
| #decision/grade-1/2/3/4 | 警情等级（1最高） |

---

## 过滤规则

| 优先级 | 标签组合 | 动作 |
|--------|----------|------|
| **优先处理** | #state/Assessed + #transition/Assessing→Assessed + #transition/Assessed→Dispatch | 直接生成调度指令 |
| **重点关注** | #transition/rollback + #value/critical + #risk/level-high | 立即补充或升级处置 |

---

## Dataview 查询示例

```dataview
TABLE current_state, risk_level, grade_suggestion, resource_recommend, file.mtime
FROM "03-研判"
WHERE contains(tags, "#topic/警情研判") AND contains(tags, "#state/Assessed")
SORT risk_level DESC, file.mtime DESC
```

## 全流程闭环

```
接警（Inquiring→Assessed）→ 研判（Assessing→Assessed→Dispatch）→ 资源调度
```
