## 1. 场景特点与挑战

### 接警场景核心特性
- 单条具体问题（位置/有人受困/烟火情况）
- 单一意图
- 回答大概率直接对应但混杂无关内容

### 接警挑战
| 挑战 | 说明 |
|------|------|
| 噪声环境 | 警笛、哭喊、背景喧哗 |
| 方言混杂 | 大陆普通话+地方口音 |
| 术语专业 | "浓烟滚滚""被困""低洼" |
| 实时性 | 必须低延迟（<600ms） |

---

## 2. 系统架构

### 高阶架构

```
前端ASR层 → 中端上下文引导层 → 后端KG驱动层 → 部署层
```

### 前端ASR层
- FunASR Paraformer-streaming（大陆主流开源）
- 或iFLYTEK商用版
- 问题嵌入热词+上下文

### 中端引导层
- 当前问题文本 → 生成槽位schema
- N-best重打分 + 相关性过滤

### 后端KG驱动层
- RDF/OWL/SHACL/SWRL本体
- LLM Agent
- 只对匹配片段提取RDF triples

## 3. 槽位定义（三个核心槽位）

| 槽位 | 问题示例 | 提取内容 |
|------|----------|----------|
| **位置（hasLocation）** | "位置在哪？" | 地名+区域特征（低洼/高层/地下室） |
| **被困（hasTrappedPeople）** | "有没有人受困？" | 人数+状态（紧急/可等待） |
| **烟火（hasFireSmokeStatus）** | "烟火情况怎么样？" | 程度+类型（浓烟/明火/爆炸风险） |

## 6. 知识蒸馏方案

### Teacher→Student架构
- **Teacher**：LLM+KG pipeline（复杂但一次性）
- **Student**：TinyBERT/DistilBERT/FunASR-small（轻量部署）

### 蒸馏策略
| 策略 | 说明 |
|------|------|
| 数据蒸馏 | Teacher生成伪标签，Student监督微调 |
| 特征蒸馏 | Student模仿Teacher中间层embedding |
| 规则注入 | SHACL-guided prompting生成数据 |

### 量化预期
- WER：Teacher ~8-10% → Student ~10-13%
- 槽位F1：>90%
- 模型参数量缩小5-10x
