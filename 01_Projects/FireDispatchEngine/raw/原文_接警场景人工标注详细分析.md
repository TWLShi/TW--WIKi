# 中国大陆119/110/120接警场景下人工标注工作的详细分析

**来源**：接警场景人工标注工作详细分析
**版本**：V1.0
**定位**：专为单问题引导槽位填充（位置/有人受困/烟火情况）+ 多语言适配 + 原ASR + 蒸馏小模型系统设计

---

## 1. 必要性（为什么必须人工标注？）

| 必要性 | 说明 |
|--------|------|
| **蒸馏核心依赖** | 小模型（Student）需要Teacher提供的**软标签（soft labels）+ 结构化gold slots**，而Teacher本身会产生幻觉，尤其在噪声转录 + 无关内容混杂 + 跨语言场景下 |
| **接警场景特性** | 单问题意图唯一，但回答碎片化、情绪化、方言重，纯自动/规则无法处理隐含表达（如"孩子出不来"→ 被困槽位） |
| **政策合规** | 应急管理部《智慧应急语音数据集建设指南》（2023-2025）明确要求"关键要素人工校验" |
| **ROI起点** | 标注一次投入，可支撑后续所有迭代，边际成本趋近于0 |

---

## 2. 标注内容（精确到每个槽位的标注粒度）

标注采用**句子级/片段级联合标注**（实体 + 槽位 + 关系 + 相关性），格式为JSONL

| 标注类型 | 具体标注项（针对单问题） | 示例（问题："烟火情况怎么样？"） | 标注要求 | 与本体/RDF映射 |
|----------|--------------------------|-------------------------------------|----------|----------------|
| **槽位实体** | 位置（hasLocation + 区域特征）、被困（hasTrappedPeople + 状态）、烟火（hasFireSmokeStatus + 程度/类型） | "浓烟滚滚，地下室出不来" | BIOES标签 + 属性（低洼/高层/人数/明火/浓烟） | 直接填RDF triple |
| **关系** | 因果/蕴含/部分（causes / implies / hasPart） | "烟大 → 无法出去" | 箭头标注（source → target） | SWRL规则验证 |
| **相关性** | 相关 / 无关 / 部分相关 | "我好怕…浓烟很大" → "浓烟很大"=相关，其余=无关 | 二分类 + 置信度（0-1） | 过滤无关内容 |
| **多语言** | 跨语实体对齐 + 翻译回退标签 | 粤语"喺边度" → 中文"位置：XX路" | 保留原语 + 中文标准化 | skos:altLabel |
| **噪声修正** | ASR转录错误修正 | 错认"浓烟"→"农民" | 标注员手动修正后标注 | 提升蒸馏鲁棒性 |

**标注粒度**：每条回答拆成**最小相关片段**（span），避免整句标注导致噪声污染

## 3. 标注流程（标准化SOP，5步闭环）

```
1. 预处理（自动）：FunASR转录 + 当前问题嵌入 + 初步本体匹配
2. 初标（标注员）：用工具标注槽位/关系/相关性（每人每天80-120条）
3. 交叉校验（双人）+ IAA计算
4. 专家审核（消防/接警专家抽查20%）
5. SHACL验证（自动）：违反本体约束的自动退回重标
6. 入库：生成Teacher软标签数据集（JSON + soft probabilities）
```

**单条耗时估算**：普通话1.5-2.5分钟/条；方言/外语3-4分钟/条

## 4. 规模、时间与人力投入（量化估算）

### 初始规模（阶段2核心产出）

| 语言 | 数量 | 说明 |
|------|------|------|
| 普通话 | 6000-8000条 | 覆盖位置/被困/烟火各2000条 + 混合无关内容 |
| 多语言 | 2000-3000条 | 优先粤语/四川话/英语 + 维语/藏语各300条 |
| **总计** | **8000-11000条** | 足够支撑蒸馏 + 主动学习 |

### 人力配置

| 角色 | 配置 | 说明 |
|------|------|------|
| 标注员 | 3-4名 | 2名全职 + 2名兼职，具备消防/接警背景最佳 |
| 专家审核 | 1名 | 每周10小时 |
| 总人月 | 约4-6人月 | 含培训 |

### 时间与成本

| 维度 | 数值 |
|------|------|
| 时间 | 4-6周（并行标注 + 每周2次IAA反馈） |
| 标注员成本 | 8000-12000元/人/月 → 总人力约4-6万元 |
| 工具/云存储 | ＜5000元 |
| **总投入** | **5-7万元**（占整个系统首3个月预算的20-25%） |

## 6. 质量控制（IAA与迭代机制）

### IAA指标（必须达标）

| 指标 | 要求 |
|------|------|
| 槽位实体 | Cohen's Kappa > 0.85 |
| 关系/相关性 | Kappa > 0.80 |
| 多语言对齐 | Kappa > 0.75 |

### 机制

- 每周计算IAA，不达标 → 返工 + 指南更新
- 黄金集（200条专家共识）作为每周基准
- 主动学习循环：小模型上线后，新接警日志 → 高不确定性样本自动推给标注员（减少50%标注量）

## 7. 多语言特殊标注要求

| 要求 | 说明 |
|------|------|
| **语言ID + 原语保留** | 增加标签 |
| **翻译+对齐策略** | 标注员先标中文标准化槽位，再回标原语span |
| **低资源语言** | 维语/藏语：可先用Qwen翻译辅助标注，再人工修正（节省30%时间） |
| **方言口语化** | "烟好大啊"需额外标注**口语-标准映射** |
