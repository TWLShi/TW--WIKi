## 1. 核心定位

**FRICP = Fire Rescue Integrated Command Platform**

在已有接处警系统 + 现场指挥系统 + 舆情模块基础上做**极简扩展**，不新增复杂采集模块。

---

## 2. 极简原则（关键区别）

| 排除项 | 原因 |
|--------|------|
| 不接入对讲机 | 简化集成复杂度 |
| 不做视频分析 | 降低成本与风险 |
| 不做物联感知 | 避免过度复杂化 |
| **无大模型** | 核心差异点 |

**新增**：人工录入事实功能（解决伤亡数据难获取问题）

---

## 3. 三个有界上下文架构

| 上下文 | 职责 |
|--------|------|
| **事实核实上下文** | 客观事实核实 + 人工录入 |
| **报告生成上下文** | 核心最严格，只允许事实部分 |
| **合规审计上下文** | 全局存证追溯 |

**数据流向**：单向流动，责任边界清晰

---

## 4. 职责清晰模式

```
前场：只打仗（救援）
后场：管指挥与上报
```

- 后方指挥中心承担**全部上报责任**
- 现场**零上报责任**
- 外部数据必须标注"消防不承担责任"

---

## 5. 后方指挥中心核心功能

| 功能 | 说明 |
|------|------|
| 接处警扩展 | 自动提示七类敏感情况和较大/重大事故 |
| 重点单位消控室联系电话 | 一键拨打与核实 |
| **人工录入信息功能** | 后方手动补充客观事实（伤亡、被困等） |
| 智能报告模板 | 自动填充事实部分 + 指挥员人工审核确认 |
| 50分钟倒计时提醒 | + 一键上报 + 实时共享 |
| 外部数据责任标注 | 自动标注 |

## 6. 成本预算（最终极简版）

| 阶段 | 金额 |
|------|------|
| **MVP阶段** | 约180万元 |
| **完整版** | 约265万元 |
| **3年运维** | 每年约38万元 |
| **总计** | **约387万元** |

**成本优化亮点**：复用现有系统 + 取消高成本模块 + 人工录入替代复杂分析

---

## 7. 分阶段迭代规划

| 迭代 | 周期 | 内容 |
|------|------|------|
| 迭代0 | 2-3周 | 基础集成 + 权限复用 + 数据标注 |
| 迭代1 | 4-6周 | MVP核心（人工录入 + 报告模板 + 一键上报） |
| 迭代2（可选） | 4-6周 | 动态续报 + 舆情辅助优化 |

## 9. 与FireDispatchEngine的关系

| 维度 | FRICP | FireDispatchEngine |
|------|-------|-------------------|
| 定位 | 极简扩展 | 智能调派引擎 |
| 核心 | 合规报告 + 职责隔离 | 12步智能决策闭环 |
| 技术路线 | 无大模型、人工录入 | AI多智能体、ML混合 |
| 适用范围 | 信息报告与指挥 | 灭火救援调派 |

**互补关系**：FRICP解决报告合规，FireDispatchEngine解决智能调派

---

## 10. 实施建议

- 建设路径：以现有接处警系统为底座，做最小化模块扩展
- 试点策略：先在支队指挥中心试点火灾场景
- 保障措施：每个迭代结束进行合规审查
- 资金建议：申请国家消防救援局科技计划或地方应急信息化专项资金
