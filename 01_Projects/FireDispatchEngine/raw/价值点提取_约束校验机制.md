## 1. 约束校验概述

**定位**：第7步（Step 7）是12步引擎的**核心安全防线**，负责在车辆匹配完成后，验证方案的可行性、安全性、资源可用性。

**目标耗时**：8-18秒

---

## 2. 五维校验体系

| 维度 | 检查内容 | 关键指标 |
|------|----------|----------|
| **资源可用性** | 实时车辆状态、队员状态 | 可用率 > 90% |
| **时间窗约束** | ETA是否满足等级要求 | 响应时间 ≤ 阈值 |
| **特殊环境约束** | 城中村/窄巷/高层/化工 | 环境适配度 |
| **规模合法性** | 最小/最大车辆限制 | 在合理范围内 |
| **政策与预案** | 跨区调派权限、重大活动限制 | 政策合规 |

---

## 3. 校验状态与分流

| 状态 | 说明 | 后续动作 |
|------|------|----------|
| **PASS** | 全部校验通过 | → 触发第8步（反向模拟） |
| **WARNING** | 部分校验警告 | → 带警告继续第8步 |
| **FAIL** | 校验不通过 | → **回退第5步**重新计算规模 |

---

## 4. 校验流程

```
输入聚合（Step6匹配方案 + Step5规模 + Step3画像 + 规则库）
      ↓
多维度检查（5个维度逐项打分）
      ↓
冲突检测与评分（Pass/Warning/Fail + 整体评分0-100）
      ↓
自动修复建议（对Warning/Fail生成调整方案）
      ↓
输出校验报告 → 分流处理
```

---

## 5. 异常处理分支

| 异常场景 | 处理逻辑 | 优先级 |
|----------|----------|--------|
| 实时资源数据同步失败 | 使用缓存数据 + 标记"数据可能滞后" | 高 |
| 特殊环境规则缺失 | 使用默认保守约束 | 高 |
| 校验引擎超时 | 回退到基础规则检查 | 高 |
| 多个严重Fail项 | 直接回退到第5步 | 最高 |
| 跨区联动权限不足 | 自动降级为本地方案 | 高 |
| GIS路况数据不可用 | 使用历史平均ETA | 中 |

## 6. 数据库表结构

### constraint_validation_report
```sql
CREATE TABLE constraint_validation_report (
    incident_id           VARCHAR(32) PRIMARY KEY,
    overall_status        VARCHAR(10),           -- PASS/WARNING/FAIL
    overall_score         NUMERIC(5,2),
    checks_detail         JSONB,                 -- 每条规则检查结果
    suggestions           JSONB[],               -- 修复建议
    repair_applied        BOOLEAN DEFAULT FALSE,
    created_at            TIMESTAMP
);
```

### constraint_check_items
```sql
CREATE TABLE constraint_check_items (
    check_id              BIGSERIAL PRIMARY KEY,
    incident_id           VARCHAR(32),
    rule_type             VARCHAR(50),           -- resource/time/special/scale/policy
    rule_name             VARCHAR(100),
    status                VARCHAR(10),           -- PASS/WARNING/FAIL
    score_factor          NUMERIC(4,3),
    details               JSONB,
    created_at            TIMESTAMP
);
```

  "strict_mode": false,
