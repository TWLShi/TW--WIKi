# 价值点提取——开锁马蜂窝等社会救助场景分析

**来源**：119消防接警中开锁/马蜂窝社会救助场景规范与系统设计
**版本**：V1.0

---

## 核心价值点

### 规范要点
- 119是法定社会救助受理渠道（开锁、马蜂窝、救宠物等均属法定范围）
- 免费服务，消防员出警不收费
- 至少1辆车+5-7名队员出动
- 官方呼吁优先市场化服务（物业/开锁公司/捕蜂队）或12345转接

### 开锁场景特性
| 维度 | 内容 |
|------|------|
| 槽位 | rescueType=DoorLock、urgencyLevel、suggestedAction |
| 决策 | 优先引导联系物业/开锁公司，仅老人/小孩/医疗紧急时派警 |
| 优先级 | 低 |
| 资源 | 社会救助车/开锁工具（≠消防车） |

### 马蜂窝场景特性
| 维度 | 内容 |
|------|------|
| 槽位 | rescueType=WaspNest、riskLevel（高度+数量）、suggestedResource |
| 决策 | 评估危险性，高空/密集区优先出警，夜间作业优先 |
| 优先级 | 中 |
| 资源 | 特勤车+防护人员 |

### 与火灾差异
- 火灾：烟火程度→高优先级→灭火资源
- 开锁/马蜂窝：自助可解决程度+安全风险→中低优先级→工具/人员+分流引导

### 开发优先级
1. 复用火灾核心模块
2. 增量添加开锁&马蜂窝本体+专项数据（2-3周）
3. 更新多Agent+机器人交互模板
4. 灰度上线（先开锁，再马蜂窝）

### 实用性保障
- 优先引导市场化服务（减少警力浪费）
- 记录分流率作为绩效指标
- 测试重点：误派车率
