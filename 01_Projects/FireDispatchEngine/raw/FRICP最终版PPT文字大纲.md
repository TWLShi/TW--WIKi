## 第3页：项目背景与建设必要性

- 原图"八条硬规矩"和"七类突出敏感情况"是信息报告的刚性纪律
- **当前痛点**：
  - 现场报告压力大，伤亡等事实数据难及时获取
  - 后方判断敏感情况依赖人工，容易遗漏或标准不统一
  - 外部数据责任边界不清
- **建设定位**：在已有接处警系统 + 现场指挥系统 + 舆情模块基础上做极简扩展
- **不新增**：对讲机接入、视频分析、物联感知、大模型，聚焦合规报告与职责隔离

---

## 第4页：核心设计原则

| 原则 | 说明 |
|------|------|
| **职责清晰、责任后移** | 后方指挥中心承担全部上报责任，现场零上报责任 |
| **报告合规优先** | 正式报告仅填写已发生事实，辅助参考部分完全隔离 |
| **数据责任明确** | 外部数据必须标注"消防不承担责任" |
| **极简可落地** | 复用现有系统，支持人工录入事实补充伤亡等关键数据 |
| **人工优先** | 取消复杂分析功能，依靠人工核实与录入确保事实准确 |

---

## 第5页：系统总体架构

- **极简扩展架构**：在现有系统基础上增加轻量报告助手层
- 公共信息 + 已有舆情模块 → 数据中台（融合 + 来源标注） → 应用层
- **应用层严格隔离**：
  - 后方指挥子系统（接处警扩展 + 事实核实 + 报告生成）
  - 现场指挥系统（保持现状，仅增加简单事实回传）
- **不包含**：对讲机、视频分析、物联感知、大模型

---

## 第6页：核心功能设计点

| 功能 | 说明 |
|------|------|
| 接处警系统扩展 | 自动提示七类敏感情况和较大/重大事故 |
| 消控室电话拨打 | 重点单位消控室联系电话一键拨打与核实 |
| **人工录入信息功能** | 后方支持手动补充客观事实（伤亡、被困、险情等） |
| 智能报告模板 | 自动填充事实部分 + 指挥员人工审核确认 |
| 50分钟倒计时提醒 | 重大以上事故专用 |
| 一键上报 + 实时共享 | 国家消防救援局 + 同级应急 |
| 数据来源自动标注 | 所有外部数据自动标注责任声明 |

---

## 第7页：现场指挥系统定位

- **保持现状为主**：不做大规模改造
- 仅增加简单客观事实回传功能（文本、数值、位置标记）
- **零上报责任**：现场界面无报告模板、无上报按钮、无分析功能
- 数据单向回传至后方事实核实上下文
- **优势**：现场人员专注救援，无额外负担

## 第8页：成本预算（最终极简版）

| 阶段 | 金额 |
|------|------|
| **MVP阶段** | 约180万元 |
| **完整版** | 约265万元 |
| **3年运维** | 每年约38万元 |
| **总计（建设+3年运维）** | **约387万元** |

**成本优化亮点**：
- 复用现有接处警、现场指挥、舆情模块
- 不接入对讲机、不做视频分析、不做物联感知
- 人工录入功能作为轻量补充，成本可控

## 第9页：迭代规划

| 迭代 | 周期 | 目标 | 主要交付 |
|------|------|------|----------|
| 迭代0 | 2-3周 | 复用权限 + 打好合规地基 | 权限体系扩展、数据标注、存证接口、消控室电话、基础表单 |
| 迭代1 | 4-6周 | MVP核心 — 合规报告闭环 | 接处警扩展、人工录入功能、报告模板、一键上报、50分钟倒计时 |
| 迭代2（可选） | 4-6周 | 优化与完善 | 舆情辅助、公共信息接入、历史查询、审计界面 |

## 第10页：预期价值与效果指标

| 指标 | 目标 |
|------|------|
| **合规性提升** | 报告合规率目标从75%提升至98%以上 |
| **时效性提升** | 敏感情况识别时间缩短至30秒以内 |
| **上报效率提升** | 重大以上事故50分钟内报告达成率100% |
| **指挥效能提升** | 后方决策时间缩短35%以上 |
| **现场减负** | 现场报告相关操作减少95%以上，专注救援 |
| **人工录入价值** | 有效解决伤亡等关键事实补充难题 |

## 第11页：实施建议

| 建议 | 内容 |
|------|------|
| **建设路径** | 以现有接处警系统和现场指挥系统为底座，做最小化扩展 |
| **试点策略** | 先在支队指挥中心试点火灾场景，验证报告合规功能 |
| **保障措施** | 每个迭代结束进行合规审查（重点检查报告是否仅含事实部分） |
| **资金建议** | 申请国家消防救援局科技计划或地方应急信息化专项资金 |

## 第12页：总结

- FRICP是在现有系统基础上**最轻量、最合规**的扩展方案
- 通过人工录入功能有效补充事实数据，实现**职责清晰与规范报告**
- 严格落实**八条硬规矩**，确保信息报告及时、客观、真实
- 以**最低成本**实现**最高实用价值**
