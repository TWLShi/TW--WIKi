## 1. 仪表盘整体布局（4K大屏）

```
┌────────────────────────────────────────────────────────────────────────────┐
│  顶部全局状态栏（每3秒自动刷新）                                          │
├────────────────────┬───────────────────────────────┬──────────────────────┤
│ 左侧：实时流程概览 │  中间：性能核心监控区         │  右侧：告警中心      │

└────────────────────────────────────────────────────────────────────────────┘
```

## 2. 面板设计

### 面板1：全局概览（顶部固定栏）

| 指标 | 显示 |
|------|------|
| 当前活跃警情数 | 12（颜色编码） |
| 平均端到端耗时 | 42秒（P95: 68秒） |
| 系统整体健康度 | 96.8%（环形图） |
| 今日AI置信度均值 | 0.89 |
| 人工介入率 | 4.2%（目标 < 12%） |
| 并发处理能力 | 28 / 50（当前/上限） |

### 面板2：12步实时流程管道

**每个步骤卡片显示**：
- 当前状态 + 耗时（实时）
- 平均耗时 vs 基准（绿色/红色）
- 成功率 / 回退率

**状态颜色定义**：
```
done（完成）= #4ade80（绿色）
running（运行中）= #fbbf24（黄色，加粗）
slow（慢）= #f97316（橙色）
```

### 面板4：实时告警中心

**分级列表**：
| 级别 | 颜色 | 示例 |
|------|------|------|
| Critical | 红色 | Step9多智能体连续失败 |
| Warning | 橙色 | Step5 ML系数异常、缓存命中率低 |
| Info | 蓝色 | Step7校验Warning次数增加 |

**一键操作**：
- 「查看详情」
- 「强制人工接管」
- 「重启当前步骤」

## 3. KPI阈值配置

| 指标类别 | 具体指标 | 正常阈值 | 警告阈值 | 严重阈值 |
|----------|----------|----------|----------|----------|
| 耗时 | 端到端平均耗时 | ≤45s | 45-65s | >65s |
| 耗时 | Step9 AI决策 | ≤25s | 25-35s | >35s |
| 质量 | AI整体置信度 | ≥0.85 | 0.75-0.85 | <0.75 |
| 稳定 | 回退率 | ≤8% | 8-15% | >15% |
| 资源 | 缓存命中率 | ≥85% | 70-85% | <70% |
| 并发 | 同时处理警情 | ≤30 | 30-45 | >45 |

## 4. 技术栈推荐

| 层级 | 技术选型 |
|------|----------|
| 前端 | React + Ant Design Pro + ECharts + Recharts |
| 实时推送 | WebSocket + Server-Sent Events |
| 监控后端 | Prometheus + Grafana |
