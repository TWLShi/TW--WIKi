# 接警场景下（单问题：位置/有人受困/烟火情况）提取规则 + 知识蒸馏成小模型的完整方案

**针对**：中国大陆119/110/120接警，强问题引导 + 槽位填充 + 无关内容过滤，已整合RDF/OWL/SHACL/SWRL本体

---

## 1. 提取规则（Rule Extraction）—— 从本体 + LLM 提炼可蒸馏知识

核心是把**复杂知识（本体约束 + 推理 + LLM判断）**转化为**结构化、可训练的规则/数据**，供小模型学习

### A. 规则层（Rule-based，从本体直接提取）

利用已有RDF/OWL/SHACL/SWRL，自动生成**槽位填充规则**（针对"位置/被困/烟火"三个具体问题）：

**位置槽位规则**（hasLocation + 场所用途 + 区域特征）：
- 必须包含地名/地址 + 区域特征（低洼/高层/地下室）
- SHACL约束：minCount=1，valueType=string + GeoSPARQL模式匹配
- 示例规则："提到'在XX路/小区/村' → 提取为location；若有'低洼/地下' → 添加regional_feature=高风险"

**被困槽位规则**（hasTrappedPeople）：
- 关键词触发："有人/孩子/老人/几个人/被困/无法出去" + 人数（数字/百分比）
- 蕴含/因果：若"烟火大 + 无法出去" → 推断trapped_status=紧急
- SWRL示例：`hasFireSmokeStatus(?e, "严重") ^ mentionsPeople(?e, true) → trappedPriority("high")`

**烟火槽位规则**（hasFireSmokeStatus）：
- 关键词/程度："浓烟/明火/烟很大/火势猛/无明火但有烟味" → 映射为low/medium/high + 类型（烟/火/爆炸风险）
- 无关过滤规则：若句子含"孩子哭了/我害怕"但无烟火关键词 → 标记为irrelevant，丢弃

### B. LLM辅助规则蒸馏（生成高质量训练数据）

- 用大模型（Qwen2或DeepSeek）作为Teacher，基于**问题完整描述**生成**合成/标注数据**
- 输入：问题（如"位置在哪？"） + 呼救者回答片段（含无关内容）
- 输出：结构化JSON/RDF（槽位填充 + 相关性标签 + 置信度） + Chain-of-Thought解释
- 过滤无关：Teacher判断"该片段是否直接回答当前问题"（概率>0.7才保留）
- 规模：从阶段2的500-1000条真实标注 + LLM生成5000-10000条合成对，形成**高质量蒸馏数据集**

**收益**：规则 + LLM数据结合，覆盖显式（关键词）和隐式（因果/蕴含）知识

## 2. 知识蒸馏（Knowledge Distillation）成小模型

采用**Targeted / Adaptive KD**

**Teacher模型**（复杂但一次性）：
- LLM + KG pipeline：FunASR转录 → 问题引导 + SHACL验证 + SWRL推理 → 输出软概率 + 结构化slots

**Student小模型**（目标部署）：
- **文本侧**：TinyBERT / DistilBERT / Qwen2-0.5B~1.5B（或ERNIE-UIE轻量版），专注slot-filling
- **语音侧**：FunASR Paraformer-small / nano变体 + LoRA微调，或直接蒸馏成端到端SpeechLLM-lite
- **混合**：先蒸馏文本SLU（槽位填充），再联合ASR（用Teacher的N-best lattice软标签）

### 蒸馏策略（分阶段，高ROI）

1. **数据蒸馏（最简单有效）**：Teacher生成高质量伪标签（soft probabilities + structured slots），Student在这些数据上做监督微调（SFT）
2. **特征/输出蒸馏**：Student模仿Teacher的中间层embedding（Adaptive Feature Distillation）或softmax输出
3. **规则注入**：在Student训练时加**规则正则化损失**（e.g. 如果输出违反SHACL约束，惩罚大）
4. **端到端蒸馏**（可选进阶）：对FunASR + SLU联合蒸馏

**量化预期**：
- WER：Teacher水平（~8-10%） → Student ~10-13%（相对下降15-25%）
- 槽位F1：>90%（位置/被困/烟火），无关过滤准确率>85%
- 速度：小模型推理延迟<200ms，参数量缩小5-10x，适合边缘部署

---

## 3. 实施步骤（融入之前5阶段）

- **阶段2扩展**（数据准备）：生成Teacher标注数据集 + 规则集（SHACL导出为if-then或正则）
- **阶段3核心**（蒸馏，投入2-4周，云算力~1-2万元）：
  1. 训练Teacher pipeline（用现有本体+LLM）
  2. 生成蒸馏数据集（问题引导 + 软标签）
  3. 训练Student（LoRA + KD损失：CE + distillation_loss + rule_constraint）
- **阶段4验证**：A/B测试（Teacher vs Student在真实接警录音上的槽位准确率 + 延迟）
- **阶段5迭代**：实时日志 → Teacher再标注 → 持续蒸馏

**总投入调整**：蒸馏阶段额外增加~20%算力，但产出是可部署小模型，长期ROI更高

---

## 4. 落地建议与风险

- **工具栈**（大陆友好）：FunASR（Paraformer-small base） + ModelScope/HuggingFace（蒸馏脚本） + GraphDB（规则验证） + DeepSeek/Qwen作为Teacher
- **评估指标**：WER + Slot F1（per问题） + 无关过滤率 + 端到端响应时间 + 资源占用
- **风险控制**：先在合成数据上验证Student，再上真实接警数据；用SHACL确保输出一致性；合规脱敏
