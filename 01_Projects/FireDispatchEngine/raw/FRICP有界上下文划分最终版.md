# FRICP 有界上下文划分（最终版含外部公共信息）

**版本**：V1.0（最终版）
**更新**：已纳入外部公共信息获取

---

## 三有界上下文划分

| 有界上下文 | 核心职责 | 主要事件/命令 | 数据边界与责任 | 与现有系统关系 | 解决的主要痛点 |
|-----------|----------|---------------|----------------|----------------|---------------|
| **事实核实上下文** | 接收并核实所有客观事实数据 + 获取外部公共信息 | 报警要素已提取、消控室电话已核实、现场事实已回传、外部公共信息已获取并标注 | 客观事实 + 外部公共信息（地震、气象、应急预警）；必须标注"外部提供，消防不承担责任" | 复用接处警系统 + 现场指挥系统 + 公共信息接口 | 伤亡数据难获取、外部数据责任不清 |
| **报告生成上下文** | 生成规范报告并执行上报 | 报告模板已生成、事实部分已人工审核、正式报告已上报 | 仅已确认事实；辅助参考部分（舆情、公共信息辅助）完全隔离；上报前必须人工确认 | 复用接处警系统数据 + 事实核实结果 | 报告不完整、预测/舆情混淆 |
| **合规审计上下文** | 全流程存证、责任追溯、合规监控 | 操作已存证、报告已归档、审计日志已生成 | 全局审计日志 + 区块链存证 | 跨所有上下文 | 责任不清、事后追溯困难 |

---

## 外部公共信息获取规则

| 规则 | 说明 |
|------|------|
| **获取范围** | 仅在事实核实上下文中通过标准API接口获取（地震台网、气象局、应急管理部门公共预警） |
| **标注要求** | 获取的数据必须自动标注来源和时间戳，并声明"外部公共信息，仅供参考，消防救援部门不承担准确性责任" |
| **使用限制** | 公共信息仅用于辅助事实核实和内部决策，不得直接写入正式报告的事实部分（需人工转化为已确认事实后使用） |
| **技术封堵** | 系统技术上确保公共信息不会自动进入报告生成上下文的"事实部分" |

---

## 上下文关系映射（最终极简版）

```
现有接处警系统 ──→ 事实核实上下文（报警要素）
现有现场指挥系统 ──→ 事实核实上下文（客观回传数据）
外部公共信息接口 ──→ 事实核实上下文（单向获取）
已有舆情模块 ──────→ 事实核实上下文（辅助参考数据，隔离显示）

事实核实上下文 ──单向推送──→ 报告生成上下文（已核实事实包）
报告生成上下文 ──触发──→ 合规审计上下文（存证）
```

**数据流向**：严格单向，外部公共信息仅流入事实核实上下文，不直接进入报告生成上下文

---

## 边界保护规则（最终版）

### 事实核实上下文

| 规则 | 说明 |
|------|------|
| 职责范围 | 负责客观事实核实 + 外部公共信息获取 |
| 责任标注 | 所有外部数据（公共信息、消控室、微型消防站）必须强制标注责任声明 |
| 内容限制 | 禁止预测分析，仅处理已获取的当前事实 |
| 输出规范 | 输出仅为"已核实事实包"推送给报告生成上下文 |

### 报告生成上下文（最严格）

| 规则 | 说明 |
|------|------|
| 内容限制 | 只允许填写和显示事实部分 |
| 辅助隔离 | 辅助参考部分（舆情、公共信息辅助等）完全隔离，技术封堵复制 |
| 上报要求 | 所有报告必须人工审核确认后才能上报 |
| 内容禁止 | 禁止自动填充任何非事实内容（包括公共信息中的预测性描述） |

### 合规审计上下文

| 规则 | 说明 |
|------|------|
| 监控范围 | 全局监控三个上下文的所有操作 |
| 存证要求 | 每一次数据获取、事实核实、报告生成、上报动作都必须记录不可篡改日志 |
