# 价值点提取——消防119火灾救援与社会救助复用差异分析

**来源**：119接警系统火灾与社会救助对比分析
**版本**：V1.0

---

## 核心价值点

### 复用比例
| 部分 | 复用度 |
|------|--------|
| 接警基础流程+机器人交互框架 | 80%+ |
| 核心本体与知识框架 | 80%+ |
| 模型与蒸馏基础 | 80%+ |
| 通用槽位与实体维度 | 80%+ |

**总体复用收益**：开发工作量减少60-70%

### 差异化重点（30-40%增量）

| 维度 | 差异内容 |
|------|----------|
| 本体扩展 | 新增`SocialRescueEvent`及子类（ElevatorTrapped/GasLeak/ElderFall/AnimalRescue） |
| 属性新增 | 求助类型、紧急程度分级、所需资源类型 |
| SWRL规则 | "电梯困人+高层→高优先级"、"煤气泄漏→causes爆炸风险" |
| 槽位扩展 | 紧急类型、干预需求（破拆/医疗/开门）、细微事实 |
| 多Agent | Help-Agent扩展、Type-Agent强化、Filter新增火灾vs社会救助分类 |
| 资源调派 | 消防车vs特勤车/工具/医疗 |

### 开发工作量分配
- 复用工作：60-70%
- 新增/差异工作：30-40%
  - 本体扩展+SWRL/SHACL：1-2周
  - 社会救助专项标注数据：2-3周
  - 多Agent扩展/蒸馏：2周
  - 机器人交互模板+决策逻辑：1-2周

### 总体建议
先完成火灾核心系统，再**增量迭代**添加社会救助模块
