# 动态热词注入工程实现细节

**来源**：119消防接警机器人（两阶段框架）
**版本**：V1.0

---

## 1. 整体流程（推荐两阶段架构）

```
输入：当前问题 + ASR第一遍CTC假设
  ↓
Pre-Retrieval（预筛选）：从大热词池选出Top-K（15-50条）
  ↓
Contextual Injection（动态注入）：boosting或prompt增强
  ↓
输出：更准确转录 → 更好画像 → 正反馈
```

---

## 2. Pre-Retrieval（预筛选）实现

### 推荐方法（优先级）
1. **FunASR原生RAG-based机制**：音素级编辑距离匹配
2. **Aho-Corasick自动机+Trie树**：极快匹配短语热词
3. **轻量向量检索（Faiss）**：embedding+ANN检索

### 代码框架示例
```python
import editdistance

def pre_retrieve_hotwords(current_question, ctc_hypothesis, hotword_pool, top_k=30):
    """
    current_question: str, e.g. "现场情况怎么样？"
    ctc_hypothesis: str, ASR第一遍输出
    """
    candidates = []
    query = current_question + " " + ctc_hypothesis
    
    for hw in hotword_pool:
        if not is_relevant_to_question(hw["type"], current_question):
            continue
        dist = editdistance.eval(query.lower(), hw["text"].lower())
        score = 1.0 / (dist + 1)
        candidates.append((hw, score))
    
    candidates.sort(key=lambda x: x[1], reverse=True)
    return [item[0] for item in candidates[:top_k]]
```

### 分组加速
把热词按类型分组（位置类、开锁类、马蜂窝类、紧急信号类），根据当前问题只激活相关组

## 3. Contextual Injection（动态注入）实现

### FunASR方式
```python
from funasr import AutoModel

model = AutoModel(model="paraformer-zh", ...)

# 动态生成热词字符串
dynamic_hotwords = "锁在门外 4.0, 进不去家 3.5, 蜂飞得厉害 4.5, 阳台蜂窝 3.0"

res = model.generate(
    input=audio,
    hotword=dynamic_hotwords,
    batch_size_s=300
)
```

### 权重动态调整
- 根据画像置信度或问题类型调整
- 开锁问题时"孩子哭"类热词boost 4.0~5.0
- 普通位置热词boost 2.0~3.0

### Prompt注入方式（LLM-ASR后处理）
```
Context hotwords: 锁在门外, 进不去家, 钥匙忘里面, 蜂飞得厉害, 阳台蜂窝...
请优先识别以上热词，并提取位置、rescueType、紧急程度。
```

## 4. 工程注意事项

| 维度 | 要求 |
|------|------|
| 延迟 | Pre-Retrieval控制在20-50ms内，额外延迟<150ms |
| 热词池大小 | 控制在800-2000条以内 |
| Top-K | 建议15-50条 |
| 权重调整 | 根据问题类型和画像置信度动态调整 |

### 风险控制
- 热词过多时强制Top-K
- 权重过高时加惩罚机制
- 监控关键词召回率和整体WER平衡
