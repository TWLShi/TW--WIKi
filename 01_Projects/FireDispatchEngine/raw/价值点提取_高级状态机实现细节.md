## 2. DispatchContext 核心接口

```typescript
interface DispatchContext {
  incidentId: string;
  currentStep: number;
  alertLevel?: number;
  portrait?: any;
  scaleResult?: any;
  matchingResult?: any;
  validationResult?: any;
  aiPlan?: any;
  finalPlan?: any;
  errorLog: any[];
  humanModifications: any[];
  confidence: number;
}
```

---

## 3. DispatchEvent 类型

```typescript
type DispatchEvent =
  | { type: 'PARSE_COMPLETE' }
  | { type: 'QUERY_COMPLETE', data: any }
  | { type: 'VALIDATION_PASS' }
  | { type: 'VALIDATION_FAIL', reason: string }
  | { type: 'HUMAN_APPROVED', plan: any }
  | { type: 'HUMAN_MODIFIED', changes: any }
  | { type: 'ERROR', error: any };
```

## 4. 关键Guards（守卫）

```typescript
const guards = {
  isHighRisk: (ctx) => ctx.alertLevel >= 3,
  validationPassed: (ctx) => ctx.validationResult?.status === 'PASS',
  humanMajorChange: (ctx, event) =>
    event.changes?.vehiclesChanged && Math.abs(event.changes.diff) >= 3,
  aiConfidenceLow: (ctx) => ctx.confidence < 0.75
};
```

## 5. 关键Actions（动作）

```typescript
const actions = {
  logTransition: (ctx, event) => {
    auditService.record({
      incidentId: ctx.incidentId,
      from: currentState,
      to: nextState,
      event: event.type,
      timestamp: Date.now()
    });
  },

  triggerHumanReview: (ctx) => {
    notificationService.pushCriticalAlert(ctx.incidentId, "需要人工介入");
  },

  saveToTrainingQueue: (ctx) => {
    trainingQueue.push({
      input: ctx.portrait,
      aiOutput: ctx.aiPlan,
      humanOutput: ctx.finalPlan
    });
  }
};
```

## 6. 分层状态结构

```
idle（空闲）
    ↓
active（活跃）← 主状态机
    ├── step1_Receive（接收）
    ├── step2_Query（问询）
    ├── step3_Portrait（画像）
    ├── step4_Level（等级）
    ├── step5_Scale（规模）
    ├── step6_Matching（匹配）
    ├── step7_Validation（校验）
    ├── step8_Simulation（模拟）
    ├── step9_AI（多智能体）
    ├── step10_Human（人工确认）
    └── step11_Dispatch（下达）
    ↓
audit（归档）← 最终状态
```

---

## 7. 回退机制

| 回退路径 | 触发条件 |
|----------|----------|
| Step7 FAIL → Step5 | 校验不通过 |
| Step8 FAIL → Step5 | 模拟冲突 |
| Step10 MODIFIED → Step7 | 重大修改需重新校验 |
| 任意ERROR → Step10 | 故障跳转人工确认 |
