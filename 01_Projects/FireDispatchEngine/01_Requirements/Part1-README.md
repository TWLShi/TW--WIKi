## 1. Part概述

**本Part专注于消防接处警指挥调度领域的核心业务需求**，以**报警人紧急求助场景**为核心驱动，系统化梳理和建模真实消防业务场景，为后续智能引擎设计提供坚实的业务基础和数据支撑。

---

## 2. 业务需求定义（核心说明）

**业务需求**在本Part中特指：

- **报警人（市民）在紧急情况下的求助需求**：包括不同类型火灾、救援场景下报警人可能提供的信息、常见描述模式、关键缺失要素等
- **消防指挥调度业务的实际运营需求**：接警研判、风险评估、资源调配、指挥决策等一线业务流程和规则
- **场景驱动的需求建模**：通过真实、多维的火灾与救援场景画像，提炼出可量化、可复用的业务知识

**本Part不涉及**软件产品的用户界面需求（如调度员界面如何设计），而是聚焦**领域业务本质**，为系统提供"做什么"和"为什么这么做"的业务依据。

## 4. 规模计算模型（核心业务规则）

### 核心公式

```
推荐车辆数 = round(基础车辆数 × 总系数)
总系数 = 建筑系数 × 风险系数 × 时间系数 × 面积系数（上限3.5）
单中队出车率 ≤ 70%
```

### 基础编成表

| 等级 | 基础编成 | 车辆数 |
|------|----------|--------|
| 一级 | 水罐1 + 抢险1 | 2辆 |
| 二级 | 水罐2 + 举高1 + 抢险1 | 4辆 |
| 三级 | 水罐4 + 举高2 + 特勤1 + 抢险1 | 8辆 |
| 四级 | 水罐6 + 泡沫3 + 举高2 + 特勤2 + 洗消1 + 医疗1 | 15辆 |

## 5. 子场景分类体系（12类）

| 场景类型 | 编码 | 特征 | 典型处置方案 |
|----------|------|------|--------------|
| 普通住宅 | RES-01 | 低层、蔓延慢 | 2-4辆水罐+抢险 |
| 高层住宅 | HIGH-01 | 垂直蔓延、救援难度大 | 举高车+抢险+供水 |
| 城中村 | VILL-01 | 巷窄、路窄、水源远 | 小型水罐+手抬泵 |
| 化工园区 | CHEM-01 | 爆炸、毒气、次生灾害 | 泡沫+洗消+防化+医疗 |
| 地下空间 | UGRD-01 | 排烟难、疏散难 | 排烟+照明+抢险 |
| 医院 | HOSP-01 | 人员密集、特殊人群 | 医疗+抢险+举高 |
| 学校 | SCHL-01 | 密集时段的未成年人 | 优先疏散+医疗 |
| 商业综合体 | MALL-01 | 体量大、业态复杂 | 多中队联动+举高 |
| 厂房 | FACT-01 | 可燃物多、结构复杂 | 泡沫+高喷+抵近 |
| 仓库 | STOR-01 | 货架多、蔓延快 | 重型泡沫+无人机 |
| 洪水救援 | FLOOD-01 | 水上救援、急流抢险 | 冲锋舟+救生衣+医疗 |
| 特殊（锂电池等）| SPEC-01 | 新兴风险、无经验 | 专家会诊+特殊装备 |
