# 应急响应与调度指挥手册 — 框架流程图生成 Prompt

## 使用说明

将以下prompt提供给支持流程图生成的大语言模型，配合Mermaid、Graphviz或其他流程图工具使用。建议输出格式为Mermaid flowchart代码，可进一步渲染为SVG/PNG。

---

## Prompt正文

请基于《天津市消防救援总队应急响应与调度指挥手册》的内容，生成一张完整的大体框架流程图。要求如下：

### 一、总体要求

1. **流程图类型**：采用 Mermaid flowchart 格式输出，自上而下（TD）布局。复杂程度为中高粒度，既体现宏观全貌，也包含关键子流程节点。

2. **分层结构**（4个层级）：
   - L1-总览层：7大模块的宏观流转关系（值班值守 → 接警调派 → 应急响应 → 跟踪指挥 → 信息报送，两侧配套应急联动与基础信息支撑）
   - L2-模块流程层：每个模块内部的主要步骤节点（5-8个关键节点）
   - L3-关键分支层：重要的判定分支和并行节点（三色响应判定、火警分级判定等）
   - L4-输出节点层：最终输出物（调派单/信息快报/态势图/作战指令等）

3. **配色方案**：使用 Mermaid classDef 样式定义：
   - 🔵 蓝色系 `#4A90D9`：值班值守、日常状态相关节点
   - 🟢 绿色系 `#27AE60`：警情跟踪、信息报送相关节点
   - 🔴 红色系 `#E74C3C`：遂行出动、重大警情应急响应节点
   - 🟡 黄色系 `#F39C12`：决策判定、分支节点
   - ⚪ 灰色系 `#95A5A6`：基础信息、支撑保障节点
   - 🟣 紫色系 `#8E44AD`：应急联动、对外协调节点

4. **节点数量**：控制在60-80个节点以内，确保A3幅面可读。

### 二、七大模块核心内容

#### 模块1：值班值守（入口模块）

需体现：
- 全勤指挥部架构：总指挥 → 副总指挥 → 指挥长 → 作战指挥班组（作战/信息/通信/政工/宣传/战保/火调7个助理）+ 行政管理班组
- 三种值守模式：日常值守（8:30-20:30）→ 应急值守（5分钟集结）→ 战备值守（重大节日/安保）
- 交接班流程：器材交接(8:10) → 晨会交接(8:30，五项议程) → 签字确认
- 值班督查：视频督导检查 + 电台点名（5种类型：总队全勤组/支队带班主官/支队指挥长/支队指挥中心值守/保卫任务上勤）→ 战备督查通报
- 双班备勤：主班（当日）+ 备班（次日，不得饮酒离驻地）
- 决策分支出动条件：10类警情跟踪情形 + 7类遂行出动情形

#### 模块2：接警调派（核心入口）

需体现：
- 首接用语："您好，天津市119"
- 接警询问六要素：地址 → 灾情 → 人员情况 → 燃烧物质 → 建筑结构 → 危险物品
- 警情六大分类：火灾扑救 / 抢险救援 / 社会救助 / 公务执勤 / 反恐排爆 / 其他出动
- 火警五级判定（绿→蓝→黄→橙→红），基于人员伤亡/被困人数
- 应急救援五级判定，同火警分级标准
- 力量调派12类作战模块：基础灭火模块 / 化工灭火模块 / 专业模块(高层/地下/大跨度/石油化工) / 高喷模块 / 供水模块 / 照明模块 / 排烟模块 / 洗消模块 / 工程机械模块 / 战保模块 / 远程供水模块 / 特种车模块
- "高、低、大、化"调派矩阵：高层/地下/大跨度/石油化工四类火灾按燃烧面积(未见明火→200m²→500m²→1000m²→2000m²+)匹配模块数量
- 接警六大严禁 + 禁忌词语 + 禁忌语气
- 特殊情形话术分支：投诉 / 举报 / 恶意骚扰 / 无险情开门开锁 / 预约救助

#### 模块3：应急响应（核心枢纽）

需体现：
- **三色响应判定核心分支**：
  - 🔵 蓝色（日常值守）：一/二级普通警情 → 主班组15人常态值守 + 备勤组/通讯组热备
  - 🟢 绿色（警情跟踪）：较大敏感警情 → 备勤组就位 + 通讯组准备遂行 + 信息初报
  - 🔴 红色（遂行出动）：三级以上警情 → 全勤指挥部出动 + 10个岗位详细分工
- 7类席位15人排布：值班主任 / 调度助理 / 值班班长 / 信息员 / 接警员(1/2/3号) / 通信值班员 / 绘图员
- 三个运行班组：主班组(接处警) + 备勤组(业务知熟/热备) + 通讯组(遂行出动)
- **火灾扑救21步响应流程**：接警调派(3步) → 信息搜集(3步) → 途中指挥(2步) → 初战指挥(3步) → 应急响应(3步) → 跟踪指挥(3步) → 辅助指挥(4步)
- **"一部六组"现场指挥部**启动与架构：
  - 综合信息组（每30分钟报送，1小时更新人员信息）
  - 指挥协调组（辅助决策/作战命令落实/专家调度）
  - 应急通信组（通信网络搭建/无人机航拍/全景鸟瞰图/三维建模）
  - 战地政工组（临时党组织/心理疏导/典型宣传/表彰慰问）
  - 战勤保障组（装备物资/饮食卫勤/技术支持）
  - 新闻宣传组（舆情管控/媒体对接/战斗画面记录）
- **跨区域增援分级响应**：
  - 地震四级：IV-176人50车 → I-853人188车
  - 洪涝：100人/200人编成
  - 森林火灾四级：IV-123人35车 → I-614人148车
  - 化工火灾三级：III-123人38车 → I-275人73车
- 保障机制：练兵循环 + 模拟演练 + 案例复盘

#### 模块4：跟踪指挥（执行主轴）

需体现：
- **"三段指挥"三大阶段及其任务**：
  - ①调度指挥：接警询问 → 调派力量(就近/多点/等级) → 联动部门
  - ②途中指挥：核实地址任务 → 跟踪车辆动态 → 研判灾情态势 → 推送处置要点 → 预见性增援调派
  - ③现场指挥：到场展开提示 → 获取现场信息 → 视频辅助指挥(单兵图传/4G布控球/无人机) → 安全风险管控 → 作战态势标绘 → 调整参战力量 → 传达作战指令
- 警情分级响应分工：
  - 一级警情：支队负责，总队备询
  - 二级警情：支队遂行出动，总队介入"三段指挥"
  - 三级以上：总队全面协调，支队属地负责，增援支队配合
- **一体化作战指挥三条并行线**：
  - 指挥中心线：受理警情 → 接警询问 → 力量调派 → 跟踪提示
  - 基层队站线：执勤备战 → 响应出动 → 途中行进 → 到场准备 → 安全防护 → 灾情侦察 → 设立警戒 → 疏散救人 → 破拆排烟 → 火场供水 → 灭火战斗 → 现场清理 → 恢复战备
  - 全勤指挥部线：一体化值守 → 信息搜集 → 辅助决策 → 远程指挥 → 灾情研判 → 遂行出动 → 现场指挥 → 安全管控 → 信息报送
- **24类典型警情**概览：
  - 12类火灾扑救：多层/高层/地下/大跨度/城市综合体/危化品仓库/油罐/露天堆垛/电化学储能电站/高速铁路/船舶/涉疫场所
  - 12类抢险救援：建筑坍塌/水域/地下有限空间/交通事故/危化品运输/燃气泄漏/山岳/地震/洪涝/森林火灾/雨雪冰冻/大风天气
  - 每类6维度：接警询问 / 调派力量 / 联动部门 / 应急响应 / 搜集信息 / 安全提示

#### 模块5：信息报送（信息输出轴）

需体现：
- **"1小时"报送时限链**：接警 → 20分钟内电话初报（有什么报什么，容错免责）→ 1小时内《值班信息快报》→ 续报 → 终报
- 报送范围5类触发：人员死伤 / 敏感场所 / 群死群伤风险 / 敏感时段 / 社会关注
- 报送四原则：首报要快、知情必报、信息要准、正规途径
- 信息核报岗位链：支队主官 → 现场最高指挥员 → 信息助理 → 灭火救援文书
- 现场关键时限：到场5分钟初步侦察 → 15分钟电话报告 → 45分钟书面信息
- 审核流程：值班干部编辑 → 值班主任审核 → 值班领导签发 → 报告主官
- 舆情管控三机制：舆情应对(专班运作) → 舆情反馈(30分钟核实) → 新闻发布(四道关口：政治政策/文字图片/编辑剪辑/发送范围)
- 三大模板体系：火灾模板(首报/续报/终报) + 12类场所快报 + 10类救援快报
- 入院跟踪：7日内跟踪医治情况
- 人员搜救核实：问/看/听/查 + 疏散清点区 + 在外人员通话确认

#### 模块6：应急联动（横向协同）

需体现：
- **突发事件应急联动三条线**：
  - 分灾种通报：7类自然灾害 + 10类事故灾难
  - 一体化处置：消防先期 → 应急局联动电力/燃气/水务/气象 → 军队武警
  - 常态化联络：月度沟通 + 半年联席会议 + 应急通讯录 + 微信工作群
- **119-120联动六项机制**：警情互通 / 联勤联动 / 联合培训演练 / 应急资源共享 / 常驻联合值守 / 联勤作战（医护入驻消防站，同吃同住同出警）
- **119-110三级联动**：
  - 一级(重大)：总队指挥中心 ↔ 市公安局合成作战中心
  - 二级(关注)：支队指挥中心 ↔ 属地公安合成作战中心
  - 三级(一般)：消防站 ↔ 派出所综合指挥室
  - 系统支撑：双向开端口互派值守 + 应急专线（27330119 / 27204944）
- 天津港联动：海事 ↔ 消防信息互通 + 海上陆上联动
- 工程机械力量：22个支队签约（挖掘机/装载机/推土机/吊车等）
- 30+社会联动单位：国家局/市政府/应急局/公安/交管/地震/气象/环保/海事/燃气/供水/排水/武警/地铁等

#### 模块7：基础信息（底层支撑）

需体现：
- 辖区概况：16区 / 1364万人口 / 11966km²
- 重点监管对象 9633家（学校医院养老院/自建房/新业态/宾馆/娱乐/商场/仓储/文博）
- 城市基础设施：30393栋高层(114栋百米以上，7栋250米+) / 73家综合体 / 9条地铁(398km/193站) / 64649个消火栓 / 1224天然水源+663人造水源
- 消防实力：22支队+10大队 → 137个执勤站 → 692部执勤车（107压缩空气泡沫/170水罐/163泡沫/109抢险/54登高/37高喷/15排烟照明/13干粉/14机器人等）
- 总载液量：水3043吨 + 泡沫563吨 + 干粉68吨 + 远程供水11套 + 工程机械30台
- **六大作战协作区**：中心(市内六区) / 东部(滨海宁河) / 西部(西青静海高新) / 北部(武清北辰蓟州宝坻) / 南部(东丽津南) / 战勤保障
- 特种消防车按协作区分布明细（超高层供水/大吨位水罐/大功率泡沫/云梯/登高平台/举高喷射/化工抢险/机器人/排烟/照明/远程供水等）
- **9类作战信息备询表**：高层建筑 / 地下空间 / 大型综合体 / 商市场 / 厂房库房 / 船舶火灾 / 化工储罐 / 化工装置 / 危化品运输车

### 三、关键路径与分支逻辑

1. **主路径（纵向贯穿）**：
   ```
   值班值守(常态) → 接警调派(警情受理) → 应急响应(三色判定) → 跟踪指挥(三段指挥) → 信息报送(初报→续报→终报)
   ```

2. **三色响应核心分叉**：
   ```
   警情受理 → [警情定级判定]
               ├── 一/二级 → 🔵蓝色 → 日常值守 → 主班组常规处置
               ├── 较大敏感 → 🟢绿色 → 警情跟踪 → 备勤组就位+信息初报
               └── 三级以上 → 🔴红色 → 遂行出动 → 一部六组+10岗位分工
   ```

3. **横向协同轴**（贯穿所有阶段）：
   应急联动（左侧⇄全流程⇄右侧）基础信息支撑

4. **信息报送时间轴**（附属每条警情线）：
   T0接警 → T+20min电话报告 → T+60min信息快报 → T+续报 → T+终报

5. **力量调派分支**：
   警情类型 → 火警/应急救援 → 等级判定(1-5级) → 匹配调派模块矩阵 → 首发力量 + 增援力量 + 社会联动

### 四、样式规范

1. 使用 Mermaid `subgraph` 将7个模块分组，模块名称用中文
2. 判定节点用菱形 `{}`，并行节点用 `||` 标记
3. 边标签（箭头上文字）标注关键条件或时限
4. 节点ID用英文缩写，节点文本用中文全称
5. 必须包含 `classDef` 样式定义并正确应用6种配色
6. 每个subgraph内节点数量控制在 5-12 个

### 五、输出格式

最终输出为一段完整的 Mermaid flowchart 代码，包裹在 ` ```mermaid ` 代码块中。流程图代码之前用1-2句话说明阅读方式，之后附一段"图例说明"。
