# FRICP 标准字典（扩展版）

**版本**：扩展版
**用途**：产品设计、开发、测试和汇报的统一语言标准
**依据**：原图"八条硬规矩"、七类敏感情况、极简设计原则

---

## 1. 核心概念与定位

| 术语 | 定义 |
|------|------|
| **FRICP** | 智慧消防救援一体化指挥平台。在现有接处警系统和现场指挥系统基础上轻量化扩展的合规报告助手。 |
| **极简设计** | 本项目采用的整体设计策略：最大化复用现有系统，不接入对讲机、不做视频分析、不做物联感知、无大模型，聚焦合规报告与职责隔离。 |
| **后方指挥中心** | 运行119接处警系统的指挥机构，承担**全部信息报告主体责任**（包括首报、续报、终报、实时共享、如实报告）。 |
| **现场救援处置单元** | 前线消防救援人员和指挥员，仅负责救援执行和客观数据回传，**零上报责任**。 |
| **事实部分** | 报告中只能填写的内容，必须是**已发生、可验证的客观事实**（伤亡人数、被困人数、已观测险情、处置进展等）。系统强制只允许此部分可编辑。 |
| **辅助参考部分** | 舆情数据、公共信息辅助分析等，仅供后方指挥中心内部决策参考，**完全隔离**，系统技术封堵复制到事实部分。 |

---

## 2. 报告与数据相关术语

| 术语 | 定义 |
|------|------|
| **报告模板** | 系统自动生成的结构化模板，分为"事实部分"（可编辑）和"辅助参考部分"（只读隔离）。 |
| **首报** | 首次正式上报，适用于达到或可能达到较大/重大事故或七类敏感情况时。 |
| **续报** | 救援过程中伤亡、被困或处置进展发生变化时生成的后续报告。 |
| **终报** | 事件处置结束后的最终报告。 |
| **已核实事实包** | 事实核实上下文输出的干净数据包，仅包含已人工审核确认的客观事实，单向推送给报告生成上下文。 |
| **人工录入信息功能** | 后方指挥中心人员手动补充或修正客观事实（伤亡、被困、险情描述、处置进展等），用于解决初期数据难获取的问题。录入内容必须经过人工审核确认。 |
| **逐条人工审核** | 对关键数据（伤亡、被困等）必须一条一条人工审核确认，不能批量跳过。系统强制要求每条关键FactItem都有独立的审核记录。 |
| **事实纯净规则** | 报告的事实部分只能包含已确认的客观事实，系统实时校验并阻止预测性、主观或辅助内容进入。 |

## 3. 敏感情况与事故等级相关术语

| 术语 | 定义 |
|------|------|
| **七类突出敏感情况** | 原图右侧列出的七类高风险、高关注事件，必须即知即报、先报后核。 |
| **较大事故** | 死亡3人以上10人以下，或重伤10人以上50人以下，或直接经济损失1000万元以上5000万元以下。 |
| **重大以上事故** | 重大事故（死亡10人以上30人以下等）或特别重大事故，必须事发50分钟内报告。 |
| **三断** | 村镇出现断水、断电、断路的情况，属于第5类敏感情况的重要触发条件。 |

---

## 4. 上下文与模型相关术语

| 术语 | 定义 |
|------|------|
| **事实核实上下文** | 负责接收、核实客观事实（包括人工录入）和外部公共信息，仅输出已核实事实包。 |
| **报告生成上下文** | 最严格的上下文，负责生成报告、强制人工审核确认、一键上报。 |
| **合规审计上下文** | 全局监控，记录所有操作存证，确保可追溯。 |
| **辅助决策区** | 后方指挥中心界面中的隔离区域，仅显示舆情、公共信息等辅助参考内容，不得进入报告。 |
| **责任标注** | 系统强制为外部数据添加的责任声明文本，防止责任混淆。 |
| **区块链存证** | 对关键操作（事实录入、人工审核、报告生成、上报等）进行的不可篡改记录，用于事后追溯和尽职免责。 |

---

## 5. 角色与操作相关术语

| 术语 | 定义 |
|------|------|
| **值班主任 / 值班指挥员** | 后方指挥中心日常值班负责人，主要操作接处警、事实核实和报告生成。 |
| **指挥长** | 支队级及以上领导，最终责任人，可覆盖审核和上报决策。 |
| **人工录入** | 后方指挥员手动补充客观事实的操作，必须经过后续审核确认。 |
| **一键上报** | 审核确认后，一键完成正式上报并实现实时共享的操作。 |
