# FRICP 后方指挥中心页面结构设计（完整版）

**版本**：V1.0（完整版）
**设计原则**：极简、复用、职责隔离、事实优先

---

## 1. 后方指挥中心页面结构总览

| 页面 | 定位 | 优先级 |
|------|------|--------|
| 首页仪表盘 | 登录默认页，快速行动入口 | Must |
| 事实核实工作台 | 核心操作页，人工录入+核实 | Must |
| 报告生成工作台 | 最严格合规页面 | Must |
| 辅助决策区 | 侧边栏/独立标签页，舆情+公共信息 | Should |
| 审计日志查询页 | 合规审计上下文支持 | Could |

---

## 2. 页面1：首页仪表盘

**定位**：登录后默认首页，快速行动

### 布局结构

```
┌─────────────────────────────────────────────────────────────────┐
│  顶部全局栏                                                      │
│  [待处理警情：3] [今日敏感警情：1] [用户：值班主任]              │
├─────────────────────────────────────────────────────────────────┤
│  中间主要区域                                                    │
│  ┌───────────────────────────┬─────────────────────────────┐   │
│  │  左侧：待处理警情列表      │  右侧：快捷操作区             │   │
│  │  （复用接处警系统列表）    │  ┌─────────────────────────┐ │   │
│  │                           │  │  [人工录入事实] 大按钮   │ │   │
│  │  高亮敏感警情             │  │     （橙色醒目）         │ │   │
│  │                           │  └─────────────────────────┘ │   │
│  │                           │  [开始事实核实]              │   │
│  │                           │  [一键拨打消控室电话]         │   │
│  │                           │  ┌─────────────────────────┐ │   │
│  │                           │  │  50分钟倒计时卡片        │ │   │
│  │                           │  │  （仅重大以上警情显示）  │ │   │
│  └───────────────────────────┴─────────────────────────────┘   │
├─────────────────────────────────────────────────────────────────┤
│  底部：最近上报记录（最近5条）                                   │
└─────────────────────────────────────────────────────────────────┘
```

### 设计要点

- "人工录入事实"按钮放在首页最显眼位置，解决伤亡数据难获取痛点
- 50分钟倒计时卡片仅在有重大以上警情时显示
- 所有数据来自现有接处警系统，FRICP仅做展示和跳转

```

│  敏感情况标记：[已触发：第3类人员被困、第5类三断]              │
├────────────┬────────────────────────────────┬──────────────────┤
│            │                                │                  │
│  左侧      │  中间主操作区                  │  右侧            │
│  已回传事实 │  【人工录入事实表单】（最突出）│  辅助决策区      │
│  列表      │  ┌──────────────────────────┐  │  （隔离显示）    │
│            │  │ 伤亡情况：               │  │                  │
│  来自现场  │  │  死亡 [  ] 人 重伤 [  ]  │  │  舆情摘要        │
│  指挥系统  │  │  轻伤 [  ] 人            │  │  热度/关键词     │
│            │  │ 被困 [  ] 人            │  │                  │
│  时间排序  │  │                          │  │  外部公共信息    │
│            │  │ 险情描述：[        ]    │  │  地震/气象       │
│            │  │ 处置进展：[        ]    │  │  + 责任标注      │
│            │  │                          │  │                  │
│            │  │ [快速标记按钮组]         │  │  "辅助参考       │
│            │  │ 发现被困 火势已控制      │  │   仅供决策"      │
│            │  │ 已破拆进入 已转移群众    │  │                  │
│            │  └──────────────────────────┘  │                  │
│            │                                │                  │
│            │  【消控室核实区】              │                  │
│            │  [一键拨打] [已核实记录列表]   │                  │
├────────────┴────────────────────────────────┴──────────────────┤
│  底部操作栏                                                      │
│  [保存并推送事实包]  [标记为待核实]  [完成核实]                 │
└─────────────────────────────────────────────────────────────────┘
```

### 人工录入表单字段（结构化）

| 字段 | 类型 | 说明 |
|------|------|------|
| 死亡人数 | 数值 |  |
| 重伤人数 | 数值 |  |
| 轻伤人数 | 数值 |  |
| 被困人数 | 数值 |  |
| 险情描述 | 多行文本 | 系统实时提示"请仅描述已发生事实" |
| 处置进展 | 多行文本 |  |

### 快速标记按钮组

- 发现被困X人
- 火势已控制
- 已破拆进入
- 已转移群众X人
- 水位已上升X米（水灾场景）
- 结构已稳定（地震场景）

### 设计要点

- 人工录入表单占据中间主要空间，操作路径最短
- 系统实时校验：如果检测到预测性词汇，弹出警告并阻止保存
- 所有操作自动记录日志并存证

## 4. 页面3：报告生成工作台（最严格合规页面）

**定位**：确保只上报事实部分，人工审核后才能上报

### 布局结构

```
┌─────────────────────────────────────────────────────────────────┐
│  顶部栏：当前警情概要  │  敏感情况标记  │  50分钟倒计时          │
├─────────────────────────────────────────────────────────────────┤
│  中间主区域：报告模板（清晰分区）                                │
│  ┌───────────────────────────────┬────────────────────────────┐ │
│  │  【事实部分】（可编辑）       │  【辅助参考部分】（隔离只读）│ │
│  │                               │                            │ │
│  │  自动填充来自事实核实上下文   │  舆情数据摘要（热度/关键词） │ │
│  │  支持从人工录入记录选择插入    │                            │ │
│  │                               │  公共信息辅助（地震/气象）  │ │
│  │  基本信息：                   │                            │ │
│  │  已确认伤亡/被困：            │  ┌──────────────────────┐  │ │
│  │  已观测险情：                 │  │ "辅助参考，仅供决策  │  │ │
│  │  处置进展：                   │  │  不得复制到事实部分" │  │ │
│  │                               │  └──────────────────────┘  │ │
│  └───────────────────────────────┴────────────────────────────┘ │
├─────────────────────────────────────────────────────────────────┤
│  底部操作栏                                                      │
│  [人工审核确认] → [生成正式报告] → [一键上报]  │  [保存为续报]   │
└─────────────────────────────────────────────────────────────────┘
```

### 操作流程

1. 人工审核确认（必须点击每条关键数据，逐条确认）
2. 生成正式报告（审核后可用）
3. 一键上报（生成正式报告后可用）

### 设计要点

- 辅助参考部分技术上不可复制到事实部分
- 上报前必须完成人工审核确认，否则按钮置灰
- "还有X条关键数据未审核"提示
- 所有上报操作自动触发合规审计上下文存证

## 5. 页面4：辅助决策区（侧边栏或独立标签页）

### 内容

| 内容 | 说明 |
|------|------|
| 舆情模块数据 | 热度、关键词、趋势 |
| 外部公共信息摘要 | 地震、气象、应急预警 |
| 明确标注 | "辅助决策区，仅供参考，不得用于正式报告" |

### 限制

- 无任何上报或报告生成按钮
- 完全隔离，不得与报告生成工作台混用

## 6. 页面5：审计日志查询页

### 功能

| 功能 | 说明 |
|------|------|
| 过滤条件 | 按警情ID、时间、操作人过滤 |
| 操作记录 | 事实核实、报告生成、人工审核、上报等 |
| 存证标记 | 所有记录不可删除，带区块链存证标记 |

## 7. 现场指挥系统页面结构（极简，只扩展回传）

**页面名称**：任务执行与回传页（复用现有 + 轻量扩展）

### 布局结构

```
┌─────────────────────────────────────────────────────────────────┐
│  顶部导航栏（复用现有）                                         │
│  警情编号 + 类型 + 任务状态 + 后方指令摘要（只读）               │
├─────────────────────────────────────────────────────────────────┤
│  中间主区域：简化态势视图（复用现有 + 轻量优化）                 │
│  - GIS地图简化版                                                 │
│  - 基本信息卡片：警情地点、已知事实摘要                          │
│  - 严格限制：不显示分析结果/预测/舆情/报告信息                   │
├─────────────────────────────────────────────────────────────────┤
│  底部：客观事实回传区（轻量扩展核心）                           │
│  ┌───────────────────────────────────────────────────────────┐   │
│  │  [快速标记按钮组]                                        │   │
│  │  发现被困X人 │ 火势已控制 │ 已破拆进入 │ 已转移群众X人    │   │
│  │  水位上升X米（汛）│ 结构稳定（地震）                    │   │
│  ├───────────────────────────────────────────────────────────┤   │
│  │  文本/数值录入：[仅允许事实描述]                         │   │
│  │  伤亡人数：[  ] 被困人数：[  ]                           │   │
│  └───────────────────────────────────────────────────────────┘   │
│                      [提交回传]（醒目橙色按钮）                  │
└─────────────────────────────────────────────────────────────────┘
```

### 严格限制

| 限制 | 说明 |
|------|------|
| 无报告模板 | 页面完全不显示任何报告相关内容或按钮 |
| 无上报按钮 | 现场人员无法看到或触发上报功能 |
| 无分析功能 | 不显示态势分析、事件链、预测结果 |
| 无舆情显示 | 完全隐藏舆情模块数据 |
| 要素交互 | 所有警情要素通过接处警系统统一推送，本页面仅负责接收和回传 |

## 8. 信息流总览

```
现场指挥系统
    ↓（单向回传）
事实核实上下文
    ↓（已核实事实包）
报告生成上下文
    ↓（人工审核确认）
一键上报 → 国家消防救援局 + 同级应急
    ↓
合规审计上下文（存证）
```
