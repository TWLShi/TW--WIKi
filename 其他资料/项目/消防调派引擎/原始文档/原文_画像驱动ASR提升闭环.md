# 画像驱动ASR提升闭环

**来源**：119消防接警机器人（火灾+开锁/马蜂窝等社会救助场景）
**版本**：V1.0

---

## 1. 为什么"画像正确"能直接提升ASR？

### 核心痛点
ASR的困难不是声学模型，而是**领域特定实体、关系和上下文**的识别困难（"锁在门外""孩子在里面哭""阳台马蜂窝"等口语变体）

### 画像作为语义先验的作用
1. **N-best Hypotheses Rescoring**：对候选进行语义一致性打分
   - 符合画像的候选分数提升，不符合的降低
   - 可额外降低WER 10-25%

2. **上下文引导与热词注入**：实时生成领域热词词典
   - 同义词altLabel：锁在门外、进不去家、蜂飞得厉害等
   - 显著提升专业/口语术语识别率

3. **错误检测与后处理**：画像作为"语义过滤器"
   - 检测转录错误（"农民滚滚"→"浓烟滚滚"）
   - 利用反义词规则和因果规则进行一致性校验

4. **闭环迭代效果**
   画像越准确 → ASR重打分越有效 → 转录质量越高 → 后续画像更准确

## 3. 如何具体实现"画像正确→ASR提升"闭环

### 画像构建（核心前提）
- 大模型Teacher先行生成高质量软标签 + CoT + RDF triples
- 6人团队轻量审核规则（同义词、反义词、紧急程度）
- 持续迭代：新日志 → Teacher提炼新规则 → 更新本体

### ASR提升具体技术
1. **N-best/Lattice Rescoring**：用画像对ASR候选重新打分
2. **热词/上下文注入**：实时从画像生成领域词典
3. **后处理纠错**：画像作为语义校验器修正明显错误
4. **可选**：轻量Graph RAG检索历史相似画像

### 开锁示例
- 候选1："我被锁在家门外，孩子在里面哭" → 画像匹配高（OpenLock+儿童风险）→ 分数+0.8
- 候选2："我被锁在农民门外" → 匹配低 → 分数降低

### 马蜂窝示例
- 候选含"蜂飞得厉害"+"孩子不敢出去" → riskLevel=high+suggestedTime=夜间优先 → 高分

## 4. 动态热词注入（基于问题的热词）

### 为什么基于问题的热词应该作为重点
- 问题意图唯一且强先验
- 对关键台阶提升最明显（KER/槽位F1相对20-50%+提升）
- 工程落地优势显著

### 两阶段架构
1. **Pre-Retrieval（预筛选）**：从大热词池快速选出Top-K（15-50条）
2. **Contextual Injection（动态注入）**：只把筛选后的小列表注入ASR

### 问题与热词映射
| 问题 | 激活热词类型 |
|------|-------------|
| 位置在哪？ | 位置类：门外、车内、电梯内、高层阳台 |
| 现场情况怎么样？ | rescueType类：锁在门外、蜂飞得厉害、孩子哭 |

### 热词池结构
```json

  "boost_weight": 3.5,
  "priority": "high",
  "context": ["现场情况怎么样？"],
  "source": "teacher_iteration_202603"

```

### 维护策略
1. **自动化发现**：Teacher大模型每周分析新日志，提炼新同义词
2. **定期清理**：低频删除、冲突合并、权重动态调整
3. **人工审核**：6人团队轻量审核（每周<4小时）
4. **闭环反馈**：高召回热词自动提升权重

### 预期效果
- 3个月后热词池覆盖率从60%提升到95%+
- 关键词召回率显著提高
- 维护人力每周<4小时
