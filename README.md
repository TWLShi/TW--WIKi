# TW--WIKi 消防调派智能系统知识库

*用 LLM 构建持续积累的个人知识库*

---

## 项目概述

TW--WIKi 是**消防调派智能系统**的个人知识库，采用 LLM Wiki 方法论构建。知识随每次导入和查询不断积累，形成复利效应。

> 类比：Obsidian 是 IDE，LLM 是程序员，Wiki 是代码库。

---

## 目录结构

```
TW--WIKi/
├── wiki/                    # ⭐ LLM Wiki 主目录（Obsidian Vault）
│   ├── 01-Part（部分）/     # Part 1-4 系统架构
│   ├── 02-Step（流程）/     # 12步核心流程
│   ├── 03-Scene（场景）/    # MOCs 场景枢纽
│   ├── 其他类/              # PARA 结构内容
│   └── 原始材料/            # 原始资料
│
├── llm-wiki-base/           # LLM 知识库结构（预留）
│   ├── concepts/           # 概念页面
│   ├── entities/           # 实体页面
│   ├── skills/            # 技能页面
│   ├── references/        # 参考资料
│   ├── synthesis/         # 综合知识
│   ├── journal/           # 日记
│   ├── projects/          # 项目知识
│   ├── _raw/              # 原始草稿（待导入）
│   └── _archives/         # 归档存储
│
├── wiki-version/           # 历史版本快照
└── wiki模板/               # 模板备份
```

## 核心内容

### Part 系统架构

| Part | 内容 | 说明 |
|------|------|------|
| **Part 1** | 需求与场景层 | 子场景画像、N_total公式、12类警情场景 |
| **Part 2** | 系统设计与数据模型 | DIKW框架、数据结构设计 |

### 12步核心流程

1. 接警接收与初步研判
2. 智能问询与信息补全
3. 多维画像构建
4. 警情等级智能判定
5. 调派规模智能计算
6. 资源匹配与任务分配
7. 约束校验与风险评估
8. 反向模拟与方案预演
9. AI多智能体协同指挥
10. 人工指挥确认与授权
11. 指挥指令下达与协同
12. 全程审计与优化反馈

|------|------|
| 普通住宅火灾 | 低层建筑、蔓延慢 |

| 地下车库火灾 | 排烟难、疏散难 |
| 商业综合体火灾 | 体量大、业态复杂 |
| 工业厂房火灾 | 可燃物多、结构复杂 |

## LLM Wiki 使用方法

### 导入资料
将文章或文档放入 `llm-wiki-base/_raw/` 目录，告诉 Claude：
```
请导入 _raw/文件名.md 到知识库
```

### 查询知识
直接向 Claude 提问：
```
根据知识库，对比 X 和 Y 的区别
知识库里关于 XX 有什么要点？
```

### 维护健康
```
请检查知识库的健康状态
```
