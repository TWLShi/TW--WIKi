# 报警求助模式与场景路由规划设计方案

*摘要：本方案提出"极简入门 + 动态场景路由"的智能对话策略，实现报警引导流程的高效率与高弹性。*

## 设计背景与目标

在真实报警接处警场景中，报警人往往无法在第一时间完整、准确地描述警情类型^[inferred]。传统固定问答流程存在以下问题：

- **效率低下**：无论何种警情都走同一套长流程，简单警情耗时过长
- **场景固化**：一旦初始判断错误，后续难以修正关键信息
- **知识缺失**：特定场景需要即时提供专业指导，却未能及时触发

本方案实现：
1. **快速起步**：基于高概率场景加载极简模板，秒级获取核心要素
2. **动态修正**：实时识别报警人语音中的热词，自动跳转到更匹配的场景集
3. **知识增强**：跳转后立即关联对应场景的风险提示、处置规范、设备清单等知识库

## 整体架构

```
报警语音/文本输入
    ↓
意图识别：匹配6大场景集（火灾/救援/危化/交通/水域/其他）
    ↓
┌─────────────┐     ┌─────────────┐
│ 匹配成功     │     │ 匹配失败     │
│ 加载极简模板 │     │ 进入缺省模式 │
└──────┬──────┘     └──────┬──────┘
       └───────────┬───────────┘
                   ↓
        并行热词监听引擎
                   ↓ 命中热词
        场景路由决策（跳转）
                   ↓
        加载新场景的问答模板 + 关联知识库
```

## 6大基础场景集及极简模板

| 场景集 ID | 名称 | 典型关键词 | 极简模板（必问项） |
|----------|------|------------|-------------------|
| S01 | 普通火灾 | 着火、冒烟、火光 | 起火物？有无被困？有无爆炸危险？ |
| S02 | 危化品事故 | 泄漏、化学、燃气、汽油 | 什么物质？有无毒害/爆炸？是否需要防化？ |
| S03 | 交通事故救援 | 车祸、追尾、侧翻 | 车辆类型？有无人员被困？有无起火？ |
| S04 | 水域救援 | 溺水、落水、洪水 | 落水人数？水域类型？ |
| S05 | 坍塌/建筑物事故 | 倒塌、塌方、埋压 | 被困人数？结构类型？ |
| S06 | 人员被困/特殊救助 | 电梯困人、反锁、跳楼 | 被困位置？有无生命危险？ |

## 热词识别与动态路由规则

系统在对话全程运行**热词扫描器**，一旦命中预设的热词，立即触发路由跳转^[ambiguous]：

| 热词 | 命中后目标 | 跳转类型 |
|------|------------|----------|
| 医院、学校、商场、客运站、地铁 | 人员密集场所场景集 | 场景集跳转 |
| 汽油、柴油、液化气、天然气、化学品 | 危化品场景集 | 场景集跳转 |
| 隧道、高架、高速、立交 | 交通事故特殊场景 | 直接进入子场景 |
| 地下室、地库、地下车库 | 地下空间火灾 | 场景内细化 |
| 老人、小孩、残疾人 | 弱势群体被困 | 子场景细化 |
| 爆炸、轰然、巨响 | 爆炸风险 | 预警升级 |

## 典型路由示例

### 示例1：普通火灾 → 危化
- **初始**：报警人说"厨房着火了"，系统匹配普通火灾（S01）
- **报警人回答**："我家里存了几桶汽油……"
- **热词触发**：汽油 → 映射到危化场景集（S02）
- **系统动作**：立即跳转，加载危化问答模板，并主动提示"请所有人撤离到上风向"

### 示例2：交通事故 → 危化
- **初始**："高速上两车追尾"，进入交通事故救援（S03）
- **进一步描述**："有一辆是油罐车，在漏油……"
- **热词触发**：油罐车、漏油 → 跳转至危化场景集

## 实施建议

1. **热词库建设**：从历史警情数据中提取高频词，人工标注目标场景
2. **性能要求**：热词识别延迟 < 500ms
3. **人工兜底**：当热词触发错误时，提供"强制重置"按钮
