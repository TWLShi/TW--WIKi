# Query_Routing 智能问询路由

*概念页 — 描述系统如何通过动态路由机制实现智能问询*

## 概述

Query_Routing（智能问询路由）是消防调派智能系统核心引擎（Part 3）在 **Step 3（智能问询）** 阶段的关键领域服务^[extracted]。

它的核心职责是：在报警人描述的碎片化信息与12类子场景画像所需的完整决策参数之间，动态计算并推送最优的问询路径。

## 核心输入

| 输入 | 说明 |
|------|------|
| **常见描述模式** | 报警人主动提供的信息碎片 |
| **子场景画像库** | 12类场景的"标准全要素模板" |
| **实时标签（Tags）** | DIKW 转化后的结构化标签 |

## 核心公式

```
关键缺失要素 = 该场景的标准全要素模板 - 报警人的常见/实际描述模式
```

## 运作流程

### Step 1: 实时标签化（DIKW 转化）
- 输入：报警人的非结构化语音或文字
- 处理：NLP 引擎将原始描述即时转化为标准化的领域标签

### Step 2: 场景画像对齐与差值计算
- 检索：自动关联12类子场景画像库中的对应模板
- 对齐：将实时标签与"标准全要素模板"进行比对
- 识别：锁定"关键缺失要素"

### Step 3: 动态问题集路由
- 优先级评估：AI 多智能体根据灾情严重程度对缺失要素排序
- 路由决策：从"问询问题集"中动态检索最优路径
- 状态机驱动：管理问询事件流，确保不重、不漏

### Step 4: 毫秒级输出
- 推送：精准追问问题被推送至调度员界面
- 反馈：调度员的追问结果再次作为输入，形成闭环

## 与其他模块的关联

- **Part 1（业务需求层）**：提供"常见描述模式"和"标准全要素模板"
- **Step 1（规模计算）**：输出完整参数供调派规模计算使用
- **Part 4（审计）**：记录问询过程用于责任追溯
