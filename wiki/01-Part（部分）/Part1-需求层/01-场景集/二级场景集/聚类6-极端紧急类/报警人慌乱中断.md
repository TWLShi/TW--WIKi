# 报警人慌乱通话中断

## 场景定义

| 属性 | 值 |
|------|------|
| 场景编号 | S6-1 |
| 一级场景 | 极端紧急类（S6） |
| 二级场景 | 报警人慌乱/通话极短 |
| 风险等级 | 极高 |
| 调派类型 | 立即调派+多手段联动 |

---

## 场景特征

- **通话时间极短**：通常 <30 秒
- **报警人情绪失控**：哭泣、尖叫、语无伦次
- **信息严重缺失**：位置、警情类型都不明确
- **高生命风险**：可能涉及人员重伤或即时危险
- **需要多手段补位**：手机定位、联动110/120、快速检索位置

---

## 关键缺失要素

| 缺失要素 | 优先级 | 说明 |
|----------|--------|------|
| location | P0 | 位置信息，严重缺失 |
| incidentType | P1 | 警情类型未知 |
| trappedCount | P1 | 是否有人员被困 |

---

## 问题集（L0 最小必要信息集）

### 核心原则

> 通话极短时，**立即调派优先**，不在通话中追问过多信息。挂断后立即通过技术手段补位。

### L0层（快速提取）

| 节点 | 问题 | 提取字段 | 时限 |
|------|------|----------|------|
| l0.root | "您好，119！请说地址！" | district, community, street | 5秒 |
| l0.pos | (挂断后通过手机定位补位) | GPS, cellTower | - |
| l0.risk | (根据现场声音判断或联动反馈) | trappedCount, severity | - |

### 关键判定

| 情况 | 处理 |
|------|------|
| 能获取位置 | 立即调派，边行动边通过短信补充信息 |
| 无法获取位置 | 立即联动110定位，同步调派 |
| 通话中断 | 回拨 + 短信链接定位 |

---

## 典型对话

**场景1：通话极短直接中断**

**报警人**（哭喊）："快来快来！救命啊！"
（通话中断，时长：4秒）

**系统**：
→ 立即调派最近消防站
→ 联动110获取报警人手机定位
→ 发送短信："请点击链接定位您的事故位置"
→ 回拨电话，持续监听现场声音

---

**场景2：位置模糊+情绪失控**

**报警人**（哭泣）："救命...有人在流血...我不知道在哪..."
（断断续续，时长：12秒）

**系统**："请不要挂断，告诉我您在哪里？"
→ 报警人："我不知道...这是地下室...好冷..."

**系统**（判断）：位置严重缺失，立即联动
→ 调派 + 110手机定位 + 120联动
→ 通过楼号/地下室特征快速检索
