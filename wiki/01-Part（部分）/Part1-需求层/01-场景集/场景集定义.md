# 1.2 场景集定义

场景是整个 Part 1 的基础。没有场景，就无法判断报警人说够没有；没有场景，就无法知道该追问什么。

---

## 场景层级结构

```
一级场景（警情大类）
    ↓
二级场景（业务场景）
    ↓
三级场景（细分场景）
```

---

## 一级场景集

| 编号 | 一级场景 | 说明 |
|------|----------|------|
| S1 | 火灾类 | 各类火灾警情 |
| S2 | 抢险救援类 | 人员被困、交通事故等救援 |
| S3 | 社会救助类 | 开门、充气、抓蛇等 |
| S4 | 灾害事故类 | 洪涝、塌方、危化品泄漏 |
| S5 | 其他警情类 | 误报、测试、重复报警 |

## 场景切换规则

1. **热词触发切换**：根据报警人描述中的关键词自动识别场景切换
2. **递进切换**：从一级→二级→三级，渐进细化
3. **回退切换**：当新描述否定当前场景假设时，回退到上级场景
4. **跨级切换**：特殊情况下可直接识别三级场景

## 场景画像聚类（六类）

根据`场景画像聚类知识.docx`，系统将警情分为六个场景画像聚类，作为问题集分层设计的基础：

| 聚类编号 | 聚类名称 | 问题集继承 | 说明 |
|----------|----------|------------|------|
| 聚类1 | 水域灾害类 | L0 + 水灾专项 | 暴雨内涝/河道涨水/地下室进水 |
| 聚类2 | 普通火灾类 | L0 + L1 | 住宅/商铺/普通建筑（默认主场景） |
| 聚类3 | 复杂空间与高危结构类 | L0 + L1 + 高危结构专项 | 高层/地下/厂房/化工 |
| 聚类4 | 人员密集类 | L0 + L1 + 人员密集专项 | 学校/医院/商场/车站 |
| 聚类5 | 重点单位类 | L0 + L1 + 重点单位专项 | 政府机关/机房/危化品单位 |
| 聚类6 | 极端紧急类 | L0 最小必要信息集 | 报警人慌乱/通话极短 |

**继承关系**：
```
极简模式(L0) → 普通火灾(+L1) → 特殊场景(+专项)
```
