# 聚类6-极端紧急类报警人类型

---

## 报警人类型详解

| 报警人类型 | 在这种场景下通常怎么说 | 容易主动提供的信息 | 容易漏掉的关键信息 |
| :--- | :--- | :--- | :--- |
| **当事人（极度恐慌）** | "快来快来！救命！着火了！烧到了！" "快来！有人……有人……" | 极度恐慌、情绪失控、声音颤抖或尖叫 | 大量结构化信息：地址、火灾情况、人员数量 |
| **当事人（通话中断）** | "喂？喂？这里是……（信号不好中断）" | 通话突然中断、信号问题 | 位置、警情类型、任何详细信息 |
| **当事人（生命危险）** | "我出不去了，烟把我围住了……" "门打不开了，热得烫……" | 生命受威胁、位置被困、无法逃生 | 标准地址、建筑信息、救援入口 |
| **当事人（重伤无法表达）** | "……痛……在流血……（声音微弱）" | 严重受伤、无法正常表达 | 任何结构化信息 |
| **报警人（混乱中）** | "着火了！很多人！快来！（背景嘈杂，有尖叫声）" | 紧急程度高、人数多、背景混乱 | 准确位置、具体楼层、伤亡情况 |
| **多方报警同一事件** | "我这里是XX路，有人报警说哪里着火了……" | 同一事件多人报警、信息碎片化 | 需要通过技术手段补位 |

---

## 典型对话示例

### 场景：通话极短中断

**报警人**："快来！救命！（通话中断，4秒）"
→ 系统识别：极短通话，信息严重缺失
→ 立即行动：
  - 立即调派最近消防站
  - 联动110获取报警人手机定位
  - 发送短信定位链接
  - 回拨电话持续监听

### 场景：位置模糊+情绪失控

**报警人**（哭喊）："救命……有人在流血……我不知道在哪……"
→ 系统识别：位置严重缺失，情绪极度恐慌
→ 优先追问："您在哪里？能看到周围有什么？"
→ 若无法获取位置：联动110定位，同步调派

### 场景：报警人语言混乱

**报警人**："那栋楼、对、烧了、好大的烟、有人在里面……"
→ 系统识别：语言混乱，信息碎片化
→ 系统判断：根据关键词"楼、烧、烟、里面"识别为建筑火灾
→ 优先获取位置：通过短信定位、联动110

## 信息可信度判断

| 报警人类型 | 信息 | 可信度 |
|------------|------|--------|
| 极度恐慌当事人 | "救命" | 高（紧急程度高） |
| 通话中断 | 无 | 严重缺失 |
| 混乱背景 | "好多人" | 中（无法量化） |
| 路人 | "好像有人" | 低（推测） |

## 路由调节策略

### 核心原则

> **先派后问** — 不追求问全，只追求"最小必要信息"。

### L0最小必要信息集

| 优先级 | 问题 | 目的 |
|--------|------|------|
| P0 | 您的位置？ | 立即定位 |
| P1 | 发生了什么？ | 判断类型 |
| P2 | 有没有人被困？ | 判断救援 |

### 特殊处理策略

| 情况 | 处理方式 |
|------|----------|
| 通话极短中断 | 立即调派 + 联动110定位 + 短信链接 |
| 无法获取位置 | 联动110/120/122多部门定位 |
| 多方报警同一事件 | 技术手段合并信息，交叉验证 |
| 报警人情绪失控 | 简化问题，单一关键信息获取 |

### 时间控制

- L0最小必要信息获取：**≤30秒**
- 超出时间未获取到位置：**立即调派，边派边问**

## 极端紧急类关键词识别

| 关键词 | 触发场景 | 风险等级 |
|--------|----------|----------|
| "救命""快来""着火了" | 紧急火灾 | 极高 |
| "救命""有人""流血" | 紧急救援 | 极高 |
| "快点""来不及了" | 紧急 | 极高 |
| 通话突然中断 | 极端紧急 | 极高 |
| "我妈在里面" | 有人员被困 | 高 |
| "不知道在哪" | 位置缺失 | 高 |
