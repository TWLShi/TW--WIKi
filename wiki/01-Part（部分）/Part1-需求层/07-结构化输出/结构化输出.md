# 1.7 结构化输出

结构化输出是 Part 1 的最终结果，输出七类结构化内容，让后续模块可以继续使用。

---

## 七类输出定义

| 输出内容 | 说明 | 示例 |
|----------|------|------|
| **场景识别结果** | 当前属于哪个警情大类、细场景 | S1-1-6（有人被困的住宅火灾） |
| **报警人表达模式** | 当前属于事件直陈/求救催促/现象描述等哪类 | 求救催促型 + 事件直陈型 |
| **报警人类型判断** | 当事人/路人/物业/单位人员等 | 当事人 |
| **已知信息字段** | 报警人已经提供了哪些信息 | 地址、起火位置、火情、人员被困 |
| **关键缺失字段** | 当前还缺哪些重要字段 | 具体地址、是否有人被困、火势阶段 |
| **推荐问题节点** | 下一步应该问哪个问题 | l0.pos → l0.risk → l0.fire.1 |
| **风险与调派价值** | 该问题为什么要问，对风险和调派有什么价值 | 影响救援优先级和调派规模 |

---

## 输出示例

### 示例1：普通火灾

**报警人说**："快来，我家厨房着火了，烟特别大。"

| 输出字段 | 结果 |
|----------|------|
| 警情大类 | 火灾扑救类 |
| 细场景 | 住宅厨房火灾 |
| 表达模式 | 求救催促型 + 事件直陈型 |
| 报警人类型 | 当事人 |
| 已知信息 | 家中、厨房、着火、烟大 |
| 关键缺失 | 具体地址、是否有人被困、火势阶段、燃烧物、燃气风险 |
| 推荐问题 | 先问具体位置，再问是否有人被困 |
| 推荐节点 | l0.pos → l0.risk → l0.fire.1 → l0.fire.2 |

### 示例2：水域灾害

**报警人说**："地下车库进水了，车里还有人，水还在涨。"

| 输出字段 | 结果 |
|----------|------|
| 警情大类 | 水域灾害类 |
| 细场景 | 地下车库进水/车辆被困 |
| 表达模式 | 事件直陈型 + 人员风险型 + 动态风险型 |
| 已知信息 | 地下车库、进水、车内有人、水位上涨 |
| 关键缺失 | 具体地址、水深、被困人数、触电风险、救援可达性 |
| 推荐问题 | 水有多深？被困人员能否转移？有没有电线或燃气风险？ |
| 推荐节点 | w.1 → w.2 → w.3 → w.5 |

---

### 示例3：人员密集场所

**报警人说**："商场三楼都是烟，里面很多人没出来。"

| 输出字段 | 结果 |
|----------|------|
| 警情大类 | 火灾扑救类 |
| 建筑场景 | 人员密集类 |
| 细场景 | 商场火灾/烟气疏散风险 |
| 表达模式 | 位置先行型 + 人员风险型 |
| 已知信息 | 商场、三楼、烟大、很多人未撤离 |
| 关键缺失 | 具体人数、人群类型、被困区域、出口情况、伤员情况、负责人 |
| 推荐问题 | 现场大概有多少人？还有多少人没撤离？出口是否畅通？ |
| 推荐节点 | c.1 → c.3 → c.5 |

---

### 示例4：重点单位

**报警人说**："我们单位配电房冒烟，有一股刺鼻味。"

| 输出字段 | 结果 |
|----------|------|
| 警情大类 | 火灾扑救类 |
| 建筑场景 | 重点单位类 |
| 细场景 | 配电房冒烟/电气风险 |
| 表达模式 | 单位专业型 + 现象描述型 |
| 报警人类型 | 单位人员 |
| 已知信息 | 单位、配电房、冒烟、刺鼻味 |
| 关键缺失 | 单位具体地址、是否断电、是否有人、危险源、负责人 |
| 推荐问题 | 请告诉我单位名称和具体地址；是否在重点区域；有没有高压设备或危险品 |
| 推荐节点 | k.1 → k.2 → k.3 |
