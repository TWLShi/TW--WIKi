# 极端紧急场景画像

极端紧急类不是某一种具体灾害，而是一种**接警状态模式**。它适用于报警人极度慌乱、通话时间极短、信号差或可能中断、无法完整描述的场景。

---

## 场景定义

| 属性 | 值 |
|------|------|
| 场景编号 | L0-Mode |
| 类型 | 接警状态模式 |
| 风险等级 | 极高 |
| 调派类型 | 先派后问 |

---

## 核心特征

适用于：
- 报警人极度慌乱
- 通话时间极短
- 信号差或可能中断
- 报警人无法完整描述
- 现场已出现人员被困、爆炸、坍塌、浓烟封堵
- 高并发情况下需要快速派警

**核心原则**：不是问全，而是"先拿到最小必要信息，先形成派警条件，再边派边问"。

---

## 场景画像要素

极端紧急类几乎全部结构化信息都会缺失，只保留L0最小必要信息：

### 最小必要要素（L0）

| 字段 | 类型 | 必须 | 说明 |
|------|------|------|------|
| district | string | 是 | 行政区划（模糊可） |
| community | string | 否 | 小区/地点名称 |
| landmark | string | 否 | 地标 |
| incidentType | string | 是 | 事件类型：火灾/水灾/被困 |
| hasTrapped | boolean | 是 | 是否有人被困 |
| trappedCount | string | 否 | 被困人数（模糊可） |
| dangerLevel | string | 是 | 危险程度：爆炸/坍塌/毒烟/高危 |

### 可选补充要素

| 字段 | 类型 | 必须 | 说明 |
|------|------|------|------|
| callerLocation | string | 否 | 报警人当前位置 |
| callerSafety | boolean | 否 | 报警人是否已安全 |
| canContact | boolean | 否 | 是否能保持联络 |

---

## 问题集：L0 最小必要信息集

### 极简必问问题

| 节点 | 问题 | 提取字段 |
|------|------|----------|
| l0.root | "您好，这里是119消防指挥中心！请问您遇到什么紧急情况？在什么位置？" | incidentType, district, community, landmark |
| l0.pos | "能再说一下具体位置吗？" | detailedAddress |
| l0.risk | "是否有人被困？有多少人？" | hasTrapped, trappedCount |

### 极端紧急类补充问题（如时间允许）

| 节点 | 问题 | 提取字段 |
|------|------|----------|
| e.1 | "危险程度现在怎么样？是爆炸、坍塌还是浓烟封堵？" | dangerLevel |
| e.2 | "您现在在哪里？是否安全？" | callerLocation, callerSafety |
| e.3 | "您的电话还能打通吗？" | canContact |

---

## 极简模式问询原则

1. **不追求完整**：不追求问全所有信息
2. **不连续追问低优先级信息**：只问最关键的
3. **满足派警条件，立即派警**：边派边补采
4. **后续由机器人、人工接警员或回拨继续补采**

## 继承关系

根据`场景画像聚类知识.docx`，六个模块之间的继承关系：

```
场景画像聚类6：极端紧急类 = L0 最小必要信息集

场景画像聚类2：普通火灾类 = L0 + L1 标准火灾研判问题集

场景画像聚类1：水域灾害类 = L0 + 水灾专项问题集

场景画像聚类3：复杂空间与高危结构类 = L0 + L1 + 高危结构专项问题集

场景画像聚类4：人员密集类 = L0 + L1 + 人员密集专项问题集

场景画像聚类5：重点单位类 = L0 + L1 + 重点单位专项问题集
```

---

## 典型场景识别

系统根据报警人回答做场景判断：

| 关键词 | 场景类型 |
|--------|----------|
| "水淹""暴雨""地下室进水" | 水域灾害类 |
| "高层""几十楼""楼梯都是烟" | 复杂空间与高危结构类 |
| "学校""医院""商场""很多人" | 人员密集类 |
| "单位""机房""实验室""配电室" | 重点单位类 |
| 无明显特殊标签 | 普通火灾类 |
| 极度慌乱/信息极少/有生命危险 | 极端紧急类 |
