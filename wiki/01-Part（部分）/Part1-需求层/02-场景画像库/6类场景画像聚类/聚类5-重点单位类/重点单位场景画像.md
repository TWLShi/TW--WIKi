# 重点单位场景画像

重点单位类包括政府机关、医院、学校、电力设施、通信机房、数据中心、危化品单位、大型仓储物流、文物古建、重要生产企业、应急保障单位等。

---

## 场景定义

| 属性 | 值 |
|------|------|
| 场景编号 | S1-10 |
| 一级场景 | 火灾类（S1） |
| 二级场景 | 重点单位 |
| 风险等级 | 高-极高 |
| 调派类型 | 重点调派（关联单位档案） |

---

## 核心特征

重点单位类场景的特点：
- **社会影响大**：涉及政府、医疗、教育等重要机构
- **资产价值高**：数据中心、文物古建等不可复制资产
- **可能涉及关键系统中断**：电力、通信、医疗系统
- **单位内部通常有预案、负责人、消防控制室**
- **需要快速关联单位档案和风险标签**

重点单位不是简单判断"火灾大小"，还要判断：
- 关键区域是否受影响
- 是否影响业务连续性
- 是否涉及危险源
- 是否需要通知负责人
- 是否需要启动单位预案

## 场景画像要素

### 位置要素

| 字段 | 类型 | 必须 | 说明 |
|------|------|------|------|
| district | string | 是 | 行政区划 |
| unitName | string | 是 | 单位名称 |
| detailedAddress | string | 是 | 详细地址 |
| buildingType | string | 是 | 单位类型：机关/医院/学校/机房/仓库/厂房 |
| keyArea | string | 是 | 关键区域：配电房/机房/实验室/仓库/控制室 |

### 危险源要素

| 字段 | 类型 | 必须 | 说明 |
|------|------|------|------|
| hazardousMaterial | boolean | 是 | 是否有危化品 |
| hazardousMaterialList | string | 否 | 危化品清单 |
| hasExplosionRisk | boolean | 是 | 是否有爆炸风险 |
| criticalEquipment | string | 否 | 关键设备类型 |

### 预案要素

| 字段 | 类型 | 必须 | 说明 |
|------|------|------|------|
| hasEmergencyPlan | boolean | 是 | 是否有应急预案 |
| planActivated | boolean | 否 | 预案是否已启动 |
| hasFireControlRoom | boolean | 是 | 是否有消防控制室 |
| fireControlRoomStaff | string | 否 | 消防控制室值守人员 |

### 协调要素

| 字段 | 类型 | 必须 | 说明 |
|------|------|------|------|
| hasResponsiblePerson | boolean | 是 | 负责人是否在现场 |
| responsiblePersonContact | string | 否 | 负责人联系方式 |
| hasCoordinator | boolean | 是 | 是否有现场协调人 |
| coordinatorContact | string | 否 | 协调人联系方式 |

### 设施状态要素

| 字段 | 类型 | 必须 | 说明 |
|------|------|------|------|
| powerCut | boolean | 是 | 是否已断电 |
| gasCut | boolean | 是 | 是否已断气 |
| sprinklerStatus | string | 否 | 消防设施状态 |
| hasBackupPower | boolean | 否 | 是否有备用电源 |

## 重点单位细分场景

| 三级场景 | 描述 | 关键缺失要素 |
|----------|------|--------------|
| S1-10-1 | 政府机关火灾 | 是否有保密区域、负责人是否在场、预案状态 |
| S1-10-2 | 医院火灾 | 患者数量、病房位置、是否需要特殊疏散 |
| S1-10-3 | 学校火灾 | 学生人数、是否有老师组织、疏散难度 |
| S1-10-4 | 机房/数据中心火灾 | 关键业务是否中断、是否有备用电源、气体灭火系统 |
| S1-10-5 | 危化品单位火灾 | 危化品种类、是否已泄漏、是否需要防化 |
| S1-10-6 | 文物古建火灾 | 是否已断电、文物价值、特殊灭火要求 |

## 问题集继承结构

```
重点单位类 = L0 + L1 普通火灾 + 重点单位专项问题集
```

### L0层（必须）

| 节点 | 问题 | 提取字段 |
|------|------|----------|
| l0.root | "您好，这里是119消防指挥中心！请问您遇到什么紧急情况？在什么位置？" | incidentType, sceneType, district, unitName |
| l0.pos | "能再说一下具体位置吗？" | detailedAddress, keyArea |
| l0.risk | "是否有人被困？有多少人？" | trappedCount, trappedLocation |

### L1层（普通火灾扩展）

| 节点 | 问题 | 提取字段 |
|------|------|----------|
| l1.fire.1 | "烟是什么颜色？浓不浓？" | smokeColor, smokeOdor |
| l1.fire.2 | "火是刚着起来，正在蔓延，还是已经烧得很大？" | fireStage, fireScale |
| l1.fire.3 | "起火位置大概在哪里？" | fireFloor, keyArea |
| l1.fire.4 | "这里是什么类型的建筑？" | buildingType |
| l1.fire.5 | "一共有多少层？起火在第几层？" | totalFloors, fireFloor |
| l1.fire.6 | "楼梯、走廊或出口有没有被烟火堵住？" | evacuationRouteClear, exitBlocked |
| l1.fire.7 | "现场有没有人正在灭火？是否已经断电断气？" | hasInitialFirefighting, powerCut, gasCut |

### 重点单位专项问题

| 节点 | 问题 | 提取字段 |
|------|------|----------|
| k.1 | "是什么单位？单位名称是什么？" | unitName |
| k.2 | "起火的具体区域是哪里？配电房、机房、实验室还是仓库？" | keyArea |
| k.3 | "现场有没有化学品、油料、气瓶或易燃易爆物？" | hazardousMaterial, hazardousMaterialList |
| k.4 | "负责人或值班人员现在在不在现场？有没有联系方式？" | hasResponsiblePerson, responsiblePersonContact |
| k.5 | "这个单位有没有消防控制室？值守人员在不在？" | hasFireControlRoom, fireControlRoomStaff |
| k.6 | "是否已经启动应急预案？" | hasEmergencyPlan, planActivated |
| k.7 | "现场有没有已经采取的应急措施？有没有断电断气？" | powerCut, gasCut |
| k.8 | "现场有没有重要设备或关键系统可能受影响？" | criticalEquipment |
| k.9 | "消防车能不能开到单位门口？有没有专门的消防通道？" | roadAccess |

## 关键缺失判断

**公式**：关键缺失要素 = 场景画像要求 - 报警人已提供

**示例**：报警人说"XX医院配电房着火了，烟很大"

| 已提供 | 缺失（按优先级） |
|--------|------------------|
| ✅ 单位名称（XX医院） | ❌ 具体位置（哪个配电房） |
| ✅ 位置类型（配电房） | ❌ 患者数量/病房位置 |
| ✅ 火情（烟很大） | ❌ 负责人是否在场 |
| | ❌ 预案状态 |
| | ❌ 是否已断电 |
| | ❌ 是否需要特殊医疗配置 |
