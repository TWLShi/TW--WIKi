# 水域灾害场景画像

水域灾害类是 S4 灾害事故类下的重要场景类型，主要对应暴雨内涝、城市积水、地下空间进水、河道涨水、人员被困、车辆被困等场景。

---

## 场景定义

| 属性 | 值 |
|------|------|
| 场景编号 | S4-1 |
| 一级场景 | 灾害事故类（S4） |
| 二级场景 | 水域灾害 |
| 风险等级 | 高 |
| 调派类型 | 特殊调派（水域救援装备） |

---

## 核心特征

水域灾害类具有以下特点：
- **高并发报警明显**：同一地点可能多人重复报警
- **报警人信息碎片化严重**：描述零散、不完整
- **生命风险与位置风险高于灾害本身**
- **水位变化具有动态性**：需要持续关注趋势
- **触电、燃气泄漏、地下空间缺氧等次生风险突出**

**判断重点不是"火势大小"，而是：**
- 水深多少
- 水是否还在上涨
- 人在哪里
- 是否能自主脱困
- 现场是否有电、气、地下空间等二次危险

## 场景画像要素

### 位置要素

| 字段 | 类型 | 必须 | 说明 |
|------|------|------|------|
| district | string | 是 | 行政区划 |
| community | string | 是 | 小区/地点名称 |
| detailedAddress | string | 是 | 详细地址 |
| waterLocation | string | 是 | 进水位置：地下室/地下车库/低洼区域/河道 |
| waterEntryPoint | string | 否 | 进水口位置 |

### 水情要素

| 字段 | 类型 | 必须 | 说明 |
|------|------|------|------|
| waterDepth | string | 是 | 水深（需归一化到人体部位） |
| waterLevelTrend | enum | 是 | 水位趋势：上涨/稳定/下降 |
| waterFlowSpeed | enum | 否 | 水流速度：缓/急 |
| waterDisasterType | string | 是 | 灾害类型：暴雨积水/河水漫出/山洪/城市内涝 |

### 人员要素

| 字段 | 类型 | 必须 | 说明 |
|------|------|------|------|
| trappedCount | int | 是 | 被困人数 |
| trappedLocation | string | 是 | 被困位置 |
| hasVulnerablePeople | boolean | 是 | 是否有老人/儿童/病人 |
| canSelfRescue | boolean | 是 | 是否能自主脱困 |

### 次生危险要素

| 字段 | 类型 | 必须 | 说明 |
|------|------|------|------|
| hasElectrocutionRisk | boolean | 是 | 是否有触电风险（电线落水/电器泡水） |
| hasGasLeak | boolean | 是 | 是否有燃气泄漏风险 |
| hasCollapseRisk | boolean | 否 | 房屋是否有倒塌风险 |
| isUnderground | boolean | 是 | 是否为地下空间 |
| hasOxygenRisk | boolean | 否 | 地下空间是否缺氧 |

### 救援可达要素

| 字段 | 类型 | 必须 | 说明 |
|------|------|------|------|
| rescueAccessible | boolean | 是 | 救援车辆能否到达 |
| needBoat | boolean | 否 | 是否需要舟艇装备 |
| roadAccess | string | 否 | 进场路线描述 |

## 水域灾害细分场景

| 三级场景 | 描述 | 关键缺失要素 |
|----------|------|--------------|
| S4-1-1 | 暴雨积水内涝 | 水深、水位趋势、被困人数 |
| S4-1-2 | 地下室进水 | 是否有触电风险、人员位置、是否能开门 |
| S4-1-3 | 河道涨水被困 | 水流速度、被困位置、是否有舟艇条件 |
| S4-1-4 | 山洪暴发 | 水位趋势、是否有泥石流风险 |
| S4-1-5 | 车辆被水淹 | 被困人数、是否在车内、水深 |

## 问题集继承结构

```
水域灾害类 = L0 极简必问 + 水灾专项问题集
```

### L0层（继承）

| 节点 | 问题 | 提取字段 |
|------|------|----------|
| l0.root | "您好，这里是119消防指挥中心！请问您遇到什么紧急情况？在什么位置？" | incidentType, sceneType, district, community |
| l0.pos | "能再说一下具体位置吗？具体到几栋几单元几楼几号？" | detailedAddress, landmark |
| l0.risk | "是否有人被困？有多少人？" | trappedCount, trappedLocation, canSelfRescue |

### 水灾专项问题

| 节点 | 问题 | 提取字段 |
|------|------|----------|
| w.1 | "是什么类型的水灾？暴雨积水内涝、河水漫出还是山洪？水大概有多深？" | waterDisasterType, waterDepth |
| w.2 | "水位还在上涨吗？水流急不急？被困的人能不能自己转移到安全的地方？" | waterLevelTrend, waterFlowSpeed, canSelfRescue |
| w.3 | "现场有没有电线落水或电器泡在水里的情况？有没有闻到燃气味？" | hasElectrocutionRisk, hasGasLeak |
| w.4 | "附近的房子有没有倒塌风险？是不是地下车库或地下室进水了？" | hasCollapseRisk, isUnderground |
| w.5 | "一共有多少人需要救援？其中有没有老人、小孩或病人？救援车辆能开过去吗？还是需要船？" | trappedCount, hasVulnerablePeople, rescueAccessible, needBoat |

## 关键缺失判断

**公式**：关键缺失要素 = 场景画像要求 - 报警人已提供

**示例**：报警人说"地下室进水了，我爸还在里面出不来"

| 已提供 | 缺失（按优先级） |
|--------|------------------|
| ✅ 位置（某地下室） | ❌ 水深（到腰部/颈部） |
| ✅ 人员被困（父亲） | ❌ 水位趋势（上涨/稳定） |
| ✅ 场景类型（地下室进水） | ❌ 是否有触电风险 |
| | ❌ 是否有燃气泄漏 |
| | ❌ 救援车辆能否到达 |
