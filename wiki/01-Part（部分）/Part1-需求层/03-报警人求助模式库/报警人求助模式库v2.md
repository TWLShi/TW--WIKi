# 报警人求助模式库

*定义：沉淀报警人在真实警情通话中的自然表达规律，解决"报警人通常会怎么描述警情"的问题。*

---

## 一、模块定义

报警人求助模式库研究的是：
- 报警人通常会怎么开口
- 报警人会怎么描述现场
- 报警人会主动说出哪些信息
- 报警人容易遗漏哪些关键信息
- 不同场景、不同身份、不同紧急程度下表达方式有什么差异

**核心价值**：帮助 AI 理解报警人的自然语言，把口语化、碎片化、模糊化的描述映射到场景画像库，支撑关键缺失识别和动态问题路由。

---

## 二、报警人自然表达的六大总体规律

### 规律1：先说危险，不先说结构化信息

报警人第一反应是说"出事了"、"很危险"、"快来"，而不是先说完整地址、楼层、人员数量。

**常见表达**：
- "着火了！"
- "快来啊！"
- "有人在里面！"
- "烟特别大！"
- "要爆炸了！"
- "我妈还在里面！"

### 规律2：先说自己看到的，不一定说真实原因

报警人描述的是**感知到的现象**，而不是专业判断。

| 感知现象 | 可能隐含的场景 |
|----------|---------------|
| "冒烟了" | 可能是火灾 |
| "有刺鼻味" | 可能是危化/燃气/电气 |
| "楼道全是黑烟" | 高层火灾+逃生受阻 |
| "水淹上来了" | 水域灾害 |
| "有人喊救命" | 人员被困 |

### 规律3：地址表达常常不完整

| 表达层级 | 示例 |
|----------|------|
| 小区级 | "就在我们小区这边" |
| 地标级 | "XX路口附近"、"那个大超市旁边" |
| 模糊级 | "我不知道具体门牌" |
| 辅助级 | "里面有个小路，导航到不了" |

### 规律4：人员风险表达常常模糊

| 模糊表达 | 对应结构化字段 |
|----------|---------------|
| "里面还有人" | 是否有人被困 |
| "我妈还在里面" | 被困人数、关系 |
| "孩子没出来" | 被困人员类型 |
| "有人晕倒了" | 是否受伤、严重程度 |
| "病人转不出来" | 是否需要特殊救援 |

**关键**："有人在里面"是高风险信号，必须触发高优先级路由。

### 规律5：报警人常用程度词，不常用量化词

| 程度词 | 量化含义（需归一化） |
|--------|---------------------|
| "火很大" | 发展阶段/蔓延状态 |
| "烟特别大" | 烟雾浓度高 |
| "水很深" | 水深到人体部位 |
| "涨得很快" | 水位上升趋势 |
| "到腰了" | 水深约0.8m |

### 规律6：报警人会重复强调最恐惧的点

重复表达是**强风险信号**：
- "快来快来快来！"
- "我妈还在里面！"
- "烟太大了！"
- "要爆炸了！"

## 三、六类基本表达形态

### 1. 事件直陈型

**特征**：直接说发生了什么

**常见表达**：
- "着火了。"
- "我家厨房着火了。"
- "楼上冒烟了。"
- "车烧起来了。"

**AI理解**：可快速识别警情大类；进入对应场景路由；但还需要补位置、人员、风险源。

### 2. 现象描述型

**特征**：不确定具体是什么，只描述感知到的现象

**常见表达**：
- "楼道里全是烟。"
- "闻到一股焦糊味。"
- "有刺鼻气味。"
- "刚才听到一声巨响。"

**AI理解**：
- "冒烟"进入火灾或电气异常判断
- "刺鼻味"可能触发危化、燃气、有毒烟气风险
- "地下车库进水"进入水域灾害或地下空间风险
- "楼道看不见"触发通道受阻和烟气风险

### 3. 求救催促型

**特征**：优先表达救援请求，信息量少但风险强度高

**常见表达**：
- "快来！"
- "快点来救人！"
- "来不及了！"
- "我出不去了！"

**AI理解**：
- 表达模式属于极端紧急
- 不适合完整问卷
- 应进入L0最小必要信息链
- 优先确认位置、人员、事件类型

### 4. 位置先行型

**特征**：先说位置，再说事情

**常见表达**：
- "XX小区8栋着火了。"
- "XX商场三楼冒烟了。"
- "XX医院住院部有烟。"
- "XX路口有车烧起来了。"

**AI理解**：
- 定位信息较好
- 可以减少位置追问
- 优先补人员、风险源、通道、发展趋势

### 5. 人员风险型

**特征**：报警人第一重点是"人"

**常见表达**：
- "有人在里面！"
- "孩子没出来！"
- "病人转不出来！"
- "学生还在楼上！"
- "有人晕倒了！"
- "车里还有人！"

**AI理解**：
- 直接触发生命风险优先级
- 问题路由应优先补充人数、位置、状态、能否自救
- 必要时同步医疗或增援判断

### 6. 不确定推测型

**特征**：报警人用"好像、可能、听说、不确定"等表达

**常见表达**：
- "好像着火了。"
- "可能有人在里面。"
- "听说是煤气。"
- "我不确定是几楼。"
- "好像是仓库那边。"

**AI理解**：
- 字段可以暂存，但置信度较低
- 需要通过后续问题确认
- 不能直接作为高确定性结论

## 四、报警人主动提供的信息分类

| 信息类别 | 主动表达示例 | 用途 |
|----------|-------------|------|
| 事件标签 | "着火了""冒烟了""爆炸了" | 场景识别 |
| 现象强度 | "烟很大""水很深""涨得很快" | 风险判断 |
| 位置线索 | "XX小区""三楼""地下室" | 定位 |
| 人员风险 | "有人在里面""孩子没出来" | 生命风险 |
| 危险源 | "煤气罐""电动车""化学品" | 次生风险 |
| 通道可达 | "楼梯都是烟""车进不来" | 救援难度 |

---

## 五、报警人常见缺失规律

| 缺失类别 | 常见缺失项 | 缺失原因 |
|----------|-----------|----------|
| 地址类 | 详细门牌、楼栋单元、楼层 | 紧张时优先说危险 |
| 人员类 | 被困人数、状态、类型 | 只知道"有人" |
| 风险源类 | 燃烧物、燃气、电池 | 不具备专业判断 |
| 发展趋势 | 火势扩大、烟扩散、水上涨 | 只描述当前现象 |
| 通道类 | 楼梯可用性、消防车入口 | 不了解对救援的影响 |

## 六、报警人回答时的反应模式

### 确定回答型
能直接回答 → 可信度高，直接填充字段

### 不确定回答型
模糊回答（"好像是12楼"） → 标记低置信度

### 回避/无能力回答型
"我不知道""我不敢过去" → 减少追问，切换极简模式

### 行动反馈型
"我已经跑出来了""物业正在疏散" → 更新风险状态

### 二次更新型
"火变大了""烟到楼下了" → 触发问题路由更新

---

## 七、报警人求助模式与问题路由的关系

```
报警人自然表达
        ↓
识别表达模式（六类形态）
        ↓
识别报警人身份/表达能力
        ↓
匹配场景画像库
        ↓
判断已提供字段
        ↓
识别高频缺失字段
        ↓
生成下一步问题路由
```

## 八、典型场景识别示例

### 示例1
**报警人说**："快来，我家厨房着火了，烟特别大！"

| 识别项 | 结果 |
|--------|------|
| 表达模式 | 求救催促型 + 事件直陈型 |
| 场景 | S1-1-1 住宅厨房火灾 |
| 已提供 | 位置(某小区)、事件类型(火灾)、位置(厨房)、烟大 |
| 缺失 | 详细地址、楼层、是否有燃气罐、是否有人员被困 |
| 路由优先级 | 1.确认地址 2.确认被困人员 3.确认危险源 |

### 示例2
**报警人说**："我们商场三楼都是烟，里面还有很多人没出来。"

| 识别项 | 结果 |
|--------|------|
| 表达模式 | 事件直陈型 + 人员风险型 |
| 场景 | S1-9 人员密集场所火灾 |
| 已提供 | 位置(商场三楼)、烟大、人员被困(很多) |
| 缺失 | 详细地址、被困人数、被困位置、是否蔓延 |
| 路由优先级 | 1.确认被困人数位置 2.确认通道情况 3.确认危险源 |
