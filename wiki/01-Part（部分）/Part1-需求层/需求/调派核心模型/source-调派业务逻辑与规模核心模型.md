# 调派业务逻辑与规模核心模型

*摘要：本文档定义消防调派智能系统的核心业务逻辑，包含规模计算模型、编成逻辑、责任机制和约束校验规则。*

## 1. N_total 规模计算模型

### 核心公式

```
N_total = N_base + α_sub + β_struct + G_special + M_support + Δ_confidence + Δ_scene
```

### 因子说明

| 因子 | 含义 | 说明 |
|------|------|------|
| N_base | 基础编成 | 根据警情等级确定的基础车辆数 |
| α_sub | 子场景系数 | 根据12类子场景画像确定的调整系数 |
| β_struct | 建筑结构系数 | 根据建筑物结构类型确定的调整系数 |
| G_special | 特殊危险物质系数 | 危险化学品、锂电池等特殊物质 |
| M_support | 增援系数 | 需要增援时的额外力量 |
| Δ_confidence | 置信度调整 | 根据信息完整度调整 |
| Δ_scene | 场景调整 | 根据实际场景特征调整 |

### 编成等级表

| 等级 | 基础编成 | 车辆数 |
|------|----------|--------|
| 一级 | 水罐1 + 抢险1 | 2辆 |
| 二级 | 水罐2 + 举高1 + 抢险1 | 4辆 |
| 三级 | 水罐4 + 举高2 + 特勤1 + 抢险1 | 8辆 |
| 四级 | 水罐6 + 泡沫3 + 举高2 + 特勤2 + 洗消1 + 医疗1 | 15辆 |

## 2. 约束校验机制

系统在生成调派方案前进行多维度约束校验：

### 2.1 资源约束
- 队站可用车辆数 ≤ 总车辆数 × 出车率上限（70%）
- 单中队出车率 ≤ 70%

### 2.2 时间约束
- 响应时间窗口
- 到达时间预估

### 2.3 安全约束
- 危险物质识别
- 爆炸/中毒风险评估
- 建筑结构稳定性

## 3. 责任机制

### 3.1 责任主体
- **系统**：AI决策推荐
- **人工**：最终确认授权

### 3.2 确认节点
关键决策点需要人工确认：
- 大规模调派（≥三级）
- 危险物质事件
- 特殊场景（医院、学校等）
