# 问题路由

*摘要：问题路由是智能问询的核心机制，通过意图识别、要素补全和风险评估三个维度，将报警人的碎片化描述转化为结构化的调派决策信息。*

## 1. 三维度模型

### 1.1 意图维
识别报警人的**核心意图**：
- 火灾报警
- 人员救援
- 危险物质泄漏
- 设施故障

### 1.2 问题维
根据意图确定**必须问询的问题集**：
- 针对不同场景预设问题模板
- 根据回答动态调整下一问

### 1.3 画像维
关联**子场景画像**：
- 匹配12类典型场景
- 调用对应的问题模板

## 2. 问题路由树

### 建筑火灾场景
```
初始问题 → 建筑类型 → 燃烧物质 → 人员被困 → 蔓延趋势
```

### 危险物质场景
```
初始问题 → 物质类型 → 泄漏范围 → 人员疏散 → 环境风险
```

## 3. 与Query_Routing的关系

问题路由是Query_Routing的技术实现：
- **问题路由** — 关注问什么
- **Query_Routing** — 关注怎么问（路径优化）
