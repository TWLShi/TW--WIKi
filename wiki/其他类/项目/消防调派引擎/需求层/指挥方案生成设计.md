# FireCommand AI 指挥方案生成设计

**标签**：#指挥方案 #AI生成 #Orchestrator #LangGraph #FireCommand
**位置**：01_Requirements/

---

## 生成流程

```
输入（SOP表单） → Orchestrator多代理处理 → 输出指挥方案 → 人机确认 → 执行
                    ↓
              Situation Analyst（灾情分析）
              Resource Dispatcher（资源调派）
              Tactics Planner（战术规划）
              Risk & Safety Advisor（风险评估）
              Report Generator（方案生成）
```

---

## 代理角色

| 代理 | 功能 |
|------|------|
| **Situation Analyst** | 分析灾情、风险、优先级 |
| **Resource Dispatcher** | 计算力量调派（车辆/人员/装备） |
| **Tactics Planner** | 生成战术部署与时间轴 |
| **Risk & Safety Advisor** | 输出风险提醒与备案 |
| **Report Generator** | 产生结构化指挥方案 |

---

## 输出结构（固定格式）

| 区块 | 内容 |
|------|------|
| **1. 事件概况** | 报警时间/类型/位置/灾情/等级判定 |
| **2. 立即行动指令** | 第一出动力量/到达时间/指挥部位置 |
| **3. 战术部署与时间轴** | 阶段1（0-10分）/阶段2（10-30分）/阶段3（30分后） |
| **4. 资源调派清单** | 已调派/后备待命/社会联动 |
| **5. 风险评估与安全提醒** | 高风险/安全措施/特殊注意 |
| **6. 指挥建议与备案** | 建议模式/备案方案 |
| **7. 后续跟进要求** | 反馈频率/确认要求 |

---

## System Prompt 要点

```ts
// 核心约束
- 你是资深消防指挥官，20年+经验
- 严格遵守《119接警调度工作规程》、GB 16281、ICS原则
- 生命救助永远优先（搜救 > 灭火 > 财产保护）
- 力量调派要合理、保守
- 语言简洁，使用标准消防术语
- 方案必须可审计、可追踪
```

## 性能目标（ MVP）

| 指标 | 目标 |
|------|------|
| 首次生成 | ≤ 30 秒 |
| 调整生成 | ≤ 10 秒 |
| 幻觉率控制 | ≤ 5% |

---

## 优化点

| 优化 | 说明 |
|------|------|
| **历史案例检索** | 每次生成前，从向量库检索 3-5 个相似事件作为上下文 |
| **实时调整** | 现场反馈上传后自动触发调整方案生成 |
| **人机确认** | AI 生成后必须指挥员点击"确认下达"才生效 |
| **审计日志** | 每次生成/修改记录操作人、时间、修改前后对比 |
