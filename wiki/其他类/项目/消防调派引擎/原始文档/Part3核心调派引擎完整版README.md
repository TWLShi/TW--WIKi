# Part 3：核心调派引擎 ★

**所属Part**：Part 3（系统核心）
**物理目录**：`06_DispatchEngine`
**文档数量**：18份
**最后更新**：2025-04-24

**一句话定义**：
本Part是整个消防调派智能系统的**大脑和执行中枢**，负责从警情接收到最终方案下达的**全链路智能决策**，采用"规则 + ML + 多智能体 + 人工确认"的混合架构。

---

## 1. 引擎总体架构

**设计原则**：
- 安全第一（多重约束 + 人工最终确认）
- 可解释性（每一步都有规则解释 + ML SHAP解释）
- 可审计性（全程不可篡改日志）
- 持续学习（人工反馈驱动模型迭代）

**核心技术栈**：
- 规则引擎 + ML混合（XGBoost/LightGBM）
- 4-Agent多智能体协同（感知、决策、执行、预测）
- GIS实时计算 + 全局资源优化
- 区块链式审计留痕

## 3. 各步骤核心职责与输出（详细版）

| 步骤 | 名称 | 核心技术 | 主要输出 | 是否可能回退 |
|------|------|----------|----------|--------------|
