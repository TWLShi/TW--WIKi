# FRICP MVP功能优先级与交付计划（最终极简版）

**版本**：最终极简版
**前提**：复用现有接处警系统、现场指挥系统、舆情模块

---

## MVP功能优先级排序（MoSCoW方法）

### Must（必须实现 — MVP核心，决定是否能上线）

| 功能 | 说明 |
|------|------|
| 接处警系统扩展 | 自动提示七类敏感情况和较大/重大事故（简单规则引擎） |
| 消控室电话拨打 | 重点单位消控室联系电话一键拨打 + 核实记录 |
| **人工录入信息功能** | 后方指挥中心支持手动录入或补充客观事实（伤亡、被困、险情描述、处置进展） |
| 智能报告模板 | 自动填充事实部分 + 指挥员人工审核确认（支持从人工录入内容中选择填充） |
| 50分钟倒计时提醒 | 重大以上事故专用 |
| 一键上报 + 实时共享 | 国家消防救援局 + 同级应急 |
| 外部数据责任标注 | 所有外部数据自动标注责任声明 |
| 单向数据推送 | 事实核实上下文 → 报告生成上下文 |
| **权限复用** | 复用现有接处警系统权限体系，仅扩展必要细粒度权限（事实核实、报告审核） |
| 区块链基础存证 | 关键操作记录 |

### Should（应该实现 — 强烈推荐放入MVP，提升可用性）

| 功能 | 说明 |
|------|------|
| 动态续报自动提示 | 当新事实回传或人工录入时触发 |
| 舆情辅助显示 | 舆情模块数据在辅助决策区显示（仅后方可见） |
| 外部公共信息接入 | 地震、气象接入并标注责任 |
| 历史记录查询 | 人工录入信息的历史记录查询与版本对比 |

### Could（可选 — 如果时间允许）

| 功能 | 说明 |
|------|------|
| 微型消防站回传 | 简单任务下达与回传 |
| 敏感情况自动标记 | 报告模板中自动标记敏感情况类型 |
| 审计日志查询 | 审计日志查询界面（基础版） |

### Won't（本阶段不做）

- 任何大模型、多模态分析、视频分析、物联感知、对讲机转写

## 分阶段交付计划

### 迭代0：基础准备（2-3周）约45人日

**目标**：复用权限 + 打好合规地基

| 任务 | 内容 |
|------|------|
| T0-01 | 复用接处警系统权限体系，扩展细粒度权限（事实核实、报告审核） |
| T0-02 | 数据来源自动标注机制 |
| T0-03 | 区块链基础存证模块 |
| T0-04 | 消控室联系电话数据库与一键拨打功能 |
| T0-05 | 人工录入信息功能的基础表单框架 |
| T0-06 | 项目基础环境配置、接口打通测试 |

### 迭代1：MVP核心 — 合规报告闭环（5-7周）约110人日

**目标**：实现"能接警、能判断、能规范上报"的最小可用版本

| 任务 | 内容 |
|------|------|
| T1-01 | 接处警系统扩展：敏感情况自动提示 |
| T1-02 | 人工录入信息功能（完整实现，与报告模板联动） |
| T1-03 | 智能报告模板（支持从人工录入内容填充事实部分） |
| T1-04 | 50分钟倒计时 + 一键上报 |
| T1-05 | 动态续报基础提示 |
| T1-06 | 事实核实上下文 → 报告生成上下文单向推送 |
| T1-07 | 端到端集成测试 + 合规审查 |
| T1-08 | 人工录入与报告模板联动测试 |

**验收重点**：人工录入内容能正确进入事实部分并支持完整上报流程；权限复用后现场无法访问报告功能。

### 迭代2：优化与完善（4-6周）约65人日

**目标**：提升易用性和覆盖度

| 任务 | 内容 |
|------|------|
| T2-01 | 舆情模块数据辅助显示 |
| T2-02 | 外部公共信息接入 |
| T2-03 | 人工录入历史记录查询与版本对比 |
| T2-04 | 审计日志查询界面 |
| T2-05 | 完整系统集成测试 + 合规审查 + 性能优化 |
| T2-06 | 用户培训材料与操作手册编写 |

## 总体时间与资源建议

| 维度 | 数值 |
|------|------|
| **总周期** | 11-16周（约3-4个月） |
| **团队建议** | 2-3名开发 + 1名产品 + 1名业务/合规专家 |
| **重点工作量** | 接口集成和人工录入功能 |
| **预算影响** | 因复用接处警权限体系，整体MVP预算可控制在**320-380万元** |
