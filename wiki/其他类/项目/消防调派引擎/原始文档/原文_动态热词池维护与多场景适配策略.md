# 动态热词池维护与多场景适配策略

**来源**：119消防接警机器人（火灾+开锁/马蜂窝等社会救助场景）
**版本**：V1.0

---

## 1. 热词池结构设计

```json

  "id": "openlock_001",

}
```

### 场景划分
- **fire**：火灾救援（烟火发展、建筑结构）
- **openlock**：开锁（门外、钥匙、孩子哭）
- **waspnest**：马蜂窝（阳台、高空、叮咬风险）
- **general**：通用位置、受困状态（所有场景共用）

---

## 2. 多场景适配核心机制

### 机制1：场景感知激活
- 根据当前问题 + 已提取画像（rescueType槽位）自动激活对应场景热词
- 示例：
  - 问题"现场情况怎么样？"+ 检测到"锁/钥匙" → 激活 openlock 场景
  - 问题"烟的情况如何？"+ 检测到"烟/火" → 激活 fire 场景

### 机制2：两阶段动态注入
1. **Pre-Retrieval**：从场景热词池快速筛选Top-20~40条
2. **Contextual Injection**：只把筛选后的小列表注入ASR

### 机制3：上下文感知过滤
当已知"起火部位=厨房"时：
- 加载：厨房冒烟、油烟、 residential组
- 过滤掉：厂房、仓库、钢结构等非住宅热词

## 4. 维护策略

### 策略1：自动化发现（大模型驱动）
- 每周/每月批量处理机器人日志
- Teacher大模型提炼新同义词/新变体
- 6人团队轻量审核

### 策略2：定期清理
- 每月一次全量维护
- 低频删除（usage_count < 5）
- 相似合并（编辑距离 < 2）
- 召回率调优（recall_rate < 0.7 → 降低权重）

### 策略3：闭环反馈
- 高召回热词自动提升权重
- 版本控制（v20260328）
- 监控指标：池大小、召回率、WER变化

## 5. 各场景热词示例

### fire场景
- 发展状况：正在蔓延、火势增大、猛烈燃烧、已经被控制、爆燃风险
- 烟：黑烟滚滚、浓烟、薄烟、黄烟
- 建筑：高层住宅、厂房、钢结构、防火分区

### openlock场景
- 锁在门外、进不去家、钥匙忘里面、车门锁住了、孩子在里面哭

### waspnest场景
- 马蜂窝、蜂飞得厉害、一大窝马蜂、阳台蜂窝、叮咬风险、夜间处理

### general场景（跨场景共享）
- 位置：小区、楼栋、电梯内、高层阳台
- 受困：老人、孩子、无法出去、被困

```python
def get_context_aware_hotwords(current_profile, current_question):
    # 第一层：场景过滤
    scene = infer_scene(current_profile)
    
    # 第二层：画像上下文过滤
    if current_profile.get("ignition_point") == "厨房":
        hotwords.extend(get_hotwords_by_group("kitchen_smoke"))
        hotwords.extend(get_hotwords_by_group("residential"))
        hotwords = [hw for hw in hotwords if hw["type"] not in ["factory", "warehouse"]]
    
    # 第三层：问题感知过滤
    if "烟" in current_question:
        hotwords = filter_by_group(hotwords, "smoke_development")
    
    return select_top_k(hotwords, k=30)
```
