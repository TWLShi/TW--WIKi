# 消防调派智能系统核心引擎流程（v3.1）

## 整体流程

🚨 119接警开始

1. 警情接收与初步解析
   语音转文字 + 地址提取 + 标签打标

2. Query_Routing 问题路由与智能问询
   - 非对称问询路由 + 动态话术
   - 标准话术库（位置/规模/伤亡）
   - 问题路由树（多分支决策）
   - 补充问题维度（时间/危化/特殊人群）

3. 多维要素识别
   - 空间维度：地址/建筑/楼层
   - 时间维度：起火时间/报警延迟
   - 规模维度：面积/烟雾
   - 风险维度：人员/危化品
   - 维度映射体系

4. 警情等级判定
   - 一级：普通住宅 2-4辆
   - 二级：较大建筑 3-5辆+举高
   - 三级：高层/地下 7-13辆
   - 四级：化工危化 13-20辆

5. 调派规模计算与编成
   规模计算模型（基础编成 + 加权系数）

6. 队站与车辆匹配
   - 队站逻辑：距离最优 + ETA
   - 车辆匹配：类型 + 状态查询
   - 多站点联动规则

7. 约束条件校验
   - 资源可用性
   - 距离/时间约束
   - 特殊环境（城中村/高层/化工）
   - 编成规模合法性
   - 政策预案约束

8. 反向模拟与冲突解决
   - 方案模拟推演
   - 资源冲突检测
   - 自动调整建议

9. AI多智能体协同决策
   - 场景画像匹配
   - DIKW推理
   - 多Agent协同
   - 指挥方案生成
   - 火势蔓延预测

10. 人工二次确认
    - 调度员二次确认
    - 修改补充入口
    - 责任追溯记录

11. 调派方案生成与输出
    指挥方案 + 路径规划 + 指令下达

12. 审计留痕与责任记录
    全流程日志 + 责任追溯 + 优化

## 决策循环

- 第8步冲突 → 返回第2步或第5步调整
- 第10步人工修改 → 返回第5步重新计算

## 核心引擎特点

- 端到端闭环（接警 → 问询 → 决策 → 确认 → 输出 → 审计）
- AI辅助 + 人工把关双保险
- 强约束 + 可追溯责任
