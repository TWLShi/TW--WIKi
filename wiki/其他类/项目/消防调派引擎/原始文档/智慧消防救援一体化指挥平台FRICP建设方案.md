# 智慧消防救援一体化指挥平台（FRICP）建设方案

**汇报对象**：消防救援机构领导、指挥中心负责人
**汇报时长**：12-15分钟
**版本**：最终极简版（2026年）

---

## 第1页：标题页

**核心思路**：
- 在已有接处警系统和现场指挥系统基础上，做一次**轻量化扩展**
- 严格落实原图"八条硬规矩"
- 实现"前场只打仗、后场管指挥与上报"的职责清晰模式

**方案原则**：极简
- 不接入对讲机
- 不做视频分析
- 不做物联感知
- **无大模型**
- 最大限度降低成本和风险
- 增加**人工录入事实功能**（解决伤亡数据难获取问题）

## 第3页：项目背景与建设必要性

**原图要求**：明确"八条硬规矩"和"七类突出敏感情况"——信息报告的刚性纪律

**当前痛点**：
- 现场人员报告压力大，伤亡等关键事实数据难以及时获取
- 后方指挥中心判断敏感情况依赖人工，容易遗漏或标准不统一
- 外部数据责任边界不清

**FRICP定位**：极简扩展，重点解决：
- 合规报告
- 职责隔离
- 不新增复杂采集模块

---

## 第4页：核心设计原则

| 原则 | 说明 |
|------|------|
| **职责清晰、责任后移** |后方指挥中心承担全部上报责任，现场零上报责任 |
| **报告合规优先** | 正式报告仅填写已发生事实，辅助参考部分完全隔离 |
| **数据责任明确** | 外部数据必须标注"消防不承担责任" |
| **极简可落地** | 复用现有系统，支持人工录入事实补充伤亡数据，无大模型、无视频、无物联 |

---

## 第5页：系统总体架构

**极简扩展架构**：在现有系统基础上增加**轻量报告助手层**

### 三个有界上下文
| 上下文 | 职责 |
|--------|------|
| **事实核实上下文** | 负责客观事实核实 + 人工录入 |
| **报告生成上下文** | 核心，最严格，只允许事实部分 |
| **合规审计上下文** | 全局存证追溯 |

**数据流向**：现场指挥系统保持现状 → 简单事实回传 → 单向流动 → 责任边界清晰

---

## 第6页：核心功能设计点

### 后方指挥中心功能（核心）
| 功能 | 说明 |
|------|------|
| 接处警扩展 | 自动提示七类敏感情况和较大/重大事故 |
| 重点单位消控室联系电话 | 一键拨打与核实 |
| **人工录入信息功能** | 后方手动补充客观事实（伤亡、被困等） |
| 智能报告模板 | 自动填充事实部分 + 指挥员人工审核确认 |
| 50分钟倒计时提醒 | + 一键上报 + 实时共享 |
| 外部数据责任标注 | 自动标注"消防不承担责任" |

### 现场系统定位
- 仅保留简单回传功能
- 确保零上报责任
- 专注救援

---

## 第7页：现场指挥系统定位

**保持现有状态不变**，仅增加：
- 简单客观事实回传功能（文本、数值、位置标记）

**现场人员完全无**：
- 报告模板
- 上报按钮
- 分析功能

数据单向回传至后方，确保专注救援。

## 第9页：迭代规划

| 迭代 | 周期 | 内容 |
