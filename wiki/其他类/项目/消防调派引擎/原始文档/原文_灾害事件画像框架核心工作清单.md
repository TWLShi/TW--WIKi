# 根据标准灾害事件画像框架及字段完成的核心工作清单

**来源**：基于RDF/OWL/SHACL/SWRL本体 + HIP Ontology/DEO/地质灾害链KG

---

## 1. 框架固化与标注指南编制（最优先，1-2周）

### 本体/框架完整化
用Protégé将现有字段正式化为RDFS/OWL（domain/range、subClassOf、TransitiveProperty如hasPart）

补充：
- SHACL shapes（强制每个DisasterEvent必须有时空+至少1个因果/部分关系）
- SWRL规则（因果传递、蕴含推断）

### 编制《灾害画像标注指南》（Annotation Guideline）

明确**每个字段的定义、边界、示例**（含台湾口语变体，如"強顱""淹大水"）

包含：
- 实体类型定义
- 关系类型定义（上位/下位、整体/部分、蕴含、因果 + 新增时空/强度）
- 歧义处理规则（e.g. "中部"指行政区还是地理区？）

输出：PDF指南 + Prodigy/BRAT标注工具配置文件

---

## 2. 实体识别维度设计与扩展（核心，决定准确率）

**通用NER起点（PER/LOC/ORG/TIME/PERCENT）在灾害画像场景严重不足，会导致遗漏80%以上领域知识**

### 推荐实体维度（8-12类）

| 维度 | 具体类型示例 | 为什么必须增加？ | 与框架字段映射 |
|------|--------------|------------------|----------------|
| 基础通用 | 人名(PER)、机构(ORG)、时间(TIME)、百分比(PERCENT) | 基础溯源 | 责任主体、发生时间、损失比例 |
| 位置扩展 | 地名(LOC)、场所用途(PlaceUsage: 居民区/基础设施/道路/地铁)、区域特征(RegionalFeature: 城市/乡村/山地/低洼/海拔) | 台湾台风/洪涝高频，需细粒度时空定位 | 承灾体位置 + 易损性 |
| 灾害核心 | 致灾因子(HazardType: 台风/暴雨)、灾害事件(DisasterEvent)、影响(Impact: 淹没/道路中断) | 画像主干 | 直接对应框架字段 |
| 环境/承灾 | 孕灾环境(GeoEnvironment: 地形/植被)、承灾体(ElementAtRisk: 人口/建筑/交通) | 因果/部分关系基础 | 整体-部分 |
| 其他扩展 | 强度/规模(Strength: "high"/"局部")、措施(ResponseMeasure) | SWRL推理需要 | 蕴含/因果 |

**建议**：**强烈增加"场所用途 + 区域特征"**（这是台湾灾害高价值维度）

例如"中部低洼居民区"→ 自动推断"高风险承灾体 + 可能洪涝放大点"

工具：BERT-BiLSTM-CRF 或 LLM-fine-tune（用我们本体altLabel做few-shot）

## 3. 关系标注（Relation Annotation）

### 必须标注的关系类型

- 已标注：上位/下位（subClassOf）、整体/部分（hasPart/partOf）、蕴含（implies）、因果（causes）
- **必须新增**（提升推理能力）：时空关系（locatedIn、occursAt）、强度关系（hasStrength）、角色关系（hasCause/hasEffect in CausalRelation类）

### 标注方式

使用Prodigy/BRAT工具，对**句子级/文档级**进行实体+关系联合标注

标注格式：BIOES（实体）+ 关系箭头（e.g. 台风 → causes → 洪水）

目标：构建至少500-1000条高质量标注样本（含台湾灾害对话录音转写文本）

---

## 4. 结构化样本构建（训练数据准备，2-4周）

### 结构化样本

从对话/报告中抽取**RDF triples**作为金标准（gold standard）

示例：`<Typhoon2025, causes, Flood>` + 属性（strength="high", location="台中低洼区"）

### 半结构化文本完全结构化

常见来源：应急报告、新闻、微信/Line报灾文本

流程：
1. 规则+LLM初步抽取 → 2. 人工审核修正 → 3. SHACL验证 → 4. 转Turtle/RDF入KG

工具：Python + rdflib + pySHACL + LangChain

### 样本规模建议

初始1000条标注句子（覆盖台风/洪涝/地震场景）→ 用于fine-tune NER/RE模型 + Agent few-shot prompt

后续持续迭代（active learning）

---

## 5. 其他支撑工作

| 工作 | 说明 |
|------|------|
| **知识融合与去重** | 多源对话可能重复，用owl:sameAs + PROV-O追踪来源 |
| **质量控制** | 标注员间一致性（IAA > 0.85）；SHACL自动验证；专家抽查 |
| **与ASR pipeline对接** | 标注样本用于N-best rescoring（本体打分）；生成领域LM词典 |
| **评估指标** | 实体F1、关系F1、画像完整率（端到端） |

---

## 6. 实际落地收益与时间线

**准确率**：基础NER→灾害专用NER + 关系标注后，画像提取F1可从65%→90%以上

**速度**：预标注样本+SHACL验证让Agent秒级结构化对话

### 时间线建议

| 周数 | 内容 |
|------|------|
| 第1周 | 指南+本体最终版 |
| 第2-4周 | 标注2000条样本 + 半结构化转换pipeline |
| 第5周 | 模型训练 + ASR集成测试 |
