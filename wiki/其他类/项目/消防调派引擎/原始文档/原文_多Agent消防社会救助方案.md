# 多Agent架构在消防119+社会救助混合场景的设计方案

**来源**：针对中国大陆119消防接警 + 社会救助（电梯困人/动物救助/煤气泄漏/老人摔倒等）混合场景
**版本**：V1.0
**定位**：4-Agent并行架构，专为消防+社会救助混合场景设计

---

## 1. 背景与场景扩展

### 社会救助占比
- 一线城市消防119中社会救助占比达40-60%
- 包括：电梯困人、动物救助、高空坠物、煤气泄漏、老人摔倒无法自理、开门服务等

### 混合场景特点
| 特点 | 说明 |
|------|------|
| 问题仍为单条具体引导 | "位置在哪？""有没有人受困/需要帮助？""现场情况怎么样？" |
| 回答碎片化 + 无关内容多 | 情绪、重复、背景噪声 |
| 槽位需求更广、更细微 | 位置 + 受困/求助人数/状态 + 紧急类型/程度 |
| 决策复杂度更高 | 需快速区分"火灾救援" vs "社会救助派车/工具/人员" |

---

## 2. 多Agent vs 单小模型量化对比

| 维度 | 单小模型 | 多Agent并行（4个） | 提升幅度 | 备注 |
|------|----------|-------------------|----------|------|
| **槽位填充F1** | 88-94% | 93-97% | +4-7% | 社会救助类型多样，专用Agent更准 |
| **细微事实识别** | 良好 | 显著提升 | +8-12% | 每个Agent专注一类社会救助子场景 |
| **无关内容过滤** | 82-90% | 89-95% | +5-8% | Filter Agent专责情绪/重复 |
| **优先级/类型判断准确率** | 85-90% | 92-96% | +6-10% | 区分火灾 vs 社会救助，减少误派车 |
| **概率校准（置信度）** | 中等 | 优秀 | ECE下降10-15% | 多Agent投票 + 证据融合 |
| **端到端延迟** | <600ms | 750-950ms | +150-350ms | 消防接警可接受 |
| **资源占用** | 低 | 中（4模型并行） | ×2.5-3.5 | 边缘设备需云边协同 |

### 核心结论
在消防+社会救助场景下，多Agent的**ROI更高**，因为单一模型难以同时精通"火灾烟火"和"电梯/老人/动物"等多样社会救助细微事实。

**推荐采用Hybrid模式**：实时主路径用**单小模型**（延迟优先），低置信或复杂社会救助案例触发多Agent并行。

---

## 3. 推荐的多Agent架构（4个Agent）

### Agent 1: Position-Agent（位置专用小模型）
- **专责**：地址 + 区域特征（电梯内/高层/地下室/阳台等社会救助高频位置）
- **先验**：加载本体`hasLocation` + SHACL（必须有时空）

### Agent 2: Trapped-Help-Agent（受困/求助专用小模型）
- **专责**：人数/状态 + 求助类型（电梯困人/老人无法站立/动物上树等）
- **先验**：`hasTrappedPeople` + 社会救助SWRL规则（e.g. 电梯困人 → 优先级高）

### Agent 3: Emergency-Type-Agent（紧急类型专用小模型）
- **专责**：烟火/社会救助类型 + 程度（火灾浓烟 vs 煤气泄漏 vs 摔倒）
- **先验**：`hasFireSmokeStatus` + 扩展社会救助本体（煤气/电梯/动物等子类）

### Agent 4: Filter-Orchestrator-Agent（过滤+融合专用小模型）
- **专责**：无关内容过滤 + 一致性投票 + 最终优先级判断（火灾 vs 社会救助）
- **输出**：统一RDF triple + 置信度 + 派车建议

---

## 4. 并行协作流程（<900ms完成）

```
ASR转录 + 当前问题
       ↓
同时分发给Position/Trapped/Emergency 3个Agent（并行推理）
       ↓
Filter-Orchestrator收集结果 → 加权投票 + SHACL校验（冲突自动降权）
       ↓
最终输出：结构化画像
e.g. "位置：XX小区电梯内 | 受困：2人 | 类型：电梯困人（非火灾） | 优先级：高"
       ↓
低置信触发（Filter置信<0.75时）：
  → 自动推给人工
  → 或启动轻量Graph RAG补全
```

### 与先验知识注入结合
- 每个Agent加载**相同Few-shot Prompt + 本体片段**（或各自子RAG索引）
- 社会救助专用：额外注入"119社会救助分类表"（电梯/煤气/老人/动物等）

## 5. 6人标注团队适配建议

### 分工调整
- 4人主标注
- 1人专项社会救助校验
- 1人质控

### 标注指南微调
- 新增一列"社会救助子类型"（电梯困人/煤气/老人摔倒等）
- 每个Agent对应一列专项span标注（工作量增加约35%）

### Teacher阶段
- 先用大模型多Agent模式生成软标签 + CoT
- 6人团队只做校验，大幅降低人工负担

- 符合119接警规范（延迟<1s）
