# 中国大陆119/110/120接警场景下提升ASR识别率的智能支持系统构建方案

**来源**：接警场景技术方案（已完全适配"单条具体问题 + 单一意图 + 回答大概率直接对应但混杂无关内容"特性）

---

## 系统整体目标与新场景强适配价值

目标是构建**question-guided slot-filling智能支持系统**（Question-Guided Emergency Slot ASR System）

在接警员每次抛出**单一具体问题**（e.g. "位置在哪？"、"有没有人受困？"、"烟火情况怎么样？"）时

实时将呼救者回答转录并**精准填充到对应槽位**（RDF triples）

自动生成结构化灾害画像（位置+被困+烟火+因果/部分关系）

### 核心创新（针对场景）

| 创新点 | 说明 |
|--------|------|
| **强先验引导** | 每个问题意图唯一 → 系统提前知道期望槽位（location / trapped_people / fire_smoke_status），直接嵌入ASR（热词+上下文prompt）+ 提取器，实现"问题-答案"一对一精准匹配 |
| **无关内容自动过滤** | 回答中混杂的闲聊/情绪/重复 → 用KG+LLM相关性判断，只保留匹配当前槽位的片段 |
| **大陆119/110/120适配** | 参考真实智慧接处警系统（CTI中间件+ASR结构化提取），支持警笛/哭喊噪声、方言、实时流；输出直接对接GIS派警或预案 |

**ROI逻辑**：接警中每降低1% WER + 槽位填充准确率，可缩短响应10-30秒（国家智慧应急标准）。此类slot-filling系统在消防接警中ROI可达1:12+（更快结构化 = 更精准出警/救人）。前期投入低，后续几乎零边际成本。

---

## 系统高阶架构（专为"单问题-单槽位"设计）

```
1. 前端ASR层：FunASR Paraformer-streaming（大陆主流开源，实时流式）或iFLYTEK商用版 + 问题嵌入热词/上下文

2. 中端引导层：当前问题文本 → 生成槽位schema（从RDF/OWL本体映射）→ N-best重打分 + 相关性过滤

3. 后端KG驱动层：只对匹配片段提取RDF triples（SHACL验证 + SWRL因果传递），忽略无关内容

4. 部署层：实时流式（接警台/边缘设备 + 云GraphDB），Dashboard实时显示"槽位填充进度 + 画像"
```

---

## 三核心槽位定义

| 槽位 | 问题示例 | 提取内容 |
|------|----------|----------|
| **hasLocation** | "位置在哪？" | 地名 + 区域特征（低洼/高层/地下室） |
| **hasTrappedPeople** | "有没有人受困？" | 人数 + 状态（紧急/可等待） |
| **hasFireSmokeStatus** | "烟火情况怎么样？" | 浓度/类型（浓烟/明火/爆炸风险） |

## 开展工作的规划与步骤（5阶段）

按ROI优先级排序，先低投入诊断，再复用资产

### 阶段1：需求评估与基线建立（1-2周）

**投入**：
- 1-2名接警员/消防专家 + 1名AI工程师
- 10-20段真实"问题-回答"录音（含具体问题）
- 几乎0预算

**产出**：
- 当前FunASR/iFLYTEK在"位置/被困/烟火"问题下的WER基准 + 槽位填充F1（目标<75%）
- 《接警单问题槽位提取规格书》（问题模板 → 对应槽位）

**ROI理由**：投入最低，立即发现痛点，避免后续浪费。类似CTI接警AI项目显示，基线评估让后续优化效率提升3倍

---

### 阶段2：本体完善 + 领域数据准备（2-4周）

**投入**：
- 2-3名标注员 + 1名本体专家
- 收集/合成200小时大陆接警语音
- 云TTS ~5000元

**产出**：
- 本体增强版（Turtle：新增精确槽位类 hasLocation、hasTrappedPeople、hasFireSmokeStatus + 场所用途/区域特征）
- 500-1000条问题-回答对标注样本（问题文本 + 回答 + 槽位标签 + "无关片段"标记）
- 热词词典（"位置""被困""烟火""浓烟""火势" → FunASR热词）

**ROI理由**：数据+本体一次投入终身复用。WER额外下降18-28%，复用率95%+

---

### 阶段3：ASR领域适配与微调（3-5周）

**投入**：
- 1名ASR工程师
- 阶段2数据 + LoRA微调
- FunASR Paraformer-streaming作为base
- 云算力 ~2-3万元

**产出**：
- **问题引导式**领域ASR（每个问题实时嵌入对应槽位热词+prompt）
- 支持流式 + 初步无关过滤（WER目标<10%，槽位相关片段召回率>90%）

**ROI理由**：FunASR已针对大陆噪声优化，直接产出"位置/被困/烟火"识别率大幅提升。ROI > 9:1

---

### 阶段4：端到端管道集成与验证（2-3周）

**投入**：
- 2名工程师 + 1名测试员
- 100条真实接警测试对
- GraphDB + pySHACL ~1万元

**产出**：
- 完整pipeline：当前问题 → 引导ASR → 相关性过滤 → 本体填充 → 结构化画像
- 端到端槽位F1 > 92%，实时延迟<600ms
- Dashboard（接警员看到"位置：已填 | 被困：3人 | 烟火：中度浓烟"）

**ROI理由**：集成投入仅前期的30%，却实现"问题先验 + KG"协同（额外降错误25%+）

---

### 阶段5：部署、持续评估与迭代（持续）

**投入**：
- 0.5人运维
- 实时日志主动学习
- 云推理每月<5000元

**产出**：
- 生产上线 + A/B测试（WER/槽位准确率/响应时间）
- 自动迭代（新无关内容自动标注）

**ROI理由**：前期总投入15-25万元，后续年运维<5万元。整体ROI > 12:1
