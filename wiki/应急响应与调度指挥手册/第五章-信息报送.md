加强和规范信息报送工作是指挥中心核心业务，队伍各级要熟知信息报送的范围、时限和内容，以确保信息及时高效、全面准确、客观清晰。

## "1小时"信息报送规定

为贯彻落实中共中央办公厅、国务院办公厅有关信息报送工作最新要求，按照部、局领导批示指示制定。

### 报送范围（5类）

1. 造成人员死亡、被困、失联或有人员被送医院救治的火灾，以及造成或可能造成3人及以上死亡、被困或失联的其他警情
2. 涉及高层建筑、地下空间、大型商业综合体、人员密集场所、商场市场、化工企业、医院、养老院、学校、幼儿园、文物古建筑、政治核心区、地标性建筑、交通枢纽、涉外等敏感场所的警情
3. 涉及爆炸、坍塌、中毒、洪涝、地质、风雹灾害等易造成群死群伤、重大社会影响及损失的突发警情
4. 重要节假日、敏感时间节点、重大活动期间发生的有影响警情
5. 其他敏感或社会高度关注的特殊警情

### 报告原则

1. **首报要快**：有什么报什么、知道什么报什么，首次电话报告不强制要求核实核准伤亡、被困或失联人数，但务必保证敏感性、时效性
2. **知情必报**：明确有人员伤亡、被困或失联等情况必须第一时间报告
3. **信息要准**：涉及火灾的《值班信息快报》关键要素要与相关部门加强沟通核实
4. **正规途径报送**：通常通过电话、融合通信平台、《值班信息快报》报送，紧急情况可第一时间电话报告局指挥中心值班主任

### 报送程序

1. 各级接警单位在接警后，凡符合1小时报送范围的，**20分钟内**先向上级指挥中心电话报告
2. 接警后队伍到场时间较长的，到场前也要第一时间电话报告
3. 各总队指挥中心接警或接到下级报告后，**20分钟内**向局指挥中心电话报告，边报告边核实
4. 电话报告后**1小时内**，上报《值班信息快报》

### 容错免责

对第一时间电话首报内容出现差错的免于追究；对迟报、漏报、瞒报等行为严格责任追究。

### 警情统计

4个以上消防救援站到场并参与处置的警情需填报"警情统计"。30分钟内必须填报相关表格。局指挥中心每日7时和17时抽查检查。

## 灭火救援现场信息核报工作要求

### 核报岗位职责

- **主要责任人**：各支队主官
- **第一责任人**：当日现场最高指挥员
- **直接负责人**：当日信息助理
- **信息提供人**：参与现场处置的指战员和灭火救援文书

### 信息沟通渠道

依托消防融合通信平台，建立以信息助理和灭火救援文书为主体的总、支队两级突发事件信息搜报队伍，从接警出动到结束归建，坚持边处置、边核实、边报告。

### 关键环节时限

- **到场后5分钟内**上报初步侦察情况
- **15分钟内**电话报告有关情况（可能造成人员伤亡或涉及敏感场所的警情）
- **45分钟内**报送正式书面信息
- 全过程坚持"知情即报、敏感快报、边处边报"

### 人员搜救核实

综合运用"问、看、听、查"手段，发挥街道乡镇、派出所、居委会、单位负责人、保安等知情人作用。坚决贯彻"救人第一、科学施救、确保安全"的原则，划分片区、合理编组、逐层实施，充分利用热成像仪等装备。

涉及人员疏散营救的场所区域，设立人员疏散清点区，分类汇总登记疏散、救出人员和现场、送医、在外人员数量。对于外出出差、逃生避险、自救互救等在外人员，必须现场通话确认。

### 入院跟踪

防火责任支队指派防火干部全程跟踪掌握相关人员医治情况（自火灾事故发生之日起7日内），每日向灭火责任支队指挥中心通报。不允许出现因具体伤亡情况不掌握、不清楚、不了解，进而因形势误判导致事态扩大、舆情发酵。

### 留痕存档

充分运用信息化手段如实、及时、简要记录过程信息，坚持自然留痕、适当留痕，做到相关信息可查可核可追溯。

### 舆情管控

**舆情应对机制**：
- 实体化运行舆情处置工作专班，建立综合协调、舆情监控、信息发布和调查处置等工作小组
- 分阶段对舆情走势、传播情况、网民观点、媒体报道等内容综合分析

**舆情反馈机制**：
- 以舆情发生地所在支队为主，30分钟内反馈核实结果
- 会同区委网信办、公安网监部门同步开展舆情引导

**新闻发布机制**：
- "快说事实、滚动发布、释疑解惑"的舆情处置"组合拳"
- 对外发布内容严把政治政策、文字图片、编辑剪辑、发送范围"四道关口"
- 突发敏感事件信息发布坚决做到"五讲四不讲"
- 严禁发布职责和权限范围之外的信息

## 信息报送流程

接报警情信息 → 值班干部核实警情基本情况 → 值班主任判断是否达到上报要求 → 报告值班领导/带班领导 → 警情信息上报（初报→续报→终报）

**审核流程**：值班干部编辑 → 值班主任审核 → 值班领导签发 → 报告主官

由指挥中心值班人员通过系统或传真上报，报送后电话予以确认。资料归档存查。

## 警情信息报送模板

### 火灾信息首报（模板）

```
天津市XX区XX（地点或场所）发生火灾（首报）
X人死亡、X人受伤、X人被困

XX月XX日XX时XX分，天津市消防救援总队接到报警：XX区XX路（镇、乡）XX（建构筑物/设备等）发生火灾，XX支队立即调派X个消防救援站，XX名消防救援人员、XX辆消防车赶赴现场处置（XX支队全勤指挥部遂行出动）。

XX时XX分，XX消防救援站到达现场。经侦察，起火建筑为XX层XX结构的XX，建筑面积约XX平方米，起火部位位于XX位置（XX设备），主要燃烧物质为XX，火势XX。现场消防救援力量按照边控火、边救人原则，正在开展XX（处置行动）。

初步了解，火灾已造成X人死亡、X人受伤（重伤/轻伤）、还有X人被困。

详情续报。
```

```
天津市XX区XX（地点或场所）发生火灾（续报）
X人死亡、X人受伤、X人被困

XX月XX日XX时XX分，天津市XX区XX街道（乡/镇）XX（地点或场所）发生火灾。XX时XX分，现场消防救援力量在XX搜救出被困人员X人，X人经医务人员确认已无生命体征，X人送医院救治。XX时XX分火势已得到控制。经核实（初步了解），XX位置仍有X人被困。截至XX，火灾共造成X人死亡、X人受伤（重伤/轻伤）、X人被困。

XX时XX分，XX、XX消防救援站增援力量相继到达现场。
XX时XX分，XX（支/总）队全勤指挥部到达现场（包括政府、联动力量到场情况）。目前XX（简要描述现场情况），主要燃烧物质为XX（储量XX吨），估算过火面积约XX平方米。在现场指挥部统一指挥下，消防救援力量采取XX（简要描述所采取的技战术措施、力量部署、处置效果等）。

初步了解因XX引发火灾。

现场共有X个消防救援站，XX名消防救援人员、XX辆消防车参与处置，最高指挥员为XX，职务XX，电话XX。

详情续报。
```

### 火灾信息终报（模板）

```

XX月XX日XX时XX分，天津市消防救援总队接到报警：天津市XX区XX路（镇/乡）XX发生火灾，XX时XX分被扑灭，过火面积约XX平方米，造成X人死亡、X人受伤。

一、参战力量
接警后，XX支（总）队先后调派XX、XX消防救援站，XX名消防救援人员、XX辆消防车赶赴现场扑救。XX时XX分，XX消防救援站首战力量到场。XX时XX分，XX、XX消防救援站增援力量相继到达现场。XX时XX分，XX（支/总）队全勤指挥部到达现场。XX时XX分，市政府XX以及应急、公安、交通等部门到达现场，成立现场指挥部。

二、处置经过
消防救援力量在现场指挥部统一指挥下，采取XX处置措施。XX时XX分，在XX处搜救（营救/疏散）出被困人员X人，X人经医务人员确认无生命体征，X人送医院救治。XX时XX分火势得到控制，XX时XX分火灾被扑灭。XX时XX分，现场处置完毕，消防救援力量组织有序归建（留守XX名消防救援人员、XX辆消防车继续对现场进行监护）。

三、人员伤亡情况
火灾共造成X人死亡、X人受伤（重伤/轻伤）。

四、火灾基本信息
据了解，起火建筑为XX层XX（砖木/砖混/钢混等）结构的XX，建筑面积XX平方米，起火部位位于XX层XX位置，主要燃烧物质为XX，据初步调查，起火原因为XX。目前，具体火灾原因及财产损失正在进一步调查中。
```

### 特定场所火灾快报模板（12类）

手册包含以下快报模板，每类含领导批示指示模板和现场概况模板：

| 模板名称 | 领导指示要点 | 特色要素 |
|---------|-------------|---------|
| 多层建筑火灾快报 | 全力搜救被困人员，总结小火亡人教训 | 建筑层数/结构，起火部位 |
| 高层建筑火灾快报 | 科学高效处置，充分利用固定消防设施 | 建筑高度/使用性质/公寓 |
| 地下建筑火灾快报 | 调集排烟/照明/通信/供气装备 | 地上/地下层数，经营内容 |
| 大跨度建筑火灾快报 | 大型机械配合扑救，预防建筑坍塌 | 厂房/仓库，存储物质 |
| 石油化工火灾快报 | 专家技术支撑/专业工艺处置/安全防护 | 罐体类型/储量/邻近罐保护 |
| 危化品仓库火灾快报 | 掌握物料理化性质，合理选用灭火药剂 | 建筑结构/存储物质 |
| 商业综合体火灾快报 | 充分利用固定消防设施侦查/控火/排烟 | 业态组成/营业状态 |
| 医疗场所火灾快报 | 落实防护防疫措施，配合医护人员 | 起火部位/楼层 |
| 露天堆垛火灾快报 | 严防飞火蔓延扩大，调集大型机械 | 堆垛占地/周边设施 |
| 交通工具火灾快报 | 联动交通/铁路/民航/海事 | 车辆/列车/船舶/飞机载客情况 |
| 储能电站火灾快报 | 注意防范爆燃爆炸，保持安全距离 | 容量/接入电压/电池类型 |
| 保温材料使用场所火灾快报 | 严禁盲目内攻或冒险作业，举高车外攻 | 冷库/保温材料 |
