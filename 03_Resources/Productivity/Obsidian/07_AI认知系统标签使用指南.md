## 标签在系统中的角色

| 角色 | 说明 |
|------|------|
| **状态跟踪器** | 随时知道笔记处理阶段（#state/inbox → #state/published） |
| **检索过滤器** | Dataview/Graph View 快速定位特定类型/来源/主题 |
| **AI自动化触发器** | AI Prompt 根据标签自动决定处理逻辑 |
| **视角导向工具** | 从"如何再次找到它"角度组织知识 |
| **输出推动器** | #out/ 标签驱动内容生产 |

---

## 各层标签规则

| 层级 | 必加标签 | 可选 |
|------|----------|------|
| **Capture** | #type/capture #state/inbox #source/xxx | #topic/xxx |
| **Aggregate** | AI 处理后：#state/processing + 调整 type/topic | #cog/xxx #value/xxx |
| **Organize** | #type/permanent #state/review + 双链 | - |
| **Output** | #type/output #out/xxx（x-post/article/thread） | #state/published |

---

## 日常操作闭环

```
1. 捕捉 → 新笔记自动打 #type/capture #state/inbox
2. AI处理 → AI Tagger 批量处理 inbox → 生成日记 + 新标签
3. 组织 → review #state/review，添加双链，建立 MOC
4. 输出 → 筛选 #out/ 笔记，AI 生成内容，更新状态
5. 维护 → 每周清理滞留笔记，每月 review 标签词表
```

## 核心原则

- 每篇笔记标签 **3-6 个**
- 小写英文 + 嵌套（#type/permanent）
- 顶层维度（四层）保持稳定
- 具体想法用双链，结构化数据用 Frontmatter，标签管视角与状态
