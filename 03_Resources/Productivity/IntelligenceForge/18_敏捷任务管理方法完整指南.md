# 敏捷任务管理方法完整指南

**标签**：#敏捷 #任务管理 #Scrum #Kanban #WIP #UserStory
**位置**：03_SystemDesign/

---

## 核心原则（Agile Manifesto）

| 原则 | 说明 |
|------|------|
| 个体与互动 | 高于流程和工具 |
| 可工作的产出 | 高于详尽文档 |
| 客户协作 | 高于合同谈判 |
| 响应变化 | 高于遵循计划 |

---

## 主流方法对比

| 方法 | 核心特点 | 适合场景 |
|------|----------|----------|
| **Scrum** | 固定 Sprint（1-4周） | 需明确交付周期 |
| **Kanban** | 持续流动 + 可视化看板 | 运维/内容创作/持续性工作 |
| **Scrumban** | Scrum + Kanban 混合 | 过渡期/混合需求 |
| **XP** | TDD、结对编程 | 代码质量要求高 |
| **Lean** | 消除浪费、价值流映射 | 流程优化/精益创业 |

---

## 用户故事格式

**As a [用户角色]，I want [目标] so that [价值/理由]**

---

## 验收标准（Definition of Done）

- 常用格式：Given-When-Then（Gherkin）
- 3-8 条、可测试、可测量
- 避免模糊词（"友好""快速"）

---

## 任务分解与编排

| 技术 | 说明 |
|------|------|
| **WBS** | 层级分解 |
| **逆向拆解** | 最终 Deliverable → 工作流节点 → 子任务 |
| **依赖关系** | 串行 / 并行 / 里程碑 |
| **估算** | Story Points / 理想小时 |

---

## 可视化与跟踪

| 工具 | 说明 |
|------|------|
| **Kanban Board** | To Do → In Progress → Review → Done + WIP Limits |
| **Burn-down Chart** | 跟踪剩余工作 |
| **Mermaid Gantt** | 自动生成时间线和依赖图 |

---

## 日常/周期实践

| 实践 | 频率 | 说明 |
|------|------|------|
| Daily Stand-up | 每天 15 分钟 | 昨天/今天/阻碍 |
| Sprint Planning | 每个 Sprint | 计划制定 |
| Retrospective | 每周/每 Sprint | Start-Stop-Continue 改进 |

---

## 常见坑

| 错误 | 正确做法 |
|------|----------|
| 用户故事太大 | 拆到 1-2 天可完成 |
| Acceptance Criteria 模糊 | 必须可验证 |
| WIP 超限 | 设置严格限制 |
| 缺乏回顾 | 每周固定 Retrospective |
