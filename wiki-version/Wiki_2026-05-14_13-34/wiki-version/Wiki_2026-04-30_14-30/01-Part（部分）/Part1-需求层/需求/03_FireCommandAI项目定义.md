# FireCommand AI 项目定义

**标签**：#项目定义 #FireCommand #消防指挥 #AI系统
**位置**：01_Requirements/

---

## 项目名称

**FireCommand AI**（消防指挥中枢 / Dispatch Orchestrator）

---

## 核心定位

实时 AI 辅助指挥系统，专注于「接警后的指挥调度」与「现场到指挥中心的反馈闭环」

| 要素 | 说明 |
|------|------|
| 任务来源 | 110/119 报警电话、APP 报警、传感器、监控系统、街道巡逻 |
| 目标 | 快速做出正确决策，形成完整可追溯指挥记录与事后分析 |

---

## 核心模块

| 模块 | 功能 |
|------|------|
| **指挥模块（Orchestrator）** | 报警信息解析 → AI 生成指挥方案 → 人机确认 → 资源调度/路线规划/风险评估 |
| **反馈模块（Feedback Loop）** | 现场实时状态上传 → AI 动态调整 → 追加指令 → 事后报告 → 案例库学习 |

---

## 指挥输出结构

- 事件概况
- 立即行动指令（派遣车辆/人员）
- 时间轴预估
- 风险与备案
- 指挥员确认按钮（人机协作）

---

## 技术栈

| 层级 | 技术 | 理由 |
|------|------|------|
| 前端 | Next.js 15 + TypeScript + shadcn/ui + Leaflet | 实时地图 + 指挥仪表盘 |
| 后端 | Next.js API Routes / Supabase / Firebase | WebSocket 实时通讯 |
| AI 指挥 | Groq / Claude 3.5 + LangGraph.js | 多代理协作 + 状态持久 |
| 实时反馈 | WebSocket + Supabase Realtime | 双向即时更新 |
| 数据存储 | PostgreSQL + Vector DB | 事件记录 + 案例相似度检索 |
| 安全审计 | AES-256 + 完整操作日志 + 人工确认 | 消防合规要求 |

---

## 开发阶段

| 阶段 | 内容 | 时长 |
|------|------|------|
| Phase 0 | 准备：接警来源确认、SOP 表单设计、事件数据模型 | 1 周 |
| Phase 1 | 指挥核心：任务接入 + Orchestrator + 人机确认界面 | 2 周 |
| Phase 2 | 反馈闭环：WebSocket + 动态调整 + 自动报告生成 | 2 周 |
| Phase 3 | 记忆与学习 + 上线：案例库 + 部署 + 审计日志 | 1-2 周 |

**总时长**：全职 6-8 周，业余 10-12 周

---

## 关键特性

- 结构化 SOP 输入（接处警专用）
- 多代理 Orchestrator 指挥
- 实时反馈回路
- 持久记忆（历史案例库 + 指挥经验学习）
- 高可靠性、安全审计
