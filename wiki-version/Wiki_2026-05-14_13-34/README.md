# TW--WIKi 消防调派智能系统知识库

*用 LLM 构建持续积累的个人知识库*

---

## 项目概述

TW--WIKi 是**消防调派智能系统**的个人知识库，采用 LLM Wiki 方法论构建。知识随每次导入和查询不断积累，形成复利效应。

> 类比：Obsidian 是 IDE，LLM 是程序员，Wiki 是代码库。

---

```
TW--WIKi/
├── wiki/                    # ⭐ LLM Wiki 主目录（Obsidian Vault）

│   ├── 02-Step（流程）/     # 12步核心流程

│   ├── _raw/              # 原始草稿（待导入）
│   └── ...                 # 其他目录结构
│
├── wiki-version/           # 历史版本快照
└── wiki模板/              # 模板备份
```

## 核心内容

### Part 系统架构

| Part | 内容 | 说明 |
|------|------|------|
| **Part 1** | 需求与场景层 | 报警人求助模式、场景画像、六聚类体系、问题路由 |
| **Part 2** | 系统设计与数据模型 | DIKW框架、数据结构设计 |
| **Part 3** | 核心调度引擎 | Query_Routing、4-Agent多智能体 |
| **Part 4** | 审计与责任 | 责任机制、人工确认、审计体系 |

### Part 1 七大核心模块

```
场景集 → 场景画像库 → 报警人求助模式库 → 报警人类型模式 → 关键缺失要素库 → 问题路由规则 → 结构化输出
```

| 模块 | 定义 | 回答问题 |
|------|------|----------|
| **场景集** | 定义"有哪些情况" | 当前报警属于哪类警情？ |
| **场景画像库** | 定义"应该知道什么" | 这个场景需要获取哪些字段？ |
| **求助模式库** | 定义"报警人通常怎么说" | 报警人说的这些话对应哪些字段？ |
| **报警人类型模式** | 定义"不同人怎么说、会漏什么" | 这类报警人可信度如何？容易漏什么？ |
| **关键缺失要素库** | 计算"说了什么和应该知道什么的差值" | 还缺哪些关键字段？ |
| **问题路由规则** | 决定"下一步问什么" | 缺失字段按什么优先级问？ |
| **结构化输出** | 把结果交给后续模块 | 输出什么给后续定级/调派/处置？ |

### 六场景画像聚类体系

| 聚类 | 名称 | 问题集继承 | 说明 |
|------|------|------------|------|
| 聚类1 | 水域灾害类 | L0 + 水灾专项 | 暴雨内涝/河道涨水/地下室进水 |
| 聚类2 | 普通火灾类 | L0 + L1 | 住宅/商铺/普通建筑（默认主场景） |
| 聚类3 | 复杂空间与高危结构类 | L0 + L1 + 高危结构专项 | 高层/地下/厂房/化工 |
| 聚类4 | 人员密集类 | L0 + L1 + 人员密集专项 | 学校/医院/商场/车站 |
| 聚类5 | 重点单位类 | L0 + L1 + 重点单位专项 | 政府机关/机房/危化品单位 |
| 聚类6 | 极端紧急类 | L0 最小必要信息集 | 报警人慌乱/通话极短 |

### 12步核心流程

1. 接警接收与初步研判
2. 智能问询与信息补全
3. 多维画像构建
4. 警情等级智能判定
5. 调派规模智能计算
6. 资源匹配与任务分配
7. 约束校验与风险评估
8. 反向模拟与方案预演
9. AI多智能体协同指挥
10. 人工指挥确认与授权
11. 指挥指令下达与协同
12. 全程审计与优化反馈

## LLM Wiki 使用方法

### 导入资料
将文章或文档放入 `llm-wiki-base/_raw/` 目录，告诉 Claude：
```
请导入 _raw/文件名.md 到知识库
```

### 查询知识
直接向 Claude 提问：
```
根据知识库，对比 X 和 Y 的区别
知识库里关于 XX 有什么要点？
```

### 维护健康
```
请检查知识库的健康状态
```
