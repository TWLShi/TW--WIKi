# 119消防接警机器人完整方案总结

**来源**：火灾救援+开锁/马蜂窝等社会救助场景完整方案
**版本**：V1.0

---

## 1. 总体目标

构建**实用型119接警机器人**：
- 自动化率 ≥ 90%
- 端到端延迟 < 800ms
- 槽位F1 ≥ 93%
- WER ≤ 15%（重点实体/关键词准确率显著更高）

---

## 2. 核心架构

### 画像层（白盒骨架）
- RDF/OWL本体 + SHACL约束 + SWRL规则
- 同义词skos:altLabel，反义词对比规则
- 覆盖火灾vs社会救助区分

### 模型层（混合可解释方案）
- 大模型Teacher先行：软标签+CoT+新规则
- 小模型Student：轻量蒸馏（实时主路径）
- **推荐最终形态**：规则70-80% + 轻量可解释模型20-30%

### 多Agent设计（可选）
- Position-Agent、Help-Agent、Type-Agent、Filter-Orchestrator
- 低置信时触发并行

---

## 3. 画像驱动ASR提升闭环（核心机制）

### 动态热词注入（基于问题的热词）
- 利用当前问题作为强先验
- 两阶段：Pre-Retrieval筛选Top-K + Contextual Biasing
- 与画像联动形成正反馈

### 其他ASR增强
- N-best/Lattice Rescoring
- 后处理纠错
- 闭环迭代

---

## 4. 工程落地要点

| 要点 | 说明 |
|------|------|
| 6人团队 | 审核规则+同义词+紧急程度（大模型预生成，人工轻量校验） |
| 机器人交互 | 单问题引导+动态安抚+画像确认+规范建议 |
| 部署模式 | 云边混合 |
| 迭代机制 | 大模型持续提炼规则→人工审核→更新系统 |

---

## 5. 最终决策推荐

**"规则为主+轻量可解释模型为辅+基于问题的动态热词注入"混合方案**

### 为什么推荐
- 可解释性高、实时性好、准确率实用
- 动态热词有效解决热词量大和口语变体多问题
- 符合消防"减副"、规范对齐和持续优化需求

### 完整闭环
```
问题引导 → 基于问题的动态热词 → 画像构建 → ASR提升 → 更好画像
```
