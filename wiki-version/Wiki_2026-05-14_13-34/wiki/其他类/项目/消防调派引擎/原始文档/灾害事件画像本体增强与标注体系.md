## 2. 实体识别维度（8-12类）

| 维度 | 具体类型 | 说明 | 与框架映射 |
|------|----------|------|------------|
| 基础通用 | 人名(PER)、机构(ORG)、时间(TIME)、百分比(PERCENT) | 基础溯源 | 责任主体、时间、损失比例 |
| 位置扩展 | 地名(LOC)、场所用途(PlaceUsage)、区域特征(RegionalFeature) | 高频需求 | 承灾体位置+易损性 |
| 灾害核心 | 致灾因子(HazardType)、灾害事件(DisasterEvent)、影响(Impact) | 画像主干 | 直接对应框架 |
| 环境/承灾 | 孕灾环境(GeoEnvironment)、承灾体(ElementAtRisk) | 因果/部分基础 | 整体-部分 |
| 其他扩展 | 强度(Strength)、措施(ResponseMeasure) | SWRL推理需要 | 蕴含/因果 |

### 推荐扩展维度
- **场所用途**：居民区/基础设施/道路/地铁
- **区域特征**：城市/乡村/山地/低洼/海拔
- 示例："中部低洼居民区" → 自动推断高风险承灾体+可能洪涝放大点

## 3. 关系标注类型

### 必须标注的关系

| 关系类型 | 说明 |
|----------|------|
| 上位/下位 | subClassOf |
| 整体/部分 | hasPart / partOf |
| 蕴含 | implies |
| 因果 | causes |
| 时空关系 | locatedIn、occursAt |
| 强度关系 | hasStrength |
| 角色关系 | hasCause / hasEffect |

### 标注方式
- 工具：Prodigy / BRAT
- 格式：BIOES（实体）+ 关系箭头
- 规模：500-1000条高质量标注样本

## 4. 结构化样本构建

### 流程
```
规则+LLM初步抽取 → 人工审核修正 → SHACL验证 → 转Turtle/RDF入KG
```

### 工具
- Python + rdflib + pySHACL + LangChain

### 样本规模
- 初始1000条标注句子（覆盖台风/洪涝/地震场景）
- 用于fine-tune NER/RE模型 + Agent few-shot prompt
