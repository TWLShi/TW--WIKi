# 119消防接警系统中同义词与反义词语义体现

**来源**：针对开锁、马蜂窝等社会救助场景
**版本**：V1.0

---

## 1. 同义词（Synonym）在语义中的作用与体现

### 开锁场景同义词/口语变体
统一映射到 `rescueType="OpenLock"`：
- 核心概念：被锁住、无法进入、钥匙问题
- 同义表达：
  - "锁在门外了""被锁外面""进不去家门""门打不开了"
  - "钥匙忘在屋里了""钥匙掉里面了""没带钥匙"
  - "车门锁住了""困在车里了"
  - 紧急变体："孩子在里面哭""老人出不来""孕妇被锁"

### 马蜂窝场景同义词/口语变体
映射到 `rescueType="WaspNest"`：
- 核心概念：马蜂窝、蜂群威胁、摘除需求
- 同义表达：
  - "马蜂窝""蜂窝""一大窝马蜂""蜂巢"
  - "马蜂飞得厉害""好多马蜂""被马蜂围了"
  - "阳台上有蜂窝""屋檐下有蜂""树上有马蜂窝"
  - 风险变体："孩子不敢出去""怕叮到小孩""有叮咬风险"

### 本体层实现（skos:altLabel）
```turtle
:OpenLock rdf:type :SocialRescueType ;
          rdfs:label "开锁"@zh ;
          skos:altLabel "锁在门外", "进不去家", "钥匙忘在里面", "车门锁住了", "被锁外面" .

:WaspNest rdf:type :SocialRescueType ;
          rdfs:label "马蜂窝"@zh ;
          skos:altLabel "蜂窝", "马蜂飞得厉害", "一大窝马蜂", "摘蜂", "蜂巢" .
```

### 大模型→蒸馏全流程落地
- **Teacher阶段**：Few-shot Prompt嵌入同义词示例，CoT明确"此表达等同于开锁语义"
- **蒸馏到小模型**：软标签包含同义词概率分布，Student学习后可直接识别变体
- **收益**：大幅降低WER影响，提升槽位召回率

## 2. 反义词（Antonym）在语义中的作用与体现

### 开锁反义词（判断自助可解决 vs 紧急派警）
| 正向（紧急） | 反义（非紧急/可自助） |
|--------------|----------------------|
| 孩子在里面哭 | 没人里面 |
| 老人出不来 | 有备用钥匙 |
| 孕妇被锁 | 联系上物业了 |
| 医疗紧急 | 不急 |

### 马蜂窝反义词（风险评估）
| 正向（高风险） | 反义（低风险/可观察） |
|----------------|----------------------|
| 孩子不敢出去 | 没人被影响 |
| 有叮咬风险 | 只是看着吓人 |
| 影响出行 | 没叮到人 |
| 规模大 | 白天飞得少 |

### SWRL规则实现
```swrl
OpenLock(?e) ^ hasPeopleInside(?e, true) ^ hasChildOrElder(?e, true) → urgencyLevel(?e, "high")
WaspNest(?e) ^ hasRiskToPeople(?e, false) → suggestedAction(?e, "可观察或找专业公司")
```

### 语义决策
- 反义词触发**分流建议**
- Filter-Orchestrator检测反义信号，调整置信度和处置建议
- 蒸馏时加入对比损失（正向/反义样本对）

## 3. 完整语义体现示例（开锁）
```
自然语言："我被锁在家门外，孩子在里面哭，好着急…"
  ↓
同义映射：开锁 + 紧急（孩子）
  ↓
反义判断：非"有备用钥匙" → 高优先级
  ↓
输出：rescueType=OpenLock, urgency=high, suggested=必要时派警
```

- 反义词提升决策准确率（规范对齐）
