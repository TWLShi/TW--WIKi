# 价值点提取——多Agent消防社会救助方案

**来源**：消防119+社会救助混合场景多Agent架构设计
**版本**：V1.0

---

## 核心价值点

### 1. 多Agent vs 单模型量化提升
| 维度 | 单模型 | 多Agent(4) | 提升 |
|------|--------|------------|------|
| 槽位F1 | 88-94% | 93-97% | +4-7% |
| 细微事实识别 | 良好 | 显著提升 | +8-12% |
| 无关过滤 | 82-90% | 89-95% | +5-8% |
| 优先级判断 | 85-90% | 92-96% | +6-10% |
| ECE校准 | 中等 | 优秀 | -10-15% |

### 2. 4-Agent架构
| Agent | 专责 | 先验 |
|-------|------|------|
| Position-Agent | 地址+区域特征（电梯内/高层/地下室） | hasLocation + SHACL |
| Trapped-Help-Agent | 人数/状态+求助类型（电梯困人/老人/动物） | hasTrappedPeople + SWRL |
| Emergency-Type-Agent | 烟火/社会救助类型+程度 | hasFireSmokeStatus + 社会救助本体 |
| Filter-Orchestrator-Agent | 无关过滤+投票+优先级判断 | RDF triple + 置信度 |

### 3. 协作流程
- 3个Agent并行推理 → Filter收集结果 → 加权投票+SHACL校验 → 最终画像
- 低置信(<0.75)触发人工或Graph RAG

### 4. 部署策略
- **边缘设备**：单模型（延迟优先）
- **云端**：多Agent并行（低置信案例）
- 延迟<1s符合119规范

### 5. 落地路线（4周）
| Week | 内容 |
|------|------|
| 1 | 更新标注指南+大模型4-Agent模式生成软标签 |
| 2 | 6人团队标注（火灾+社会救助各50%） |
| 3 | LoRA蒸馏4个小模型+Filter融合代码 |
| 4 | A/B测试验证 |

### 6. ROI
- 响应时间缩短：25-35%
- 误派车率下降：15%+
- 社会救助处理能力提升：40%+
