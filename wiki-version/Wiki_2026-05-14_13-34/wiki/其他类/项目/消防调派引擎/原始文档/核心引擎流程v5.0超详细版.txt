# 消防调派智能系统核心引擎 - 超详细流程 v5.0

## 1. 警情接收与初步解析（0-30秒）
- 1.1 系统自动接听119来电并开始录音
- 1.2 实时语音转文字（支持方言、噪音过滤）
- 1.3 实体抽取（地址、建筑名、报警人联系方式、时间）
- 1.4 初步关键词标签（明火、浓烟、爆炸、人员被困等）
- 1.5 GPS/基站辅助定位（若报警人开启）
- 1.6 生成初始警情卡片 + 分配唯一Incident ID
- 输入：原始语音
- 输出：结构化警情对象 + 录音文件 + 初步标签

## 2. Query_Routing 智能问询路由（30-120秒）
- 2.1 缺失要素智能检测（地址精度、火情规模、人员情况等）
- 2.2 优先级排序待问问题（最重要 → 次重要）
- 2.3 动态生成或匹配标准话术
- 2.4 通过语音助手或调度员发起问询
- 2.5 实时解析回答并更新警情字段
- 2.6 判断是否需要继续追问（路由树分支）
- 2.7 若超时或拒绝回答则标记"信息不完整"
- 输出：补全率 ≥ 85% 的警情画像

## 3. 多维要素识别与画像构建
- 3.1 空间维度：精确地址 → 建筑类型 → 楼层/地下 → 周边水源/道路
- 3.2 时间维度：起火时间估算 → 报警延迟 → 当前蔓延时间
- 3.3 规模维度：着火面积估算 → 烟雾浓度 → 明火/阴燃判断
- 3.4 风险维度：人员密集度 → 危化品标识 → 特殊人群（老人/儿童/病人）→ 重要设施
- 3.5 自动匹配历史相似案例库
- 3.6 生成完整多维画像标签 + 置信度评分
- 输出：警情画像向量 + 标签集合

## 4. 警情等级自动判定
- 4.1 加载等级映射规则表（规则引擎）
- 4.2 多因子打分（基础分 + 各维度加权）
- 4.3 输出建议等级 + 置信度（例：三级 87%）
- 4.4 特殊场景标签自动标记（城中村、高层医院、化工园区等）
- 4.5 若置信度 < 70% 则标记"需人工复核"
- 输出：最终等级 + 解释性理由

## 5. 调派规模计算与力量编成
- 5.1 调用规模计算核心模型
- 5.2 输入：等级 + 建筑类型 + 风险系数 + 季节/时间权重
- 5.3 计算基础车辆数 + 特殊装备需求
- 5.4 生成详细编成清单（水罐×N、泡沫×N、举高、特勤、医疗等）
- 5.5 考虑全区资源平衡（避免单中队超负荷）
- 输出：推荐编成方案（含最小/推荐/最大三档）

## 6. 队站与车辆精准匹配
- 6.1 GIS实时计算各中队距离与ETA（考虑路况）
- 6.2 查询车辆实时状态（在勤、空闲、维修、保养）
- 6.3 匹配度打分（距离 + 装备 + 专业能力）
- 6.4 自动生成首选/备选方案
- 6.5 若需跨区调派则触发联动协议
- 输出：具体中队 + 车辆推荐列表 + ETA预估

## 7. 多维度约束校验（核心风控环节）
- 7.1 资源可用性实时校验
- 7.2 时间窗约束（必须X分钟内到达）
- 7.3 特殊环境约束库（城中村窄巷、断头路、高层接力供水、化工防护等级）
- 7.4 规模合法性与政策约束
- 7.5 整体可行性评分（0-100分）
- 7.6 生成详细校验报告（通过项 / 问题项 / 建议）
- 输出：校验结果（Pass / Warning / Fail）

## 8. 反向模拟与冲突解决
- 8.1 方案推演（到达时间、覆盖能力、火势预测）
- 8.2 冲突检测（中队超负荷、装备冲突、时间冲突）
- 8.3 自动生成调整方案（增减车辆、替换中队、变更编成）
- 8.4 若无法自动解决则推送至人工确认
- 8.5 记录所有模拟版本
- 输出：最终可执行方案 或 回退信号

## 9. AI多智能体协同决策
- 9.1 精准匹配11类火灾子场景画像
- 9.2 DIKW深度推理链
- 9.3 多Agent协同工作（感知Agent → 决策Agent → 执行Agent → 预测Agent）
- 9.4 生成结构化指挥方案（力量分工、处置路径、战术要点、注意事项）
- 9.5 火势蔓延预测 + 资源增援需求预测
- 9.6 输出解释性文本（Why推荐此方案）
- 输出：智能辅助指挥方案（带置信度）

## 10. 人工二次确认节点（责任关口）
- 10.1 调度员查看AI完整方案 + 解释
- 10.2 逐项确认关键点（等级、车辆数、特殊措施）
- 10.3 可小幅修改（必须填写修改理由）
- 10.4 最终电子签字确认
- 10.5 系统锁定方案（后续修改需更高权限）
- 输出：带人工签名的最终调派方案

## 11. 调派方案生成与多端下达
- 11.1 格式化生成指挥单（标准模板）
- 11.2 自动规划最佳路径 + 备用路线
- 11.3 推送渠道：车载终端、消防App、对讲机、短信
- 11.4 生成现场处置要点语音/文字提示
- 11.5 启动出警计时与跟踪
- 输出：正式下达的调派指令

## 12. 全流程审计留痕与责任记录
- 12.1 记录每个节点的输入、AI建议、人工决策、修改痕迹
- 12.2 区块链式不可篡改日志
- 12.3 责任主体自动映射（接警员、调度员、AI贡献度）
- 12.4 生成审计报告（时间线 + 决策路径 + 问题点）
- 12.5 数据脱敏后进入优化训练集
- 输出：完整审计档案

## 决策循环（细化）
- 7步或8步 Fail/Warning → 自动返回2步（补问）或5步（重算）
- 10步修改幅度大 → 强制返回5步重新计算 + 重新校验
