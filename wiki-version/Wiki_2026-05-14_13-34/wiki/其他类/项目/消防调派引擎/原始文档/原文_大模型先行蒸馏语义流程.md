# 大模型先行→蒸馏小模型全流程中的语义体现

**来源**：119消防接警+社会救助场景（开锁/马蜂窝）
**版本**：V1.0

---

## 1. 四步闭环语义体现机制

| 阶段 | 语义体现方式 | 输出 | 对开锁/马蜂窝的体现 |
|------|--------------|------|---------------------|
| **1. Teacher大模型先行** | Few-shot + 本体 + CoT深层解析 | CoT解释 + 软标签 + RDF triples | 识别"孩子哭+锁门外"→ 紧急度high + 建议联系物业 |
| **2. 提炼规则/数据** | 从Teacher输出提取规则 + 生成合成数据 | JSONL + 软标签数据集 | 提炼"开锁+老人"→ 高优先级SWRL规则 |
| **3. 蒸馏转移** | 损失函数 + 规则正则化 | Student继承软标签 + 本体约束 | 语义从Teacher深层理解转移到Student快速映射 |
| **4. Student小模型运行** | 实时推理时继承语义 | 结构化槽位 + 置信度 + 处置建议 | 实时输出分流建议 |

---

## 2. 阶段1：大模型Teacher先行（语义捕捉与grounding）

### 开锁示例
**输入**：呼救："我被锁在家门外了，钥匙忘在里面，孩子在屋里哭，好着急啊…"

**Teacher输出**：
```json
{
  "cot": "①位置语义='家门外'；②求助类型='OpenLock'；③紧急程度='high'（孩子哭→生命风险）；④无关='好着急啊'（过滤）；⑤规范对齐=优先建议联系物业",
  "slots": {
    "location": {"value": "家门外", "probability": 0.96},
    "rescueType": {"value": "OpenLock", "probability": 0.94},
    "urgencyLevel": {"value": "high", "probability": 0.88}
  },
  "suggestedAction": "优先联系物业/备用钥匙，必要时派开锁工具",
  "rdf": ":CallXXX :hasRescueType :OpenLock ; :urgencyLevel \"high\" ."
}
```

### 马蜂窝示例
**输入**：呼救："阳台上有马蜂窝，好多马蜂飞，孩子不敢出去，能不能晚上来处理？"

**Teacher输出**：
- CoT：①位置="阳台"（高空风险）；②类型="WaspNest"；③风险="high"（孩子影响）；④时间偏好="夜间"（规范对齐）
- riskLevel="high" + suggestedTime="夜间作业优先"

### Teacher核心Prompt模板
```text
你是119消防接警Teacher模型，必须严格遵守本体和消防规范。

先验知识（必须遵守）：
1. 本体定义：rescueType包括OpenLock、WaspNest等；urgencyLevel分high/medium/low
2. 消防规范：开锁优先分流物业；马蜂窝评估风险+夜间优先
3. 标注指南：最小span原则、过滤无关情绪

当前问题：{question}
呼救转录：{asr_text}

请一步步CoT思考：
1. 提取位置语义
2. 提取求助类型+细微事实（开锁/马蜂窝/紧急程度）
3. 过滤无关内容
4. 对齐消防规范，给出处置建议
5. 输出JSON + RDF triples + 软标签概率
```

## 3. 阶段2：提炼规则/数据（语义提炼）
- **规则提取**：从CoT总结SWRL规则（如"OpenLock + 儿童 → urgency high"）
- **合成数据**：生成10000+条变体（不同噪声、口语、紧急度）
- **语义载体**：高质量JSONL数据集（带软标签），供6人团队校验

---

## 4. 阶段3：蒸馏转移（语义转移）
- **损失函数**：Student同时学习
  - 硬标签（6人标注的金标准）
  - Teacher软标签（KL散度损失）
  - 规则正则化损失（违反SHACL/SWRL则惩罚）
- **可选多Agent**：每个Slot-Agent单独蒸馏

---

## 5. 阶段4：Student小模型运行（语义继承）
- 直接输出结构化槽位 + 置信度（继承Teacher的概率校准）
- 低置信时触发轻量Graph RAG补全语义
- 机器人端动态播报："已记录您被锁在家门外，孩子在屋内，建议先联系物业，如需我们立即派开锁工具，确认吗？"

---

## 6. 6人标注团队的作用
- 只需**校验**Teacher生成的软标签/CoT（而非从零标注）
- 重点检查"紧急程度""分流建议"等规范语义是否准确
- 每周IAA计算语义一致性（目标>0.85）
