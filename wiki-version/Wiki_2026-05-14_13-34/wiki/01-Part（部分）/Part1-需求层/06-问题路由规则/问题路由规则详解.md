# 问题路由规则详解

问题路由不是固定问卷，而是由"场景画像差值"和"报警人描述模式"共同驱动的动态追问机制。

---

## 三层问题结构

基于数据结构定义.json，问题路由分为L0、L1、L2三层：

### L0层（所有来话必问）

| 节点 | 问题 | 提取字段 |
|------|------|----------|
| l0.root | "您好，这里是119消防指挥中心！请问您遇到什么紧急情况？在什么位置？" | incidentType, sceneType, district, community, detailedAddress, street, landmark, floor, roomNumber |
| l0.pos | "能再说一下具体位置吗？具体到几栋几单元几楼几号？" | district, community, detailedAddress, street, landmark |
| l0.risk | "是否有人被困？有多少人？" | trappedCount, trappedLocation, hasInjury, severity |

### L1层（按警情类型标准问题）

#### 火灾场景L1问题

| 节点 | 问题 | 提取字段 |
|------|------|----------|
| l1.fire.1 | "烟是什么颜色？浓不浓？" | smokeColor, smokeOdor |
| l1.fire.2 | "火是刚着起来，正在蔓延，还是已经烧得很大？" | fireStage, fireScale |
| l1.fire.3 | "起火位置大概在哪里？" | trappedLocation, burningMaterial, hasDangerousItem |
| l1.fire.4 | "这里是什么类型的建筑？" | buildingType |
| l1.fire.5 | "一共有多少层？起火在第几层？" | totalFloors, fireFloor, hasBasement |
| l1.fire.6 | "楼梯、走廊或出口有没有被烟火堵住？" | evacuationRouteClear, exitBlocked, stairwaySmoke |
| l1.fire.7 | "现场有没有人正在灭火？是否已经断电断气？" | hasInitialFirefighting, powerCut |

#### 水域灾害L1问题

| 节点 | 问题 | 提取字段 |
|------|------|----------|
| l2.water.1 | "是什么类型的水灾？暴雨积水内涝、河水漫出还是山洪？水大概有多深？" | waterDisasterType, waterDepth |
| l2.water.2 | "水位还在上涨吗？水流急不急？被困的人能不能自己转移到安全的地方？" | waterLevelTrend, waterFlowSpeed, canSelfRescue |
| l2.water.3 | "现场有没有电线落水或电器泡在水里的情况？有没有闻到燃气味？" | hasElectrocutionRisk, hasGasLeak |
| l2.water.4 | "附近的房子有没有倒塌风险？是不是地下车库或地下室进水了？" | hasCollapseRisk, isUnderground |
| l2.water.5 | "一共有多少人需要救援？其中有没有老人、小孩或病人？救援车辆能开过去吗？还是需要船？" | trappedCount, hasVulnerablePeople, rescueAccessible |

### L2层（场景专项问题）

#### 高危结构类L2问题

| 节点 | 问题 | 提取字段 |
|------|------|----------|
| l2.highrisk.1 | "起火楼层是第几层？整栋楼大概多少层？" | fireFloor, totalFloors |
| l2.highrisk.2 | "烟是往楼上、楼下，还是向走廊扩散？" | smokeSpreadDirection |

| l2.highrisk.4 | "楼内消防栓是否能用？水压正常吗？" | sprinklerStatus |

| l2.highrisk.6 | "现场是厂房、仓库、地下车库还是商场中庭？" | buildingType |
| l2.highrisk.7 | "现场有没有化学品、油料、气瓶或易燃易爆物？" | hazardousMaterial |
| l2.highrisk.8 | "消防车能不能开到楼下或门口？" | roadAccess |
| l2.highrisk.9 | "现场有没有物业、保安或企业负责人？" | hasCoordinator |

| l2.crowded.6 | "现场是否混乱？有没有拥挤、摔倒或踩踏情况？" | crowdDensity, hasInjury |
| l2.crowded.7 | "是否已经有人受伤或吸入浓烟？" | hasInjury |
| l2.crowded.8 | "现场有没有负责人、保安、老师或医护人员可以配合？" | hasCoordinator |

## 路由决策规则

### 规则1：先问L0，满足最小派警条件

L0的目标不是问全，而是先形成派警条件。L0应采集：位置、警情类型、是否有人被困、燃烧物基本判断。

### 规则2：根据警情类型进入不同主链

完成L0基础识别后，系统进入 incident.router：

| 主路由规则 | 识别结果 | 进入问题链 |
|------------|----------|------------|
| 水域灾害类 | 水灾/暴雨/内涝/地下室进水 | w.1 → w.5 |
| 火灾扑救类 | 着火/冒烟/明火 | l0.fire.1 → l1.fire.7 |
| 应急救援类 | 燃气泄漏/坍塌/电梯困人 | rescue.1 → rescue.router |
| 社会救助类 | 肢体被卡/反锁/动物处置 | assist.1 → assist.router |
| 未明确 | 默认 | 进入火灾链 |

### 规则3：火灾先走L1标准火灾画像

如果识别为火灾扑救类，先进入标准火灾链（L1），相当于"普通火灾画像模板"。

### 规则4：L1后进入建筑场景分类路由

火灾标准信息采集后，系统进入 building.router：

| 建筑路由规则 | 触发关键词 | 进入链路 |
|-------------|-----------|----------|
| 复杂空间与高危结构类 | 高层/地下室/厂房/仓库/化工厂/危化品 | l2.highrisk |
| 人员密集类 | 学校/医院/商场/影院/养老院/酒店/车站 | l2.crowded |
| 重点单位类 | 政府/数据中心/通信枢纽/实验室/配电房 | l2.keyunit |
| 普通场景 | 无特殊标签 | complete |

### 规则5：L2根据差异化风险补问

L2不是所有场景都问，而是根据场景标签和缺失要素补问。只有当building.router识别出特殊场景时才进入对应L2问题链。

### 规则6：有高风险缺失时，打断原顺序优先追问

当出现高风险缺失时，要允许插队：

| 优先级 | 缺失类型 | 示例 |
|--------|----------|------|
| P0 | 位置缺失 | 不知道具体地点 |
| P1 | 生命风险缺失 | 不知道是否有人被困 |
| P2 | 即时危险缺失 | 不知道是否爆炸/触电/毒气/坍塌 |
| P3 | 火势/水位趋势缺失 | 不知道是否扩大 |
| P4 | 调派装备缺失 | 不知道楼层/地下/危化品/通道 |
| P5 | 协同信息缺失 | 不知道负责人/消控室/预案 |

### 规则7：已提供信息不重复问

如果报警人已经明确说了某项信息，不重复问同一问题。

### 规则8：根据报警人类型调整问法

| 报警人类型 | 问法调整 |
|------------|----------|
| 老人/儿童 | 简单直接，避免专业术语 |
| 旁观者 | 先确认位置，再问现场情况 |
| 专业人员 | 可用专业术语，直接问关键点 |
| 当事人 | 安抚情绪，简短明确 |

## 动态追问示例

### 场景：报警人说"我们小区8栋1201厨房着火了，烟很大，我妈还在里面"

**已获得信息**：
- ✅ 地址：小區8栋1201
- ✅ 起火位置：厨房
- ✅ 火情：烟很大
- ✅ 人员：有人被困（母亲）

**不应该再问**：
- ❌ "请问哪里着火？"（已获得）
- ❌ "请问有没有人被困？"（已获得）

**应该优先问**（按优先级）：
1. "老人现在是否还能说话或移动？"（致命：被困人员状态）
2. "门是否能打开？"（辅助：救援通道）
3. "厨房里是否有燃气罐或天然气？"（致命：爆炸风险）
4. "你现在是否已经撤离到安全位置？"（致命：报警人安全）

## 完整对话流程示例（火灾+高层建筑）

### 场景：报警人说"我们小区3号楼18楼着火了，黑烟很大，有两个老人困在里面"

1. **l0.root**：问位置+情况 → 用户说"xx小区3号楼着火"
   → 提取 `incidentType=火灾`，位置 `community=xx小区`，`fireStage` 缺失
   → 因位置已有 `community`，直接进入 `l0.risk`

2. **l0.risk**：问是否有人被困 → 用户说"有两个老人"
   → 提取 `trappedCount=2`，进入 `incident.router`

3. **incident.router**：匹配 "着火" → 火灾分支 → `l0.fire.1`

4. **l0.fire.1**：缺少 `fireStage` → 追问"是只有烟、看到明火，还是火势已经很大？"
   → 用户说"明火很大" → 提取 `fireStage=明火很大` → 进入 `l0.fire.2`

5. **l0.fire.2**：问燃烧物 → 用户说"电动车电池"
   → 提取 `burningMaterial=电动车电池`，`hasDangerousItem=yes` → 进入 `l1.fire.1`

6. **l1.fire.1** → `l1.fire.2` → … → `l1.fire.7`（标准火灾7问）
   → 进入 `building.router`

7. **building.router**：根据"高层""18楼""3号楼"等关键词 → 匹配 `highrisk` → 进入 `l2.highrisk.1`

8. **l2.highrisk.1**：问起火楼层、总层数 → 用户说"18楼，总共26层"
   → 提取 `fireFloor=18, totalFloors=26` → 进入 `l2.highrisk.2`

9. **l2.highrisk.2~9**：依次问完高危专项问题（烟囱效应、消防栓、通道、负责人等）
   → 进入 `complete`

### 场景：水域灾害

1. **l0.root** → **l0.pos** → **l0.risk** → **incident.router**

2. **incident.router**：匹配 "内涝""积水"等 → `l2.water.1`

3. **l2.water.1**：问水灾类型、水深 → 用户说"暴雨积水，地下室进水"
   → 进入 `l2.water.2`

4. **l2.water.2~5**：依次问水位趋势、触电风险、坍塌风险、被困人数
   → 进入 `complete`

### 场景：社会救助（肢体被卡）

1. **l0.root** → **l0.pos** → **l0.risk** → **incident.router**

2. **incident.router**：匹配 "卡住""被卡" → `assist.1`

3. **assist.1**：问具体情况 → 用户说"小孩手卡在玩具里"
   → 提取 `assistanceType=trapped_body` → 进入 `assist.router`

4. **assist.router**：匹配 `trapped_body` → `assist.body.1`

5. **assist.body.1**：问卡住部位、是否肿胀 → 用户说"右手手指，有点肿"
   → 进入 `complete`

## 六场景问题集继承体系

根据场景画像聚类知识，系统问题集采用分层继承结构：

```
┌─────────────────────────────────────────────────────────────┐
│ 极端紧急类 = L0 最小必要信息集                               │
├─────────────────────────────────────────────────────────────┤
│ 普通火灾类 = L0 + L1 标准火灾研判问题集                      │
├─────────────────────────────────────────────────────────────┤
│ 水域灾害类 = L0 + 水灾专项问题集                              │
│ 复杂空间类 = L0 + L1 + 高危结构专项问题集                    │
│ 人员密集类 = L0 + L1 + 人员密集专项问题集                    │
│ 重点单位类 = L0 + L1 + 重点单位专项问题集                    │
└─────────────────────────────────────────────────────────────┘
```

| 继承组合 | 触发条件 | 问题链长度 |
|----------|----------|-----------|
| L0 only | 极端紧急，无法获取位置 | 2-3问 |
| L0 + L1 | 普通火灾、无特殊建筑标签 | 10问 |
| L0 + L1 + 高危结构 | 高层/地下室/厂房/仓库/危化品 | 19问 |
| L0 + L1 + 人员密集 | 学校/医院/商场/影院/养老院 | 18问 |
| L0 + L1 + 重点单位 | 政府/数据中心/实验室 | 18问 |
| L0 + 水域专项 | 水灾/内涝/山洪 | 8问 |
| L0 + 救援专项 | 被困/坍塌/电梯/燃气 | 4-6问 |
| L0 + 救助专项 | 肢体被卡/钥匙/动物 | 4-6问 |
