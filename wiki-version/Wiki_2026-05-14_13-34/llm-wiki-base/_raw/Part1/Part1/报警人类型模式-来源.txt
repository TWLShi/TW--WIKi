### 6. 报警人类型模式：不同场景下，不同身份的人怎么说、容易漏什么

#### 6.1 模块定义（升级）

报警人类型模式是在求助模式库基础上的进一步细化，但它不能脱离场景独立存在。

同一个报警人身份（如物业、当事人、路人），在火灾、水域灾害、危化品泄漏、电梯困人等不同场景中，表达习惯和信息偏差完全不同。

因此，本模块采用“场景 × 报警人类型”的二维结构：
- 纵向是四大警情主场景：火灾扑救、水域灾害、应急救援、社会救助。
- 横向是六类典型报警人：当事人、邻居/同楼人员、路人/旁观者、物业/保安、单位负责人/专业人员、老人儿童/表达弱者。

每个交叉格子里，回答三个问题：
1. 在这种场景下，这类报警人一般会怎么说？
2. 他们最容易主动提供哪些信息？
3. 他们最容易漏掉哪些关键信息？

#### 6.2 场景一：火灾扑救类

| 报警人类型 | 在这种场景下通常怎么说 | 容易主动提供的信息 | 容易漏掉的关键信息 |
| :--- | :--- | :--- | :--- |
| **当事人** | “我家厨房着火了，烟特别大，我出不去！” “卧室在烧，我妈还在里面！” | 起火部位、是否被困、烟气大小、情绪紧迫度 | 标准地址、楼层房号、建筑结构、外部入口、燃烧物性质、是否断气断电 |
| **邻居/同楼人员** | “楼上那家冒烟了，楼道全是烟，我们不敢下楼。” | 楼栋位置、楼层、楼道烟气情况、疏散受阻感受 | 起火户内人数、起火原因、室内危险源、起火点的具体房间 |
| **路人/旁观者** | “那栋楼窗口在往外冒黑烟，不知道几楼。” “路边那个店铺卷帘门缝在冒烟。” | 外部可见现象、大致地标、建筑外观、烟雾颜色 | 内部人员情况、具体楼层、内部结构、是否有人呼救、起火单位名称 |
| **物业/保安** | “8栋12层烟感报警，楼道里有烟，消防通道已经打开。” | 小区/单位名称、楼栋单元、消防通道状态、报警信号、入口指引 | 起火户内人员状态、被困人数、室内燃烧物、是否有危化品 |
| **单位负责人/专业人员** | “我们配电房冒烟有味，已经切断该区域电源，正在组织员工疏散。” | 单位名称、起火区域、危险设备、初期处置、疏散启动 | 被困人员具体位置、外部救援入口、实时火势变化、残火风险 |
| **老人儿童/表达弱者** | “着火了，我怕，门好烫……” “不知道地址，就是那个红色楼上面。” | 危险感受、生命受威胁、对环境的有限观察（如“门烫”“烟辣眼睛”） | 大量结构化字段：地址、火势、燃烧物、通道、自救能力 |

#### 6.3 场景二：水域灾害类（暴雨内涝、洪水、地下空间进水等）

| 报警人类型 | 在这种场景下通常怎么说 | 容易主动提供的信息 | 容易漏掉的关键信息 |
| :--- | :--- | :--- | :--- |
| **当事人** | “地下室进水了，车还在地库，水已经到腰了！” “家里进水了，我们堵不住，一楼全淹了。” | 进水位置、水位高度大致感受、是否危及财产或自身 | 具体地址、水流速度、是否断电、燃气是否关闭、周边是否还有人被困 |
| **邻居/同楼人员** | “我们单元门口水已经齐膝了，路上有好几辆车泡在水里。” | 楼栋周边水情、车辆被淹情况、公共区域积水深度 | 每户室内被困人数、地下空间的详细情况、配电房是否进水 |
| **路人/旁观者** | “前面桥洞下面水特别深，有辆车淹了一半，好像有人。” “河道水漫出来了，路边那个小区进水了。” | 具体积水地点、车辆被淹、河水漫溢、地标位置 | 被困车内人数、被困人员状态、是否漏电、水流速度 |
| **物业/保安** | “小区地下车库进水，排水泵已经不够用了，我们在通知业主挪车。” | 进水区域、排水措施、车辆转移情况、已采取的行动 | 地库内是否有人未撤离、配电房或燃气调压箱是否涉水、进水趋势 |
| **单位负责人/专业人员** | “厂区低洼车间进水，已经切断设备电源，正在转移危险物料。” | 进水车间位置、断电情况、危险物料转移、员工撤离进度 | 具体被困人数、进水源和趋势、是否影响周边环境 |
| **老人儿童/表达弱者** | “水进来了，我走不动，出不去……” “水快要到床了，我害怕。” | 生命危险、行动能力受限、情绪恐慌 | 精确地址、周边环境、有无触电或燃气风险、水位上涨速度 |

#### 6.4 场景三：应急救援类（危险化学品、燃气泄漏、坍塌、电梯困人等）

| 报警人类型 | 在这种场景下通常怎么说 | 容易主动提供的信息 | 容易漏掉的关键信息 |
| :--- | :--- | :--- | :--- |
| **当事人（燃气泄漏/被困）** | “家里全是煤气味，我开了一下灯就听见砰一声！” “电梯突然停了，里面五六个人，有人喘不上气。” | 异常气味、声响、被困位置、被困人数、身体不适 | 燃气泄漏源是否关闭、单元楼层、电梯编号、是否有心脏病等基础病患 |
| **邻居/同楼人员** | “楼道里有很浓的煤气味，不知道是哪家。” “电梯卡在五楼，里面有人拍门。” | 公共区域异常、大致楼层、异常持续情况 | 泄漏点具体位置、电梯内人员状态准确情况、是否需要同步医疗 |
| **路人/旁观者** | “那个工地围挡里面塌了一块，有个人被压在下面！” “路边有个罐车在冒白烟，味道很呛。” | 事故外部特征、涉事车辆或工地大致地标、可见被困人员 | 塌方深度、被困人准确位置、危化品类型、罐体是否破裂、周边是否还有人 |
| **物业/保安** | “燃气总阀已经关了，是中压管道有泄漏声，范围在3号楼附近。” “电梯困在B2，里面有老人，我们已经联系电梯维保。” | 处置措施、影响范围、电梯内大致情况、专业人员到场情况 | 电梯内是否完全停电、是否有孕妇或急症患者、漏气点是否靠近火源 |
| **单位负责人/专业人员** | “实验室有剧毒气体泄漏报警，区域已封锁，正在疏散，泄漏点是气瓶柜。” “工地基坑支护变形，有坍塌征兆，人员已撤离但清点少一人。” | 泄漏物质、处置动作、疏散状态、业务术语 | 外界救援接口、具体被困或失联人位置、事故对周边的影响 |
| **老人儿童/表达弱者** | “有臭味，我头晕，不知道该怎么办……” “电梯不动了，我怕……” | 症状感受、恐慌 | 地址、泄漏源、电梯编号、是否有人同乘、有无自救可能 |

#### 6.5 场景四：社会救助类（肢体被卡、取钥匙、马蜂窝、动物处置等）

| 报警人类型 | 在这种场景下通常怎么说 | 容易主动提供的信息 | 容易漏掉的关键信息 |
| :--- | :--- | :--- | :--- |
| **当事人** | “我手指卡在戒指里拔不出来，手指都发紫了。” “钥匙忘在家里面了，有个小孩反锁在屋里。” | 被卡部位、被卡物品、屋内人员情况、紧迫程度 | 确切地址、楼层高度、防盗门类型、手指肿胀时长、是否有基础病 |
| **邻居/同楼人员** | “我邻居家钥匙锁里面了，正在找开锁的，也报了你们。” | 当事人大致情况、现场协助意愿 | 屋内是否有火源或风险、被困人员年龄和状态、门锁特殊结构 |
| **路人/旁观者** | “那边树上有个马蜂窝，下面好多人经过。” “路边广告牌快掉下来了。” | 隐患位置、物体类型、对公共影响 | 蜂种大小、是否有人被蜇、广告牌归属、掉落风险高度和范围 |
| **物业/保安** | “业主说手指卡在儿童座椅孔里，我们没办法破拆。” “二楼阳台有个马蜂窝，业主不敢开窗。” | 现场尝试了什么、专业工具需求、具体房号 | 儿童年龄、是否已受伤、马蜂窝准确位置及摘除难度 |
| **单位负责人/专业人员** | “我们食堂一个阿姨手指卡在绞肉机料斗口，已经停机但取不出来。” | 机器类型、已停机、被困肢体情况、医疗需求 | 机器结构、破拆空间、是否需要工程机械辅助 |
| **老人儿童/表达弱者** | “奶奶的手肿了好大，你们快来……” “我房门打不开了，我害怕一个人在家。” | 人员类型、害怕、大致情况 | 准确地址、楼层、门锁类型、是否有人能协助开门或安抚 |

#### 6.6 这一模块如何作用于路由

升级后的“场景×报警人类型”模式，在系统中的实际价值是：
- **先匹配场景**：报警人开口后，系统根据关键词和求助模式库快速确认是火灾、水域、应急救援还是社会救助。
- **再定位报警人类型**：根据用词、信息密度、角色词（“我们小区”“我们单位”“我看见”“我家里”），判断报警人身份。
- **调用对应格子的信息偏差模型**：系统就知道“这个场景下，这个人通常会漏掉什么”，从而优先追问那些高缺失概率、高风险的字段。

例如：
- 在火灾场景下，路人说“窗口冒黑烟”，系统会立刻知道路人漏掉了内部人员信息、准确楼层和起火单位，所以追问序列倾向“能否看到有人呼救？”“能确认是几楼吗？”
- 在水灾场景下，物业说“地库进水了，我们在挪车”，系统会警觉物业常漏掉“地库里是否还有人没出来”，所以会优先追问“地库里还有没有没上来的业主？”
- 在燃气泄漏场景下，邻居说“楼道有煤气味”，系统会识别出邻居大概率不知道漏气源头，追问会优先锁定“哪层味道最重，哪户可能没人”。

这样，**问题路由的“下一步问什么”不仅依赖缺失字段的优先级，还受报警人类型的信息可信度与遗漏特征调节，从而提高问话的针对性和效率**。
