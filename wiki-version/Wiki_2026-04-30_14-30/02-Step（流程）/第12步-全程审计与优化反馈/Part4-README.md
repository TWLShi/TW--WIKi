## 4. 责任闭环（领域事件流）

```
AI辅助决策 → 人工最终确认 → 方案锁定 → 审计留痕 → 责任追溯
     ↓
[Step9: CommandGenerated] → [Step10: HumanApproved] → [Step11: DispatchExecuted] → [Step12: AuditRecorded]
```

### 责任主体映射

| 步骤 | 领域事件 | 主要责任人 | 责任类型 |
|------|----------|-----------|----------|
| 第1-2步 | `AlertReceived` → `QueryInitiated` | 接警员 | 信息采集 |
| 第3-8步 | `PortraitBuilt` → `ConflictDetected` | AI系统 | 分析与方案生成 |
| 第10步 | `HumanApproved` | 调度员 | 最终确认（关键责任关口） |
| 第11-12步 | `DispatchExecuted` → `AuditRecorded` | 系统 | 执行与审计 |

## 5. 审计日志设计（领域服务）

| 字段 | 说明 | 领域语义 |
|------|------|----------|
| `trace_hash` | SHA-256区块链式哈希链 | 不可篡改证据 |
| `responsibility` | 每个步骤的责任人 + 贡献度 | 责任归属 |
| `human_modifications` | 所有修改记录（old/new/reason） | 变更追溯 |
| `timestamp` | 不可篡改时间戳 | 时间线重建 |

## 6. 数学模型体系（领域服务）

| 公式 | 描述 | 约束 |
|------|------|------|
