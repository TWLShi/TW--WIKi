# FireCommand AI 消防接处警 SOP 表单设计

**标签**：#SOP #接处警 #表单设计 #FireCommand #指挥模块
**位置**：01_Requirements/

---

## 表单五大区块

| 区块 | 性质 | 说明 |
|------|------|------|
| **区块1：基本接警信息** | 必填，快速录入 | 报警时间/方式/人/位置/类型/等级 |
| **区块2：灾情详细描述** | 强烈建议，影响指挥方案 | 报警人描述/建筑类型/人员情况/危险源/状态 |
| **区块3：资源与环境评估** | AI 部分自动补充 | 天气环境/风险/已出动/可用资源 |
| **区块4：指挥偏好与限制** | 核心个性化部分 | 指挥模式/优先考量/时间敏感度/特殊限制 |
| **区块5：反馈与跟进** | 支持实时更新 | 现场反馈/调整记录/等级判定/报告需求 |

---

## 区块1：基本接警信息（必填）

| 字段 | 类型 | 示例 |
|------|------|------|
| 报警时间 | 自动时间戳 | 2026-04-04 14:32:15 |
| 报警方式 | 下拉选择 | 119电话 / APP报警 / 自动火警系统 / 其他 |
| 报警人信息 | 文字 + 电话 | 张先生 / 0912-345-678 |
| 事件位置 | 文字 + 地图选点 | 臺中市西屯區XX路XX號 |
| 事件类型 | 下拉多选 | 火灾 / 救援 / 危化品泄漏 / 交通事故 / 群众事件 / 其他 |
| 火势/灾情等级 | 下拉选择 | 一般 / 较大 / 重大 / 特别重大 |

---

## 区块4：指挥偏好与限制（核心个性化）

| 字段 | 类型 | 示例 |
|------|------|------|
| 指挥模式偏好 | 下拉 | 进攻模式 / 防御模式 / 调查模式 |
| 优先考量 | 多选 | 生命救助优先 / 财产保护 / 环境防护 / 快速控制火势 |
| 时间敏感度 | 选择 | 高（需最快到达）/ 中 / 低 |
| 特殊限制或注意事项 | 文字 | 「该区域历史上有多次电气火灾」「避免使用某条道路」 |
| 指挥员/值班长备注 | 文字 | 「请优先保护东侧居民楼」 |

---

## 设计原则

| 原则 | 说明 |
|------|------|
| **AI 友好** | 所有字段结构化，方便转成 Prompt |
| **人机协作** | 接警员填前半部，AI 辅助补充分析，指挥员可修改确认 |
| **实时反馈** | WebSocket 更新现场反馈，AI 自动调整方案 |
| **审计友好** | 所有填写与修改记录时间戳与操作人 |
