## 1. Ollama自动打标Prompt（三模块统一版）

**使用说明**：
- 输入：原始接警记录、研判笔记草稿或调度反馈文本
- 输出：结构化JSON（可直接解析为Frontmatter + tags）
- 推荐模型：qwen2.5、llama3.1、gemma2
- 温度：0.1~0.3

**JSON Schema**：
```json
{
  "module": "接警问询 | 警情研判 | 资源调度",
  "tags": ["#tag1", "#tag2", ...],
  "frontmatter": {
    "title": "字符串，简洁描述",
    "status": "proposed | accepted",
    "version": "1.0",
    "date": "YYYY-MM-DD",
    "current_state": "状态名称",
    "transition": "Transition名称",
    "related_incident": "[[Incident-XXXX-简要描述-聚合根]]",
    "risk_level": "低 | 中 | 中高 | 高 | 极高",
    "confidence": 数字0-100,
    "additional_fields": {}
  },
  "summary": "一句话摘要"
}
```

## 2. Incident聚合根模板

**文件夹**：`00-Incident/Incident-2026-XXXX-事件类型-聚合根.md`

```yaml
---
title: "Incident-聚合根"
status: active
version: "1.0"
date: "2026-04-13"
tags: [#topic/状态机, #topic/警情研判, #topic/资源调度]
current_state: "调度执行中"
risk_level: "高"
---

# 全流程状态机

1. **接警问询阶段**
   - Inquiring → Assessed（#transition/Inquiring→Assessed）
   - 关联笔记：[[接警记录-2026-XXXX]]

2. **警情研判阶段**
   - Preliminary → Deep → Confirmed（#transition/Deep→Confirmed）
   - 关联笔记：[[研判-2026-XXXX]]

3. **资源调度阶段**
   - Confirmed → Pending → Assigned → Enroute → Arrived → Executing → Completed
   - 关联笔记：[[调度-2026-XXXX]]

# 关联笔记查询
```dataview
LIST
WHERE related_incident = [[Incident-2026-XXXX]]
SORT file.mtime DESC
```
```

**状态迁移规则**：
- 只有前一模块达到"输出状态"（Assessed/Confirmed/Assigned）才能推进下一模块
- 出现 #transition/rollback 时，必须在聚合根记录回退原因

## 3. Dataview三模块综合仪表板

```dataview
# 警情处置闭环仪表板

## 1. 待研判清单（接警已完成）
TABLE current_state AS "状态", risk_level_suggestion AS "建议风险", file.mtime AS "更新时间"
FROM "02-接警问询"
WHERE contains(tags, "#state/Assessed") AND contains(tags, "#topic/警情研判")
SORT file.mtime DESC
LIMIT 10

## 2. 研判待确认 / 高危警情
TABLE current_state AS "状态", risk_level AS "风险", confidence AS "置信度"
FROM "03-研判"
WHERE contains(tags, "#topic/警情研判") AND (contains(tags, "#state/Preliminary") OR contains(tags, "#value/critical"))
SORT risk_level DESC

## 3. 资源调度执行监控
TABLE current_state AS "调度状态", resources AS "资源", eta AS "ETA"
FROM "04-调度"
WHERE contains(tags, "#topic/资源调度") AND !contains(tags, "#state/Completed")
SORT eta ASC

## 4. 全流程高危Incident概览
TABLE current_state AS "当前总状态", risk_level AS "风险", file.mtime AS "最后更新"
FROM "00-Incident"
WHERE contains(tags, "#topic/状态机") AND risk_level = "高"
SORT file.mtime DESC
```

## 闭环成熟度

| 维度 | 状态 |
|------|------|
| 数据流转 | 接警Assessed → 研判Confirmed → 调度Assigned（已打通） |
| 可追溯性 | 全部通过 related_incident 聚合根 |
| 自动化 | Ollama打标 + Dataview仪表板 |
